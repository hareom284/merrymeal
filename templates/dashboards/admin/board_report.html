{% comment %} Story 6.5 — Board report template. Dual-purpose: * Served as ``text/html`` to the browser when ``?format=html`` (the default) — the admin uses File → Print → Save as PDF as a built-in fallback when the server-side WeasyPrint isn't available. * Rendered to PDF bytes by ``services.board_report_pdf.render_pdf`` in the Docker image where WeasyPrint and its native libs are installed. The page is deliberately self-contained: inline CSS, no external font loads, no JavaScript. WeasyPrint does not run JS and we don't want a print-time network round-trip to a Google Fonts CDN. ``for_pdf`` is True when the PDF renderer invokes the template; we use that to suppress the "Print to PDF" instructions and the link back to the admin home, both of which would look out of place in a saved PDF. {% endcomment %} Board report — {{ report.period_label }} · MerryMeal

{% comment %} Service worker registration + install hints. Two install paths because the two mobile platforms behave differently: * **Android / desktop Chrome / Edge.** Fires ``beforeinstallprompt`` once the PWA install criteria are met (manifest + SW + icon + start_url reachable). We stash the event and expose an "Install app" pill — Chrome no longer auto-shows the URL-bar install icon on mobile, so without this pill the install flow is invisible. * **iOS Safari / WebKit.** Does NOT fire ``beforeinstallprompt`` and has no programmatic install API. The only path is the user tapping Share → Add to Home Screen themselves. We detect iOS, hide the hint once installed (``navigator.standalone === true``), and let the user dismiss the hint to localStorage so it doesn't pester them every visit. Both hints are bottom-anchored, single-row, and respect ``safe-area-inset-bottom`` so they don't sit under the iOS gesture bar or the Android nav bar. ``pointer-events-none`` on the wrapper with ``pointer-events-auto`` on the children means the pill is tappable but doesn't block clicks on the rest of the page. {% endcomment %}

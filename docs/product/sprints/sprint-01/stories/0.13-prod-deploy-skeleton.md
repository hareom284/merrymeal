## Task breakdown

### Task 1: Production compose override

- [ ] **Step 1.1 — Create `compose.prod.yaml`.**

```yaml
# Layered on top of compose.yaml. Bring up with:
#   docker compose -f compose.yaml -f compose.prod.yaml up -d

services:
  web:
    restart: always
    environment:
      DJANGO_SETTINGS_MODULE: config.settings.prod
      DJANGO_DEBUG: "False"
      DJANGO_SECURE_SSL_REDIRECT: "True"
      DJANGO_SESSION_COOKIE_SECURE: "True"
      DJANGO_CSRF_COOKIE_SECURE: "True"
      DJANGO_SECURE_HSTS_SECONDS: "31536000"  # 1 year
      DJANGO_SECURE_HSTS_INCLUDE_SUBDOMAINS: "True"
      DJANGO_SECURE_HSTS_PRELOAD: "True"
    ports: !reset []   # production exposes only via nginx, not directly
    expose:
      - "8000"
    command:
      - gunicorn
      - config.wsgi:application
      - --bind=0.0.0.0:8000
      - --workers=4
      - --access-logfile=-
      - --error-logfile=-

  worker:
    restart: always
    environment:
      DJANGO_SETTINGS_MODULE: config.settings.prod

  mysql:
    restart: always
    # In prod, point this at managed MySQL (RDS / DigitalOcean DB) instead.
    # The volume below is fine for single-VPS deploys; swap for an external
    # host once you migrate.

  redis:
    restart: always
```

> Verify your Compose CLI supports `!reset` for list overrides (Compose ≥ v2.24).
> If not, document a manual port-removal step in `docs/deploy.md` instead.

### Task 2: Nginx sample

- [ ] **Step 2.1 — Create `deploy/nginx.conf.example`.**

```nginx
# /etc/nginx/sites-available/merrymeal — sample, NOT auto-loaded.
# Copy to /etc/nginx/sites-enabled/, replace YOUR_DOMAIN, then `nginx -t && systemctl reload nginx`.

upstream merrymeal_web {
    server 127.0.0.1:8000 fail_timeout=0;
}

# HTTP → HTTPS redirect.
server {
    listen 80;
    server_name YOUR_DOMAIN;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name YOUR_DOMAIN;

    ssl_certificate     /etc/letsencrypt/live/YOUR_DOMAIN/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/YOUR_DOMAIN/privkey.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;
    ssl_prefer_server_ciphers on;

    # Tightening — match Django's HSTS settings in compose.prod.yaml.
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains; preload" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    client_max_body_size 25M;          # proof-of-delivery photos in Epic 04
    keepalive_timeout 65;

    gzip on;
    gzip_types text/plain text/css application/json application/javascript image/svg+xml;
    gzip_min_length 1024;

    # Static files served straight from disk.
    location /static/ {
        alias /srv/merrymeal/static/dist/;
        expires 1y;
        access_log off;
        add_header Cache-Control "public, immutable";
    }

    location /media/ {
        # Phase 0 only — Epic 04 moves media to S3 via django-storages.
        alias /srv/merrymeal/media/;
        expires 30d;
    }

    location / {
        proxy_pass http://merrymeal_web;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 30s;
    }
}
```

### Task 3: Systemd unit sample

- [ ] **Step 3.1 — Create `deploy/gunicorn.service.example`.**

```ini
# /etc/systemd/system/merrymeal-gunicorn.service — sample.
# Install with:
#   sudo cp deploy/gunicorn.service.example /etc/systemd/system/merrymeal-gunicorn.service
#   sudo systemctl daemon-reload
#   sudo systemctl enable --now merrymeal-gunicorn

[Unit]
Description=MerryMeal Gunicorn
After=network.target mysql.service

[Service]
Type=notify
User=merrymeal
Group=merrymeal
WorkingDirectory=/srv/merrymeal
EnvironmentFile=/etc/merrymeal/env
ExecStart=/srv/merrymeal/.venv/bin/gunicorn \
    config.wsgi:application \
    --bind 127.0.0.1:8000 \
    --workers 4 \
    --access-logfile - \
    --error-logfile -
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

### Task 5: `docs/deploy.md` runbook

- [ ] **Step 5.1 — Create `docs/deploy.md`.**

```markdown
# Production deploy — single-VPS Docker recipe

> Target: a fresh Ubuntu 24.04 VPS (e.g. Hetzner CX21, 2 vCPU / 4 GB RAM).
> Time-to-live target: 30 minutes.

## 1. Provision

Create the VPS in your provider's UI. SSH in as root (or via a sudoer).

## 2. Install Docker

```bash
sudo apt update && sudo apt install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu noble stable" \
  | sudo tee /etc/apt/sources.list.d/docker.list
sudo apt update && sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
```

## 3. Clone and configure

```bash
sudo mkdir -p /srv && cd /srv
sudo git clone <repo-url> merrymeal
cd merrymeal
sudo cp .env.prod.example .env
sudo nano .env   # fill in real values
```

## 4. Bring up the stack

```bash
sudo docker compose -f compose.yaml -f compose.prod.yaml up -d --build
```

The first build takes 3–4 minutes. Once `docker compose ps` shows all
services healthy:

```bash
sudo docker compose run --rm web python manage.py createsuperuser
```

## 5. Reverse proxy + TLS

Install nginx + certbot:

```bash
sudo apt install -y nginx certbot python3-certbot-nginx
sudo cp /srv/merrymeal/deploy/nginx.conf.example /etc/nginx/sites-available/merrymeal
sudo nano /etc/nginx/sites-available/merrymeal   # replace YOUR_DOMAIN
sudo ln -s /etc/nginx/sites-available/merrymeal /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
sudo certbot --nginx -d YOUR_DOMAIN
```

Certbot adds the SSL block automatically and sets up auto-renewal.

## 6. Smoke test

From your laptop:

```bash
curl -I https://YOUR_DOMAIN/accounts/login/
```

Expected: `HTTP/2 200`, `strict-transport-security` header present.

## 7. Verify Django's deploy checks

```bash
sudo docker compose run --rm web python manage.py check --deploy
```

Expected: 0 issues. If any warning is reported, fix the corresponding
setting in `.env` and restart `web`.

## Daily ops

- **Pull updates:**
  `cd /srv/merrymeal && sudo git pull && sudo docker compose -f compose.yaml -f compose.prod.yaml up -d --build`
- **Tail logs:** `sudo docker compose logs -f web worker`
- **DB backup:** see Story 7.8 (Epic 07) for the nightly backup script.

## What's NOT in this recipe (and why)

- **Managed MySQL.** Use it once the charity scales beyond ~500 members; the docker MySQL service is fine for v1.
- **CDN / WAF.** Add once Story 5.4 (Stripe) is live and donor traffic justifies it.
- **Multi-VPS / load balancer.** Out of scope until uptime SLAs are signed.

## Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| `502 Bad Gateway` from nginx | `web` container down or healthcheck not yet green | `docker compose ps` — restart `web`; check `docker compose logs web` for the migration error. |
| `Bad Request (400)` after deploy | `DJANGO_ALLOWED_HOSTS` doesn't include your domain | Edit `.env`, restart `web`. |
| `Mixed content` warnings in browser | `DJANGO_SECURE_SSL_REDIRECT` not enabled or nginx not forwarding `X-Forwarded-Proto` | Confirm `proxy_set_header X-Forwarded-Proto $scheme;` is present in nginx config. |
```

## Pitfalls reviewer will check

- **`!reset` Compose syntax** — works only on Compose ≥ v2.24. Document the minimum version.
- **`X-Forwarded-Proto` forwarding** — without it, Django thinks every request is HTTP and HSTS doesn't kick in.
- **`client_max_body_size`** — proof-of-delivery photos in Epic 04 are larger than nginx's 1 MB default. 25 MB is the picked size.
- **`SECURE_PROXY_SSL_HEADER`** — Django needs `SECURE_PROXY_SSL_HEADER = ('HTTP_X_FORWARDED_PROTO', 'https')` in `prod.py` for the redirect logic to trust nginx.

# Story 0.10 — GitHub Actions CI

**Epic:** [00 — Foundation](../../../epics/00-foundation.md)
**Sprint:** 01
**Status:** planned (sprint-01)
**Estimated effort:** 1 day
**Track:** Ops
**Depends on:** Story 0.9 (the test job uses the same MySQL image as compose)
**Blocks:** Story 0.11 (the Playwright job runs after the smoke job in CI)

---

## User story

> **As a** developer reviewing a PR
> **I want** CI to tell me the PR is safe to merge
> **so that** I do not need to run lint, tests, and the frontend build by hand.

## Acceptance criteria

- [ ] `.github/workflows/ci.yml` runs on every `push` and `pull_request` to `main`.
- [ ] Three jobs run in parallel (no inter-job dependency except the docker-smoke job which depends on `test`):
  - `lint` — `ruff check .`
  - `test` — `pytest` against a MySQL 8 service container; coverage report uploaded.
  - `frontend` — `npm ci && npm run build:css`; fails if compiled CSS changed (git diff).
- [ ] A fourth `docker-smoke` job runs `./tests_e2e/docker_smoke.sh` on `pull_request` only.
- [ ] All four jobs are marked **required** in branch protection (configured manually after merge).
- [ ] Caching: pip cache keyed on `requirements.txt` hash; npm cache keyed on `package-lock.json` hash.
- [ ] Coverage artefact uploaded for the latest run.

## Files to create or modify

| File | Action | Purpose |
|---|---|---|

| `.github/workflows/README.md` | create | One-paragraph explanation of the jobs. |

## Task breakdown

### Task 1: Create the workflow file (RED)

- [ ] **Step 1.1 — Create `.github/workflows/ci.yml`.**

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs:
  lint:
    name: Lint (ruff)
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
          cache: pip
      - run: pip install -r requirements.txt
      - run: ruff check .

  test:
    name: Test (pytest)
    runs-on: ubuntu-latest
    services:
      mysql:
        image: mysql:8
        env:
          MYSQL_DATABASE: merrymeal
          MYSQL_USER: merrymeal
          MYSQL_PASSWORD: merrymeal
          MYSQL_ROOT_PASSWORD: rootpass
        ports:
          - 3306:3306
        options: >-
          --health-cmd="mysqladmin ping -h localhost -u merrymeal -pmerrymeal"
          --health-interval=5s
          --health-timeout=5s
          --health-retries=12
    env:
      DJANGO_SETTINGS_MODULE: config.settings.test
      MYSQL_HOST: 127.0.0.1
      MYSQL_DATABASE: merrymeal
      MYSQL_USER: merrymeal
      MYSQL_PASSWORD: merrymeal
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
          cache: pip
      - run: |
          sudo apt-get update
          sudo apt-get install -y default-libmysqlclient-dev pkg-config
      - run: pip install -r requirements.txt
      - run: pytest --cov=apps --cov-report=xml --cov-report=term
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: coverage
          path: coverage.xml

  frontend:
    name: Frontend (Tailwind build)
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: npm
      - run: npm ci
      - run: npm run build:css
      - name: Fail if compiled CSS changed
        run: |
          if ! git diff --quiet -- static/dist; then
            echo "::error::Compiled CSS differs from committed version."
            git --no-pager diff -- static/dist
            exit 1
          fi

  docker-smoke:
    name: Docker smoke
    if: github.event_name == 'pull_request'
    runs-on: ubuntu-latest
    needs: [test]
    steps:
      - uses: actions/checkout@v4
      - run: cp .env.example .env
      - run: ./tests_e2e/docker_smoke.sh
```

- [ ] **Step 1.2 — Create `.github/workflows/README.md`.**

```markdown
# CI workflows

| Job | Runs on | What it checks |
|---|---|---|
| lint | push + PR | `ruff check .` |
| test | push + PR | `pytest` against a MySQL 8 service container; uploads `coverage.xml`. |
| frontend | push + PR | `npm ci && npm run build:css`; fails if `static/dist` differs from committed CSS. |
| docker-smoke | PR only | Brings up `compose.yaml` and confirms the login page is reachable within 60 s. |

All four jobs are required for merge to `main` (enforce in branch
protection: Settings → Branches → Add rule).
```

### Task 2: Validate the workflow

- [ ] **Step 2.1 — Push to a draft PR branch.**

```bash
git checkout -b epic-0/story-10-ci
git add .github/workflows/
git commit -m "ci: add lint + test + frontend + docker-smoke jobs [0.10]"
git push -u origin epic-0/story-10-ci
gh pr create --draft --title "[0.10] CI workflow (draft)" --body "Draft to validate the matrix."
```

- [ ] **Step 2.2 — Watch the run.**

```bash
gh run watch
```

**Expected:** all four jobs go green.

- [ ] **Step 2.3 — If any job fails, fix in place and push.**

Common failures and fixes:

| Failure | Fix |
|---|---|
| `mysqlclient` install fails | Apt step missing `default-libmysqlclient-dev` and `pkg-config`. Already in the workflow above — verify line not lost. |
| `ruff check` finds new errors | Fix lint errors first (do not silence them). |
| `frontend` says CSS differs | Run `npm run build:css` locally, commit the regenerated `static/dist/*.css`. |
| `docker-smoke` times out | Re-run; if persistent, increase the smoke script's poll deadline from 60 → 90 s. |

### Task 3: Demonstrate failure detection (RED-style validation)

- [ ] **Step 3.1 — Make a deliberately-bad commit.**

On the same branch:

```bash
# Force a lint error.
echo "import os, sys  # noqa" >> apps/core/decorators.py  # remove `# noqa` to make it fail
git add apps/core/decorators.py
git commit -m "ci: prove lint job catches errors"
git push
```

- [ ] **Step 3.2 — Confirm `lint` job goes red.**

```bash
gh run watch
```

**Expected:** `lint` fails with a `ruff` error.

- [ ] **Step 3.3 — Revert the bad commit.**

```bash
git revert HEAD
git push
```

Confirm CI goes green again.

### Task 4: Mark PR ready + merge

- [ ] **Step 4.1 — Mark the PR ready.**

```bash
gh pr ready
```

- [ ] **Step 4.2 — Get review + merge.**

After teammate review, merge via the GitHub UI (or `gh pr merge --squash`).

- [ ] **Step 4.3 — Enable branch protection (one-time manual step).**

In the GitHub repo settings:

- Settings → Branches → Branch protection rules → Add rule
- Branch name pattern: `main`
- ✅ Require status checks to pass before merging
- ✅ Required checks: `lint`, `test`, `frontend`, `docker-smoke`
- ✅ Require branches to be up to date before merging
- ✅ Require pull request before merging (1 reviewer)

Take a screenshot of the configured rule and attach it to the closing
PR comment so the team can audit later.

## Pitfalls reviewer will check

- **Service-container DB host inside the test job is `127.0.0.1`, not `mysql`** — service containers are reached via the runner's loopback, not by service name as in compose.
- **Caching key includes the lockfile hash** — otherwise the cache stays stale across dependency upgrades.
- **`concurrency` block cancels older runs** — saves CI minutes on rapid pushes.
- **`docker-smoke` only on PR** — avoids running it twice on each push-then-PR (the push-to-main case still runs the other three).

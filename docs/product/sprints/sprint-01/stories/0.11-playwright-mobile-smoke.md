## User story

> **As** the MerryMeal team
> **I want** one Playwright test that opens the login screen at 375 × 667 px
> **so that** the mobile-first promise is enforced by CI on every PR.

## Acceptance criteria

- [ ] `tests_e2e/playwright.config.ts` configures the viewport to `{ width: 375, height: 667 }` (iPhone SE).
- [ ] `tests_e2e/login.spec.ts` opens `/accounts/login/` and asserts:
  - The email input is visible.
  - The email input has `clientHeight >= 44` (touch-target rule).
  - The submit button has `clientHeight >= 44`.
  - No horizontal scrollbar is present (`document.documentElement.scrollWidth <= 375`).
- [ ] `npm run test:e2e` runs the suite headless locally.
- [ ] A new CI job `playwright` runs the suite against the Docker stack and is required for merge.

- [ ] **Step 1.3 — Create `tests_e2e/playwright.config.ts`.**

```typescript
import { defineConfig, devices } from '@playwright/test';

export default defineConfig({
  testDir: '.',
  timeout: 30_000,
  fullyParallel: false,         // single worker — the login form is a single screen
  retries: process.env.CI ? 2 : 0,
  reporter: process.env.CI ? [['github'], ['html', { open: 'never' }]] : 'list',
  use: {
    baseURL: process.env.BASE_URL ?? 'http://localhost:8000',
    viewport: { width: 375, height: 667 },   // iPhone SE — mobile-first floor
    trace: 'on-first-retry',
    screenshot: 'only-on-failure',
  },
  projects: [
    {
      name: 'mobile-chrome',
      use: { ...devices['iPhone SE'] },
    },
  ],
});
```

- [ ] **Step 1.4 — Write the failing test in `tests_e2e/login.spec.ts`.**

```typescript
import { test, expect } from '@playwright/test';

test.describe('Login screen — mobile viewport floor (375 × 667)', () => {
  test('email input is visible and tappable', async ({ page }) => {
    await page.goto('/accounts/login/');

    const email = page.getByLabel(/email/i);
    await expect(email).toBeVisible();

    const emailBox = await email.boundingBox();
    expect(emailBox).not.toBeNull();
    expect(emailBox!.height).toBeGreaterThanOrEqual(44);
  });

  test('submit button is in the thumb-zone and ≥ 44 px tall', async ({ page }) => {
    await page.goto('/accounts/login/');
    const submit = page.getByRole('button', { name: /log in|sign in/i });
    await expect(submit).toBeVisible();
    const box = await submit.boundingBox();
    expect(box!.height).toBeGreaterThanOrEqual(44);
  });

  test('no horizontal scrollbar at 375 px', async ({ page }) => {
    await page.goto('/accounts/login/');
    const scrollWidth = await page.evaluate(() => document.documentElement.scrollWidth);
    expect(scrollWidth).toBeLessThanOrEqual(375);
  });
});
```

- [ ] **Step 2.2 — Run the suite.**

```bash
npm run test:e2e
```

**Expected:** 3 passed.

If a test fails because the login template lacks an explicit `aria-label="Email"` or the button text does not match `/log in|sign in/i`, **fix the template**, not the test. The mobile-first contract belongs to the UI, not the test.

If horizontal scrollbar test fails: a parent container is wider than the viewport. Common causes:
- A `min-w-[420px]` Tailwind class on a wrapper.
- An `<img>` without `max-w-full`.
- A `.input` class set to a fixed `width: 400px` somewhere.

### Task 3: Add the CI job

- [ ] **Step 3.1 — Edit `.github/workflows/ci.yml`** to add a `playwright` job:

```yaml
  playwright:
    name: Playwright (mobile 375 px)
    if: github.event_name == 'pull_request'
    runs-on: ubuntu-latest
    needs: [docker-smoke]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "20"
      - name: Start app stack
        run: |
          cp .env.example .env
          docker compose up -d --build
          until curl -fs http://localhost:8000/accounts/login/ >/dev/null; do sleep 2; done
      - name: Install Playwright
        working-directory: tests_e2e
        run: |
          npm ci
          npx playwright install --with-deps chromium
      - name: Run mobile-viewport tests
        run: npm run test:e2e
      - name: Upload Playwright report
        if: failure()
        uses: actions/upload-artifact@v4
        with:
          name: playwright-report
          path: tests_e2e/playwright-report
```

## Pitfalls reviewer will check

- **`viewport` in `config.ts` AND in `devices['iPhone SE']`** — `devices` already sets it; do not contradict.
- **`fullyParallel: false`** — a single worker keeps logs readable. If/when we have ≥ 10 tests, revisit.
- **Test selectors use roles + accessible names**, not CSS classes — survives Tailwind refactors.
- **Browsers are installed in CI** — `npx playwright install --with-deps chromium`. Without `--with-deps`, system libs are missing on the ubuntu runner.

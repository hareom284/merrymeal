## User story

> **As a** developer joining the MerryMeal project
> **I want** to run `docker compose up` and reach a working app
> **so that** I do not have to install MySQL or Redis on my host, and my local stack matches CI matches prod.

## Acceptance criteria

- [ ] `compose.yaml` at repo root with services: `web`, `mysql`, `redis`, `worker`.
- [ ] `web` reads `.env`, waits for `mysql` healthcheck, runs migrations on start, serves on host port 8000.
- [ ] `mysql` uses MySQL 8 image, persists to a named volume `mysql_data`, env vars match `.env.example`.
- [ ] `redis` uses Redis 7-alpine.
- [ ] `worker` runs `python manage.py qcluster` against the same image as `web`.
- [ ] `docker compose up` from a fresh clone (after `cp .env.example .env`) reaches `http://localhost:8000/accounts/login/` (HTTP 200) within 60 seconds.
- [ ] `README.md` gains a "Setup — Docker" section with a 5-line quickstart.

## Tech / library notes

- Base image for runtime: `python:3.12-slim-bookworm` (Debian-based; mysqlclient builds cleanly).
- Base image for Tailwind build: `node:20-alpine`.
- Healthcheck for MySQL: `mysqladmin ping -h localhost -u $MYSQL_USER -p$MYSQL_PASSWORD`.
- Healthcheck for web: `curl -fs http://localhost:8000/accounts/login/` (curl already in slim image; if not, install).
- Use `depends_on: { mysql: { condition: service_healthy } }` so `web` blocks on MySQL.

```bash
#!/usr/bin/env bash
# Story 0.9 acceptance test:
# `docker compose up` from a fresh clone reaches the login page in ≤ 60 s.
set -euo pipefail

cleanup() {
  docker compose down -v >/dev/null 2>&1 || true
}
trap cleanup EXIT

# 1. Bring up the stack detached.
docker compose up -d --build

# 2. Poll the login URL for up to 60 s.
deadline=$(( $(date +%s) + 60 ))
while [ "$(date +%s)" -lt "$deadline" ]; do
  if curl -fs "http://localhost:8000/accounts/login/" >/dev/null; then
    echo "OK: login page reachable"
    exit 0
  fi
  sleep 2
done

echo "FAIL: login page not reachable within 60 s"
docker compose logs --no-color web
exit 1
```

- [ ] **Step 2.2 — Create `Dockerfile`.**

```dockerfile
# Stage 1 — Build Tailwind CSS
FROM node:20-alpine AS css-builder
WORKDIR /build
COPY package.json package-lock.json tailwind.config.js ./
COPY static/src ./static/src
COPY templates ./templates
RUN npm ci && npm run build:css

# Stage 2 — Python runtime
FROM python:3.12-slim-bookworm AS runtime

ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1 \
    DJANGO_SETTINGS_MODULE=config.settings.dev

# System deps: mysqlclient + curl (healthcheck) + dockerize for wait-for
RUN apt-get update && apt-get install -y --no-install-recommends \
    build-essential \
    default-libmysqlclient-dev \
    pkg-config \
    curl \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app

# Python deps
COPY requirements.txt ./
RUN pip install --no-cache-dir -r requirements.txt

# App source
COPY . .

# Bring in the compiled CSS from stage 1
COPY --from=css-builder /build/static/dist ./static/dist

# Entrypoint
COPY docker-entrypoint.sh /usr/local/bin/docker-entrypoint.sh
RUN chmod +x /usr/local/bin/docker-entrypoint.sh

EXPOSE 8000
ENTRYPOINT ["docker-entrypoint.sh"]
CMD ["gunicorn", "config.wsgi:application", "--bind", "0.0.0.0:8000", "--workers", "3"]
```

- [ ] **Step 2.3 — Create `docker-entrypoint.sh`.**

```bash
#!/usr/bin/env bash
set -euo pipefail

# Wait for MySQL to be ready before running migrations.
echo "Waiting for MySQL at ${MYSQL_HOST:-mysql}:3306 …"
until python -c "import socket; socket.create_connection(('${MYSQL_HOST:-mysql}', 3306), timeout=2)" 2>/dev/null; do
  sleep 1
done
echo "MySQL is up."

# Apply migrations. Safe to run repeatedly.
python manage.py migrate --noinput

# Hand off to whatever was passed as CMD (gunicorn, qcluster, etc.).
exec "$@"
```

### Task 3: Write `compose.yaml` (GREEN — part 2)

**Files:** `compose.yaml`

- [ ] **Step 3.1 — Create `compose.yaml`.**

```yaml
services:
  mysql:
    image: mysql:8
    restart: unless-stopped
    environment:
      MYSQL_DATABASE: ${MYSQL_DATABASE:-merrymeal}
      MYSQL_USER: ${MYSQL_USER:-merrymeal}
      MYSQL_PASSWORD: ${MYSQL_PASSWORD:-merrymeal}
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD:-rootpass}
    volumes:
      - mysql_data:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-u", "$MYSQL_USER", "-p$$MYSQL_PASSWORD"]
      interval: 5s
      timeout: 5s
      retries: 12

  redis:
    image: redis:7-alpine
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 5

  web:
    build: .
    restart: unless-stopped
    env_file: .env
    environment:
      MYSQL_HOST: mysql
      REDIS_URL: redis://redis:6379/0
    ports:
      - "8000:8000"
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy
    command: ["gunicorn", "config.wsgi:application", "--bind", "0.0.0.0:8000", "--workers", "3"]

  worker:
    build: .
    restart: unless-stopped
    env_file: .env
    environment:
      MYSQL_HOST: mysql
      REDIS_URL: redis://redis:6379/0
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy
    command: ["python", "manage.py", "qcluster"]

volumes:
  mysql_data:
```

- [ ] **Step 3.3 — Confirm `config/settings/base.py` reads `MYSQL_HOST` from env (defaults to `localhost`).**

Open `config/settings/base.py`. The `DATABASES['default']['HOST']` should be `os.environ.get('MYSQL_HOST', 'localhost')`. If it is hard-coded to `localhost`, change it:

```python
DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'NAME': os.environ['MYSQL_DATABASE'],
        'USER': os.environ['MYSQL_USER'],
        'PASSWORD': os.environ['MYSQL_PASSWORD'],
        'HOST': os.environ.get('MYSQL_HOST', 'localhost'),
        'PORT': os.environ.get('MYSQL_PORT', '3306'),
        'OPTIONS': {'charset': 'utf8mb4'},
    }
}
```

## Pitfalls reviewer will check

- **Secrets in `compose.yaml`** — must use `${VAR:-default}` style, never hard-coded.
- **`mysql_data` volume is named, not anonymous** — anonymous volumes do not persist across `down`.
- **`worker` runs `qcluster`, not `runserver`** — typo here means jobs silently never run.
- **Healthcheck escape** — `$$` inside YAML is `$` at runtime. If you wrote `$MYSQL_PASSWORD` in the healthcheck, compose will substitute it before MySQL sees it (broken).
- **`db.sqlite3` in `.dockerignore`** — otherwise the SQLite file from local dev sneaks into the image (~100 KB but wrong DB engine).

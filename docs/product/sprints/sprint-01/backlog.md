# Sprint 01 — Backlog

> The "backlog" of a sprint is everything **considered** for the sprint but
> not committed. If a story slips, it lands here. If we finish early, we
> pull from here.

## Spillover protocol

If a story committed in `sprint-plan.md` is **not** `done` by Friday week 2 demo:

1. Update its `STATUS:` in the epic file to `STATUS: planned (sprint-02)`.
2. Add a line to `docs/product/sprints/sprint-02/sprint-plan.md` under "Carried over from Sprint 01".
3. Move the story file in `stories/` to `../sprint-02/stories/`.
4. Note the carry-over in the sprint retro section (below).

## User story

> **As** Sarah at Margaret's gate when no one answers
> **I want** a clearly secondary "Couldn't deliver" link, leading to a
> bottom sheet where I pick a reason and add a note
> **so that** Margaret's daughter is alerted within seconds and the admin
> can follow up tomorrow — without me accidentally tapping "fail" when I
> meant "deliver".

## Acceptance criteria

- [ ] On the current stop card, a small text link "Couldn't deliver" sits
      **under** the primary CTA (Story 4.8), styled as a low-contrast
      `text-warm-500 underline` — visually secondary by design (failure is
      not the easy default).
- [ ] Tap → an Alpine.js bottom sheet slides up containing:
  - 4 reason chips: `Not home`, `No answer`, `Refused`, `Address wrong`
    (one selectable at a time, radio behavior).
  - Notes textarea (optional, max 500 chars).
  - "Submit" button (≥ 44 px) at the bottom of the sheet.
  - "Cancel" link top-right.
- [ ] Submit → `POST /volunteer/delivery/<id>/mark-failed/` (HTMX).
- [ ] Server-side:
  - Adds a new column `Delivery.failure_reason` (text, nullable) via a
    migration.
  - Sets `Delivery.status = 'failed'`.
  - Writes the chosen chip slug + notes into `failure_reason` as
    `f"{reason}: {notes}"` (or just `reason` if notes blank).
  - Writes an `auditlog` row.
  - Triggers Story 4.13's caregiver-alert side effect (via the same
    post-save signal — this story just needs to ensure the transition
    fires).
- [ ] Auto-advance: the screen swaps the failed stop card for the next
      pending one (same HTMX pattern as Story 4.9).
- [ ] All chips ≥ 44 px tall; bottom sheet renders cleanly at 375 × 667
      px (no horizontal scrollbar, sheet does not exceed viewport).

## Tech / library notes

- **Reason choices** — define as a `TextChoices` class on the form (not
  the model — these are UI labels, not domain enums):
  ```python
  REASON_CHOICES = [
      ("not_home", "Not home"),
      ("no_answer", "No answer"),
      ("refused", "Refused"),
      ("address_wrong", "Address wrong"),
  ]
  ```
- **`failure_reason` is a single text column**, not a foreign key. We do
  not need a `FailureReason` table for v1; the four slugs are stable.
- **Bottom sheet** — pure Alpine `x-data="{open: false}"` with
  `x-transition` on a `fixed inset-x-0 bottom-0` div. No HTMX needed to
  open / close the sheet, only to submit.
- **The caregiver-alert signal** lives in Story 4.13. This story must
  **not** instantiate or send any alert directly — it only flips status
  and trusts the signal.

### Task 3: Service GREEN

**Files:** `apps/delivery/services/mark_failed.py`

- [ ] **Step 3.1 — Write the service.**

```python
"""Service: mark a Delivery as failed with reason + notes."""
from django.db import transaction
from apps.delivery.models import Delivery


VALID_REASONS = {"not_home", "no_answer", "refused", "address_wrong"}


@transaction.atomic
def mark_failed(delivery: Delivery, *, reason: str, notes: str) -> Delivery:
    if reason not in VALID_REASONS:
        raise ValueError(f"invalid reason: {reason}")

    locked = Delivery.objects.select_for_update().get(pk=delivery.pk)
    if locked.status == "failed":
        return locked  # idempotent

    locked.status = "failed"
    locked.failure_reason = f"{reason}: {notes}" if notes else reason
    locked.save(update_fields=["status", "failure_reason", "updated_at"])
    return locked
```

```python
from apps.delivery.services.mark_failed import mark_failed
from apps.delivery.forms.mark_failed import MarkFailedForm


@login_required
@role_required("volunteer")
@require_POST
def mark_failed_view(request, pk: int):
    delivery = get_object_or_404(Delivery, pk=pk, volunteer=request.user)
    form = MarkFailedForm(request.POST)
    if not form.is_valid():
        return HttpResponse(status=400)

    mark_failed(
        delivery,
        reason=form.cleaned_data["reason"],
        notes=form.cleaned_data["notes"],
    )

    context = get_today_route(request.user)
    return render(request, "delivery/volunteer/_route_fragment.html", context)
```

```typescript
import { test, expect } from "@playwright/test";

test.use({ viewport: { width: 375, height: 667 } });

test("mark-failed flow at 375x667", async ({ page }) => {
  await page.goto("/volunteer/today/?fixture=sarah-three-stops");

  // Couldn't deliver link is visually secondary
  const link = page.getByTestId("couldnt-deliver-link");
  await expect(link).toBeVisible();
  // Anti-pattern check: link must not be a giant red button.
  // Its computed font-size is the body 18 px or smaller, not heading size.
  const fontSize = await link.evaluate(
    (el) => parseFloat(getComputedStyle(el).fontSize)
  );
  expect(fontSize).toBeLessThanOrEqual(18);

  await link.click();

  // Bottom sheet visible, does not push horizontal scrollbar
  const sheet = page.getByTestId("fail-sheet");
  await expect(sheet).toBeVisible();
  const overflow = await page.evaluate(
    () => document.documentElement.scrollWidth > 375
  );
  expect(overflow).toBe(false);

  // Each chip is at least 44 px tall
  const chips = page.getByTestId("reason-chip");
  const count = await chips.count();
  for (let i = 0; i < count; i++) {
    const box = await chips.nth(i).boundingBox();
    expect(box!.height).toBeGreaterThanOrEqual(44);
  }

  // Submit
  await page.locator("input[name=reason][value=no_answer]").check({ force: true });
  await page.locator("textarea[name=notes]").fill("rang twice");
  await page.getByTestId("fail-submit").click();

  // Failed stop is replaced; next stop becomes current
  await expect(page.getByText(/Mark .+ delivered/)).toBeVisible();
});
```

## Pitfalls reviewer will check

- **Visual hierarchy** — if the "Couldn't deliver" link gets a button
  style by accident, it competes with the primary CTA. Reviewer eyes the
  rendered screen, not just the Playwright pass.
- **Required `reason` field** — server-side `is_valid()` must reject an
  empty form. A radio group with no checked option submits no `reason`
  key at all.
- **`failure_reason` text format** — keep machine-readable: `slug` or
  `slug: notes`. Epic 06 will graph counts by slug. Free-text-only would
  break the report.
- **No alert in this story** — do **not** import the alerts service from
  Story 4.13. The signal owns the side effect; this view only flips
  status.
- **Bottom-sheet inside fixed-positioned container** — `fixed inset-x-0
  bottom-0` ignores the parent scroll; that's correct here. But the
  overlay click-to-close must `@click.stop` on the sheet itself,
  otherwise a tap on the chip closes the sheet.
- **`max-h-[80vh]` + `overflow-y-auto`** — without these, a 5-line note
  textarea on a small screen can push the submit button off-screen.
- **HTMX swap target** — `#route-fragment` must exist in
  `_route_fragment.html`. Double-check the ID matches Story 4.9's swap.
- **Migration ordering** — if Sprint 07 added a different migration
  number, `makemigrations` will renumber; commit the actual filename
  Django picked, not a hard-coded one.

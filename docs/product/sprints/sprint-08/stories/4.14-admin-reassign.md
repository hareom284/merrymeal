## User story

> **As** an admin on Wednesday morning when a volunteer phones in sick
> **I want** a simple table of today's deliveries with a "Reassign" button
> per row
> **so that** I can drag a stop onto another volunteer's route in under
> 30 seconds — without filing a ticket.

## Acceptance criteria

- [ ] New route `GET /admin/today/` (custom view, **not** Django admin)
      restricted to `role='admin'`. Returns a table of today's deliveries
      with columns: member, current volunteer, status, route #, action.
- [ ] Each row has a "Reassign" button → opens a modal listing volunteers
      who are **available today** (Story 4.1 `Availability` for today's
      day_of_week + day_phrase) AND not already over the 12-stop cap from
      Story 4.7.
- [ ] On modal submit:
  - `Delivery.volunteer_id` is updated to the new volunteer.
  - `Delivery.route_id` is updated to that volunteer's route for today
    (creating a new `Route` row if they have none yet).
  - The previous route's stop count drops by 1; the new route's grows.
  - An `auditlog` entry is written with the old → new volunteer IDs.
- [ ] The previous volunteer's `/volunteer/today/` reflects the change on
      their next 60-second poll (Story 4.12's pattern — but the volunteer
      today screen also auto-polls). For v1 we accept eventual
      consistency: no push.
- [ ] Desktop-primary layout (table renders well at ≥ 1024 px), but
      remains readable on mobile (table falls back to stacked cards via
      Tailwind `md:` breakpoints).
- [ ] Admin who tries to reassign to a non-volunteer or to themselves
      gets a 400 with a clear error.

## Tech / library notes

- **Two viewports.** Playwright spec uses 1024 × 768; manual mobile
  read at 375 px confirms the stacked card fallback works (single
  assertion: no horizontal scrollbar at 375 px).
- **"Available today" query** — re-use Story 4.1 `Availability` filter:
  ```python
  Availability.objects.filter(
      volunteer__role="volunteer",
      day_of_week=today.weekday(),
      day_phrase="morning",  # v1
  )
  ```
- **Route creation when target volunteer has none.** Story 4.3 model;
  create `Route(volunteer=new, route_date=today, status="planned")`.
- **12-stop cap** — duplicated from Story 4.7. Centralise into a
  constant `MAX_STOPS_PER_ROUTE = 12` in
  `apps/delivery/services/dispatch.py` and import from both places.
- **Modal pattern** — HTMX `hx-get` loads the modal HTML; submit
  HTMX `hx-post` returns the updated row partial; HTMX swaps in place
  via `hx-target="#delivery-row-<id>"` and closes the modal via
  `HX-Trigger` response header (`closeModal`).
- **No push to volunteer phone**. Story 4.12's polling makes the
  reassignment visible within 60 s. Document in Pitfalls so reviewer
  does not ask.

### Task 2: Service GREEN

**Files:** `apps/delivery/services/reassign.py`

- [ ] **Step 2.1 — Write the service.**

```python
"""Service: reassign one Delivery to a different volunteer."""
from django.db import transaction
from django.utils import timezone
from apps.delivery.models import Delivery, Route


MAX_STOPS_PER_ROUTE = 12  # mirrors Story 4.7


@transaction.atomic
def reassign_delivery(delivery: Delivery, *, new_volunteer) -> Delivery:
    if getattr(new_volunteer, "role", None) != "volunteer":
        raise ValueError("target user is not a volunteer")

    today = delivery.scheduled_date or timezone.localdate()
    route, _ = Route.objects.get_or_create(
        volunteer=new_volunteer,
        route_date=today,
        defaults={"status": "planned"},
    )

    stops_on_route = Delivery.objects.filter(route=route).count()
    if stops_on_route >= MAX_STOPS_PER_ROUTE:
        raise ValueError(
            f"route already at cap ({MAX_STOPS_PER_ROUTE} stops)"
        )

    locked = Delivery.objects.select_for_update().get(pk=delivery.pk)
    locked.volunteer = new_volunteer
    locked.route = route
    locked.save(update_fields=["volunteer", "route", "updated_at"])
    return locked
```

```python
from django.contrib.auth.decorators import login_required
from django.shortcuts import render
from django.utils import timezone
from apps.core.decorators import role_required
from apps.delivery.models import Delivery


@login_required
@role_required("admin")
def admin_today_view(request):
    today = timezone.localdate()
    deliveries = (
        Delivery.objects.filter(scheduled_date=today)
        .select_related("member", "volunteer", "route")
        .order_by("route_id", "id")
    )
    return render(
        request,
        "delivery/admin/today.html",
        {"deliveries": deliveries, "today": today},
    )
```

- [ ] **Step 3.3 — `apps/delivery/views/admin_reassign.py`.**

```python
from django.contrib.auth.decorators import login_required
from django.http import HttpResponse
from django.shortcuts import get_object_or_404, render
from django.views.decorators.http import require_http_methods
from apps.core.decorators import role_required
from apps.delivery.forms.reassign import ReassignForm
from apps.delivery.models import Delivery
from apps.delivery.services.reassign import reassign_delivery


@login_required
@role_required("admin")
@require_http_methods(["GET", "POST"])
def reassign_view(request, pk: int):
    delivery = get_object_or_404(Delivery, pk=pk)

    if request.method == "GET":
        form = ReassignForm()
        return render(
            request,
            "delivery/admin/_reassign_modal.html",
            {"delivery": delivery, "form": form},
        )

    form = ReassignForm(request.POST)
    if not form.is_valid():
        return HttpResponse(status=400)

    try:
        reassign_delivery(delivery, new_volunteer=form.cleaned_data["volunteer"])
    except ValueError as e:
        return HttpResponse(str(e), status=400)

    resp = render(request, "delivery/admin/_row.html", {"delivery": delivery})
    resp["HX-Trigger"] = "closeModal"
    return resp
```

## Pitfalls reviewer will check

- **Reassign-to-self silent no-op** — service writes the same volunteer
  ID and the row swap shows nothing changed. Either reject or accept
  with a clear UI hint; do not silently succeed.
- **Cap check before lock** — checking the count outside the
  `select_for_update` allows two simultaneous reassigns to overrun the
  cap. Move the count inside the atomic block (already structured
  that way in the service above).
- **`get_or_create(volunteer, route_date)` racing** — `get_or_create`
  is safe under a unique constraint `(volunteer, route_date)`. Confirm
  the unique index exists; if not, add a migration.
- **403 vs 404 on non-admin access** — must be 403 so the test asserts
  cleanly. The decorator returns 403 by convention.
- **HTMX modal not closing** — relies on the `HX-Trigger: closeModal`
  response header firing the body-level event listener. If swap target
  is the row (`outerHTML`) the modal stays open until the trigger
  fires. Check both response code 200 + header in Playwright.
- **`ModelChoiceField` rendering full email list** — by default Django
  renders the User's `__str__`. Confirm `User.__str__` returns a name,
  not the email, or the dropdown is unreadable.
- **Pushing change to volunteer phone** — not in scope; rely on Story
  4.12 polling. Don't add websockets here.
- **Routes never deleted** — when the old route's last stop moves away,
  the empty `Route` row remains. Acceptable for v1 (Epic 06 cleanup).
- **Cross-day reassign** — service uses `delivery.scheduled_date`, not
  `today`. Confirm test fixtures set `scheduled_date=today`, otherwise
  the new route lands on the wrong day.

## User story

> **As** Margaret, a 78-year-old member, sitting with my warm meal
> **I want** to rate the meal with two taps — stars + one chip — without
> typing anything
> **so that** the kitchen knows what I think without me opening my email.

## Acceptance criteria

- [ ] On the member home today card (Story 3.4), if today's
      `Delivery.status == 'delivered'` and no `DeliveryFeedback` exists
      yet, a **feedback prompt** card replaces the static "Delivered"
      badge.
- [ ] The prompt shows 5 star buttons (☆☆☆☆☆ → ★★★★★ as taps land) and
      5 chip tags: `Great`, `Bland`, `Too cold`, `Too small`, `Loved it`.
- [ ] One or more chips can be selected (multi-select). Stars are single
      value 1–5.
- [ ] Submit button enabled only after a star is tapped (chips optional).
- [ ] Submit → `POST /member/feedback/<delivery_id>/` → creates a
      `DeliveryFeedback` row with `rating` + `note = json.dumps({"tags": [...]})`.
- [ ] HTMX returns a thank-you card that replaces the prompt in place.
- [ ] One feedback per delivery (model already enforces unique); a
      second submit returns 409 with a "Thanks — already recorded"
      partial.
- [ ] Restricted to `role='member'`, and the member must own the
      delivery (`Delivery.member_id == user.id`).
- [ ] Mobile-first at 375 px, 18 px base, every interactive element
      ≥ 44 × 44 px.

## Tech / library notes

- **Tags as JSON in `note`** — `DeliveryFeedback.note` is a `LONGTEXT`
  per the schema. We store `json.dumps({"tags": ["loved_it", "great"]})`.
  A future migration could split tags into their own column; for v1
  this is fine because the dashboards (Epic 06) parse the JSON once
  per aggregation.
- **Stars implementation** — five `<input type="radio" name="rating">`
  inputs (1–5), styled with `peer-checked:` Tailwind tricks. No JS for
  basic functionality; Alpine optional for instant-fill UX.
- **Chips** — `<input type="checkbox">` per tag, styled chip via
  `peer-checked` like Story 4.10.
- **Idempotency** — model has `UNIQUE KEY (delivery_id)`. Catch
  `IntegrityError` in the service and return the existing row.

### Task 2: Service GREEN

**Files:** `apps/delivery/services/feedback.py`

- [ ] **Step 2.1 — Write the service.**

```python
"""Service: record a member's feedback for one delivery."""
import json
from django.db import IntegrityError, transaction
from apps.delivery.models import Delivery, DeliveryFeedback


@transaction.atomic
def record_feedback(delivery: Delivery, *, rating: int, tags: list[str]) -> DeliveryFeedback:
    if not 1 <= rating <= 5:
        raise ValueError("rating must be 1..5")
    try:
        return DeliveryFeedback.objects.create(
            delivery=delivery,
            rating=rating,
            note=json.dumps({"tags": tags}),
        )
    except IntegrityError:
        return DeliveryFeedback.objects.get(delivery=delivery)
```

from django import forms


TAG_CHOICES = [
    ("great", "Great"),
    ("bland", "Bland"),
    ("too_cold", "Too cold"),
    ("too_small", "Too small"),
    ("loved_it", "Loved it"),
]


class FeedbackForm(forms.Form):
    rating = forms.IntegerField(min_value=1, max_value=5, required=True)

- [ ] **Step 3.2 — Create `apps/delivery/views/feedback.py`.**

```python
from django.contrib.auth.decorators import login_required
from django.http import HttpResponse
from django.shortcuts import get_object_or_404, render
from django.views.decorators.http import require_POST
from apps.core.decorators import role_required
from apps.delivery.forms.feedback import FeedbackForm, TAG_CHOICES
from apps.delivery.models import Delivery, DeliveryFeedback
from apps.delivery.services.feedback import record_feedback


@login_required
@role_required("member")
@require_POST
def feedback_view(request, pk: int):
    delivery = get_object_or_404(Delivery, pk=pk, member=request.user)
    if delivery.status != "delivered":
        return HttpResponse(status=400)
    if DeliveryFeedback.objects.filter(delivery=delivery).exists():
        return render(request, "delivery/member/_feedback_thanks.html",
                      {"already": True}, status=200)

    form = FeedbackForm(request.POST)
    if not form.is_valid():
        return HttpResponse(status=400)

    record_feedback(
        delivery,
        rating=form.cleaned_data["rating"],
        tags=form.cleaned_data["tags"],
    )
    return render(request, "delivery/member/_feedback_thanks.html",
                  {"already": False})
```

import { test, expect } from "@playwright/test";

test.use({ viewport: { width: 375, height: 667 } });

test("2-tap feedback at 375x667", async ({ page }) => {
  await page.goto("/?fixture=margaret-just-delivered");
  const card = page.getByTestId("feedback-card");
  await expect(card).toBeVisible();

  // No horizontal scrollbar
  const overflow = await page.evaluate(
    () => document.documentElement.scrollWidth > 375
  );
  expect(overflow).toBe(false);

  // Stars and chips each ≥ 44 px
  for (const tid of ["star", "tag-chip"]) {
    const els = page.getByTestId(tid);
    const count = await els.count();
    for (let i = 0; i < count; i++) {
      const box = await els.nth(i).boundingBox();
      expect(box!.height).toBeGreaterThanOrEqual(44);
    }
  }

  // Two taps: 4th star + Loved it chip
  await page.locator("input[name=rating][value='4']").check({ force: true });
  await page.locator("input[name=tags][value=loved_it]").check({ force: true });
  await page.getByTestId("feedback-submit").click();

  await expect(page.getByTestId("feedback-thanks")).toBeVisible();
});

## Pitfalls reviewer will check

- **`required=True` on rating** — without it, an empty form passes
  validation and creates a row with `rating=None`.
- **Single radio group, not five separate radios** — same `name="rating"`,
  five different `value`s. Easy to write five different names by accident.
- **Chips submit as repeated `tags`** — make sure
  `MultipleChoiceField` + `CheckboxSelectMultiple`, not a single
  `ChoiceField`.
- **Star icon size — visual ≠ hit area.** The visible glyph can be
  36 px while the label (the actual tap target) is 44 × 44 px. The
  Playwright test asserts the label height, not the glyph.
- **IDOR via `pk`** — `get_object_or_404(... member=request.user)` is
  load-bearing. Without it any logged-in member can rate any delivery.
- **Race on uniqueness** — two near-simultaneous submits would raise
  `IntegrityError`; the service catches it and returns the existing row.
- **Forgetting to gate by `status='delivered'`** — a member could rate
  a pending delivery otherwise. 400 is the right code.
- **Feedback card swapping itself** — `hx-target="#feedback-card-<id>"`
  must match the wrapper ID exactly, or the thanks card appears
  duplicated.

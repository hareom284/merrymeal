## User story

> **As** Sarah, a volunteer on a bike
> **I want** one screen for today's route — pickup at the top, stops in order,
> a thumb-zone "Mark delivered" button — that I can drive with one hand
> **so that** I never have to scroll, zoom, or two-hand the phone in traffic.

## Acceptance criteria

- [ ] New route `GET /volunteer/today/` resolves only for users with
      `role='volunteer'`; everyone else gets a 403 from `@role_required`.
- [ ] A volunteer sees **only their own** route (object-level scoping in the
      service: `Delivery.objects.filter(volunteer=user, scheduled_date=today)`).
- [ ] **Top card** displays the kitchen name, kitchen address, and the
      pickup time (route `start_time - 5 min`; if `Route` has no `start_time`,
      fall back to `08:00` Melbourne time).
- [ ] **Stop list** is rendered in the same order as the greedy-nearest
      packing produced by Story 4.7 (read `Delivery.id` order within the
      route — do not re-sort client-side).
- [ ] The **current stop** (first `pending` or `out_for_delivery`) is
      visually highlighted (warm-100 background + brand-green left border).
- [ ] Each stop card shows:
  - Member first name + last initial (e.g. "Margaret S.")
  - Address (street line only, suburb hidden until tap)
  - Special instructions (free text from `MemberPreferences`, if any)
  - **Allergen warning badge** — uses
        `apps.planning.services.allergen.meal_allergens_for_member(meal, member)`
        from Story 3.5. Renders as `⚠ Contains <allergen>` chip in brand-orange
        if the list is non-empty, hidden otherwise.
- [ ] **Bottom-anchored CTA** — `Mark <first_name> delivered` button is
      `position: sticky; bottom: 0;` inside the scroll container, ≥ 44 px
      tall, full-width, brand-green. Sits in the bottom 1/3 of a
      375 × 667 viewport (i.e. its top edge is ≥ 444 px from the page top
      when the page is scrolled to the current stop).
- [ ] Tapping a stop card **expands** it inline (Alpine.js `x-show`) to
      reveal full address + member phone number; tapping again collapses.
      No full page reload.
- [ ] Page renders cleanly on 375 × 667 viewport: no horizontal scrollbar,
      18 px base body font, all interactive elements ≥ 44 × 44 px.
- [ ] An empty-route case (volunteer logged in but has no `Delivery` for
      today) shows an "All done — nothing scheduled" empty state, not a 500.

## Tech / library notes

- **Service shape** — `get_today_route(user)` returns:
  ```python
  {
      "kitchen": {"name": str, "address": str, "pickup_time": time},
      "route": Route | None,
      "stops": list[
          {
              "delivery": Delivery,
              "member_display": "Margaret S.",
              "address_line": str,
              "phone": str,
              "special_instructions": str,
              "allergens": list[Allergy],  # may be []
              "is_current": bool,
          }
      ],
  }
  ```
- **Greedy-nearest order is already encoded** by Story 4.7. Do not re-sort
  in this service — read `Delivery.objects.filter(route=route).order_by('id')`.
- **Tailwind mobile-first** — every utility is base (mobile) first; no
  `md:` / `lg:` prefixes needed for v1.
- **Sticky bottom CTA** — use `<div class="sticky bottom-0 ...">` inside
  the scroll container. **Do not** use `fixed` — it breaks when iOS Safari
  reveals the URL bar mid-scroll.
- **Touch targets** — `min-h-[44px]` on every interactive element. Add
  `py-3` to chip-style elements so the hit area is bigger than the glyph.
- **Allergens** — call the existing Story 3.5 service. Do not inline the
  allergen lookup in this story.
- **Phone field** — `User.phone` (Epic 01). If null, hide the call link;
  do not render a broken `tel:` href.

```python
"""Service: shape today's route for the volunteer screen.

Pure read; no side effects.
"""
from datetime import time, timedelta
from django.utils import timezone
from apps.delivery.models import Delivery, Route
from apps.planning.services.allergen import meal_allergens_for_member


DEFAULT_PICKUP = time(7, 55)  # 5 min before 08:00 if Route has no start_time


def _member_display(member) -> str:
    last_initial = (member.last_name[:1] + ".") if member.last_name else ""
    return f"{member.first_name} {last_initial}".strip()


def get_today_route(user) -> dict:
    today = timezone.localdate()
    deliveries = (
        Delivery.objects
        .filter(volunteer=user, scheduled_date=today)
        .select_related("member", "member_address", "meal_plan", "meal_plan__kitchen", "route")
        .order_by("id")
    )

    route = deliveries[0].route if deliveries else None
    kitchen = route.kitchen if route and hasattr(route, "kitchen") else (
        deliveries[0].meal_plan.kitchen if deliveries else None
    )

    pickup_time = DEFAULT_PICKUP
    if route and getattr(route, "start_time", None):
        dt = timezone.datetime.combine(timezone.localdate(), route.start_time)
        pickup_time = (dt - timedelta(minutes=5)).time()

    stops = []
    current_assigned = False
    for d in deliveries:
        is_current = (not current_assigned) and d.status in {"pending", "out_for_delivery"}
        if is_current:
            current_assigned = True
        stops.append({
            "delivery": d,
            "member_display": _member_display(d.member),
            "address_line": d.member_address.street if d.member_address else "",
            "phone": getattr(d.member, "phone", "") or "",
            "special_instructions": getattr(d.member, "special_instructions", "") or "",
            "allergens": meal_allergens_for_member(d.meal_plan.meal, d.member) if d.meal_plan_id else [],
            "is_current": is_current,
        })

    return {
        "kitchen": {
            "name": kitchen.name if kitchen else "",
            "address": getattr(kitchen, "address", "") if kitchen else "",
            "pickup_time": pickup_time,
        },
        "route": route,
        "stops": stops,
    }
```

```typescript
import { test, expect } from "@playwright/test";

test.describe("Volunteer today screen @ 375x667", () => {
  test.use({ viewport: { width: 375, height: 667 } });

  test("no horizontal scrollbar", async ({ page }) => {
    await page.goto("/volunteer/today/");
    const overflow = await page.evaluate(
      () => document.documentElement.scrollWidth > 375
    );
    expect(overflow).toBe(false);
  });

  test("Mark-delivered CTA sits in the bottom third of the viewport", async ({ page }) => {
    await page.goto("/volunteer/today/");
    const cta = page.getByTestId("mark-delivered-cta");
    await expect(cta).toBeVisible();
    const box = await cta.boundingBox();
    expect(box).not.toBeNull();
    expect(box!.y).toBeGreaterThanOrEqual(444); // bottom 1/3 of 667
    expect(box!.height).toBeGreaterThanOrEqual(44);
  });

  test("every stop card has at least one 44px tap target", async ({ page }) => {
    await page.goto("/volunteer/today/");
    const cards = page.getByTestId("stop-card");
    const count = await cards.count();
    for (let i = 0; i < count; i++) {
      const btn = cards.nth(i).locator("button").first();
      const box = await btn.boundingBox();
      expect(box!.height).toBeGreaterThanOrEqual(44);
    }
  });

  test("allergen badge renders when meal contains member allergen", async ({ page }) => {
    await page.goto("/volunteer/today/?fixture=margaret-shellfish");
    await expect(page.getByTestId("allergen-badge").first()).toBeVisible();
  });
});
```

## Pitfalls reviewer will check

- **`fixed` vs `sticky` bottom CTA** — `fixed` jumps when iOS Safari's URL
  bar shows/hides. Use `sticky bottom-0`.
- **Server-side sort vs greedy-nearest** — do not call `.order_by('distance')`
  here. Story 4.7 already encoded the order via `Delivery.id` insertion.
- **Allergen badge bypass** — do not inline the allergen logic. Re-use
  `meal_allergens_for_member` from Story 3.5 verbatim.
- **403 vs 404 for cross-volunteer access** — the page itself is volunteer-
  scoped; there is no per-route URL parameter to leak (no IDOR vector).
- **`tel:` href with null phone** — must be omitted entirely, not rendered
  as `<a href="tel:">`.
- **Empty route 500** — verify by deleting all deliveries for today and
  reloading. Must show the empty-state card.
- **18 px base** — `text-[18px]` on the body wrapper; do not let Tailwind's
  default 16 px shrink stop card copy. Check `dictsort:"is_current"|last`
  actually returns the current stop (Django dictsort is stable on bool).

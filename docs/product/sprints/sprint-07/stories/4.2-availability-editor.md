## User story

> **As a** volunteer
> **I want** a simple grid (days × morning/afternoon/evening) on my phone
> **so that** I can update my availability in 30 seconds before bed.

## Acceptance criteria

- [ ] Route `/volunteer/availability/` (login-required, role-restricted to `volunteer`).
- [ ] Page renders a 7-row (Mon–Sun) × 3-column (Morning / Afternoon / Evening) grid.
- [ ] Tap a cell → HTMX `POST /volunteer/availability/toggle/` → server
      adds the row if absent or deletes it if present → returns the
      single replaced `<button>` partial. No full reload.
- [ ] Service `apps/volunteers/services/availability.py::toggle_slot(volunteer, day, phrase)`
      is the **only** place that writes. View is a thin wrapper.
- [ ] Mobile-first at 375 px width. Each cell ≥ 44 × 44 px touch target.
      No horizontal scroll at 375 px.
- [ ] Selected cells are visually distinct (filled background, accessible
      contrast). Aria-pressed reflects state.
- [ ] Playwright test at 375 × 667 px: log in as a volunteer, tap Mon
      morning, reload, the cell is still selected; tap again, reload,
      the cell is unselected.

## Tech / library notes

- **HTMX endpoint contract.** The toggle endpoint receives `day` + `phrase`
  as form data and returns the **single cell partial** with updated
  `aria-pressed` + classes. The client swaps via `hx-swap="outerHTML"`.
  Each cell has `id="slot-{day}-{phrase}"`.
- **Service contract.**
  ```python
  def toggle_slot(volunteer: User, day: str, phrase: str) -> bool:
      """Return True if slot now exists, False if it was removed."""
  ```
  Uses `get_or_create` then deletes on collision. Validates inputs against
  `Availability.DAY_OF_WEEK_CHOICES` + `DAY_PHRASE_CHOICES`. Calls
  `instance.full_clean()` so the role check from Story 4.1 fires.
- **Role gate.** Use `apps.core.decorators.role_required("volunteer")`
  (already exists per Epic 00). 403 on other roles, 302→login if anon.
- **Tailwind classes.** Stick to the warm palette from `README.md`. Selected
  cell uses `bg-brand-green text-white`, unselected uses
  `bg-warm-100 text-warm-800 border border-warm-200`. Cells are
  `min-h-[44px] min-w-[44px]` for the 44 px target.
- **No JS framework on this page** — HTMX + the existing Alpine.js base is
  enough. The grid is server-rendered; client mutations are 1-cell HTMX swaps.

```python
# apps/volunteers/services/availability.py
from django.contrib.auth import get_user_model
from django.db import transaction

from apps.volunteers.models import Availability

User = get_user_model()

_VALID_DAYS = {d for d, _ in Availability.DAY_OF_WEEK_CHOICES}
_VALID_PHRASES = {p for p, _ in Availability.DAY_PHRASE_CHOICES}


def toggle_slot(volunteer, day: str, phrase: str) -> bool:
    """Add the slot if missing, remove it if present.

    Returns `True` if the slot is now active, `False` if it was just removed.

    Raises:
        ValueError: if `day` or `phrase` is not a valid choice.
        django.core.exceptions.ValidationError: if `volunteer.role != "volunteer"`
            (re-raised from `Availability.clean()`).
    """
    if day not in _VALID_DAYS:
        raise ValueError(f"Invalid day_of_week: {day!r}")
    if phrase not in _VALID_PHRASES:
        raise ValueError(f"Invalid day_phrase: {phrase!r}")

    with transaction.atomic():
        existing = Availability.objects.filter(
            volunteer=volunteer, day_of_week=day, day_phrase=phrase
        ).first()
        if existing:
            existing.delete()
            return False

        slot = Availability(volunteer=volunteer, day_of_week=day, day_phrase=phrase)
        slot.full_clean()  # fires role check from Story 4.1
        slot.save()
        return True
```

- [ ] **Step 4.1 — Create `apps/volunteers/views/availability.py`.**

```python
from django.contrib.auth.decorators import login_required
from django.http import HttpResponseBadRequest
from django.shortcuts import render
from django.views.decorators.http import require_GET, require_POST

from apps.core.decorators import role_required
from apps.volunteers.models import Availability
from apps.volunteers.services import toggle_slot


def _cell_context(volunteer, day: str, phrase: str, active: bool) -> dict:
    return {"day": day, "phrase": phrase, "active": active}


@login_required
@role_required("volunteer")
@require_GET
def availability_view(request):
    active = {
        (a.day_of_week, a.day_phrase)
        for a in Availability.objects.filter(volunteer=request.user)
    }
    grid = [
        {
            "day": day,
            "day_label": label,
            "cells": [
                _cell_context(request.user, day, phrase, (day, phrase) in active)
                for phrase, _ in Availability.DAY_PHRASE_CHOICES
            ],
        }
        for day, label in Availability.DAY_OF_WEEK_CHOICES
    ]
    return render(request, "volunteers/availability.html", {"grid": grid})


@login_required
@role_required("volunteer")
@require_POST
def toggle_view(request):
    day = request.POST.get("day", "")
    phrase = request.POST.get("phrase", "")
    try:
        active = toggle_slot(request.user, day, phrase)
    except ValueError as exc:
        return HttpResponseBadRequest(str(exc))
    return render(
        request,
        "volunteers/_slot_cell.html",
        _cell_context(request.user, day, phrase, active),
    )
```

```python
import pytest
from playwright.sync_api import expect


@pytest.mark.e2e
def test_availability_grid_mobile_toggle(page, live_server, volunteer_user):
    """At 375 × 667 px, tap a cell, reload, it stays selected; tap again, gone."""
    page.set_viewport_size({"width": 375, "height": 667})

    # Log in via the existing flow (helper assumed from Story 0.11 / sprint-01).
    page.goto(f"{live_server.url}/accounts/login/")
    page.fill('input[name="email"]', volunteer_user.email)
    page.fill('input[name="password"]', "pw12345!")
    page.click('button[type="submit"]')

    page.goto(f"{live_server.url}/volunteer/availability/")

    cell = page.locator("#slot-mon-morning")
    expect(cell).to_have_attribute("aria-pressed", "false")

    # Confirm 44 px touch target.
    box = cell.bounding_box()
    assert box["height"] >= 44, f"cell height {box['height']} < 44 px"

    cell.click()
    expect(cell).to_have_attribute("aria-pressed", "true")

    # Reload — server-rendered state should match.
    page.reload()
    expect(page.locator("#slot-mon-morning")).to_have_attribute("aria-pressed", "true")

    # Untoggle.
    page.locator("#slot-mon-morning").click()
    expect(page.locator("#slot-mon-morning")).to_have_attribute("aria-pressed", "false")

    # No horizontal scroll at 375 px.
    overflow = page.evaluate("document.documentElement.scrollWidth > window.innerWidth")
    assert overflow is False
```

## Pitfalls reviewer will check

- **CSRF on HTMX POST** — the partial includes `hx-headers` with
  `X-CSRFToken`. Without it, Django 403s the POST. Easy miss.
- **Returning the wrong partial** — the toggle view must render
  `_slot_cell.html`, not the full page. The HTMX swap is `outerHTML` on
  the single button.
- **Day order** — the grid follows the schema enum order (`mon..sun`,
  Mon first). Don't reorder to Sun-first US style.
- **Phrase is morning/afternoon/evening, not "AM/PM/EVE"** — the labels
  in the template are display-only; the values posted to the server must
  match the enum strings.
- **44 px target compliance** — Tailwind's `min-h-[44px]` only enforces a
  minimum; a parent `gap` is fine, but don't add `h-10` (40 px) which
  would override.
- **Role gate above login gate** — `@login_required` must be the
  outermost decorator so anonymous users hit a 302→login, not a 403.
  Tests cover this.
- **`hx-vals` JSON quoting** — JSON requires double quotes. If you write
  `hx-vals="{day: '{{ day }}'}"`, HTMX silently sends nothing.

## Tech / library notes

- `Route` is a thin model: id, FK volunteer, date, status, timestamps.
  All packing / assignment logic lives in `apps/delivery/services/dispatch.py`
  (Story 4.7).
- `created_at` / `updated_at` use `auto_now_add` / `auto_now` so the
  schema's `TIMESTAMP NULL` columns are populated by Django on insert /
  update. The DB allows NULL but our code always writes a value.
- The schema only indexes `volunteer_id`. We add a second composite
  `(volunteer_id, route_date)` index because Story 4.7's idempotency
  check is `Route.objects.filter(volunteer=v, route_date=d)`.
- **Why no `UniqueConstraint(volunteer, route_date)`?** The route packer
  may create multiple chunks for one volunteer on one day if a kitchen
  serves >12 members. Each chunk becomes its own `Route`. The
  idempotency strategy is "wipe-and-rebuild for `planned` routes" not
  "skip duplicates" ‚Äî see Story 4.7.

- [ ] **Step 3.1 ‚Äî Create `apps/delivery/models/routes.py`.**

```python
from django.conf import settings
from django.db import models


class Route(models.Model):
    """Schema only ‚Äî one volunteer's deliveries on one day.

    Status transitions and packing logic live in
    `apps/delivery/services/dispatch.py` (Story 4.7).
    """

    STATUS_PLANNED = "planned"
    STATUS_IN_PROGRESS = "in_progress"
    STATUS_COMPLETED = "completed"
    STATUS_CANCELLED = "cancelled"

    STATUS_CHOICES = [
        (STATUS_PLANNED, "Planned"),
        (STATUS_IN_PROGRESS, "In progress"),
        (STATUS_COMPLETED, "Completed"),
        (STATUS_CANCELLED, "Cancelled"),
    ]

    volunteer = models.ForeignKey(
        settings.AUTH_USER_MODEL,
        on_delete=models.PROTECT,
        related_name="routes",
        db_column="volunteer_id",
    )
    route_date = models.DateField()
    status = models.CharField(
        max_length=20, choices=STATUS_CHOICES, default=STATUS_PLANNED
    )
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    class Meta:
        app_label = "delivery"
        db_table = "routes"
        indexes = [
            models.Index(fields=["volunteer"], name="idx_route_volunteer"),
            models.Index(fields=["volunteer", "route_date"], name="idx_route_vol_date"),
            models.Index(fields=["route_date"], name="idx_route_date"),
        ]

    def __str__(self) -> str:
        return f"Route #{self.pk or '?'} {self.route_date} vol={self.volunteer_id}"
```

## Pitfalls reviewer will check

- **`on_delete=PROTECT` on `volunteer`** ‚Äî schema implies a hard FK. Don't
  use `CASCADE` (would delete every route when a volunteer is removed ‚Äî
  catastrophic for audit/history). `PROTECT` forces explicit reassignment.
- **`auto_now_add` / `auto_now` vs `default=timezone.now`** ‚Äî both work,
  but the auto variants are the convention used in `accounts.User`. Stay
  consistent.
- **No `UniqueConstraint(volunteer, route_date)`** ‚Äî see Tech notes. The
  packer creates one `Route` *per chunk*, not one per volunteer per day.
- **`indexes` order** ‚Äî `(volunteer, route_date)` not `(route_date,
  volunteer)`. The hot path is "this volunteer's routes today" not
  "everyone on this date".
- **Admin filter naming** ‚Äî `list_filter = ("status", "route_date",
  "volunteer")`. Putting `volunteer` first would render a giant dropdown
  before the more selective `status` filter.
- **`app_label`** ‚Äî `"delivery"`, not `"apps.delivery"`. Same reason as
  Story 4.1.

## User story

> **As a** prospective member
> **I want** to enter my delivery address with city + postcode
> **so that** the charity knows where to deliver.

## Acceptance criteria

- [ ] Route `GET /apply/address/` renders step 2 of the wizard.
- [ ] Step 2 requires `request.session["application_id"]` — if missing, redirect to `/apply/` (step 1).
- [ ] Step 2 requires the application to be in `status='draft'` — if not, redirect to `/apply/` and clear the session key.
- [ ] Form fields: `label` (default `"Home"`, max 120), `street` (required, max 255), `postal_code` (required, max 20), `city` (required, dropdown bound to `City.objects.all()`).
- [ ] On submit, the existing `Application` row is **updated in place** (no new row is created) with the address fields. The visitor is redirected to `/apply/dietary/` (step 3 — built in Story 1.9; 404 until then is acceptable).
- [ ] The progress indicator shows "Step 2 of 3" with dot 1 muted and dot 2 active.
- [ ] Mobile-first at 375 px: inputs span full width, no horizontal scroll, touch targets ≥ 44 px.
- [ ] No `Address` row is written. The user does not exist yet — `user_addresses.user_id` cannot point anywhere. The real `Address` row is created on approval (Story 1.12).

## Tech / library notes

- Map preview is **out of scope**. No geocoding here; `latitude` / `longitude` stay null and only get computed on approval (or in a later story — flagged in Epic 01).
- The `city` field on the form is `ModelChoiceField(queryset=City.objects.all())`. Django renders it as a native `<select>`, which is fine on mobile.
- Service signature: `update_application_address(*, application, label, street, postal_code, city) -> Application`. Pass the model instance, not just an id, so the view's session-loaded row is the one mutated.
- Validation: postcode is a string (max 20). No country-specific format check yet — Sprint 06 polish story will add it.

from apps.accounts.models import Application, User  # already imported above

# ... existing create_draft_application above ...


def update_application_address(
    *,
    application: Application,
    label: str,
    street: str,
    postal_code: str,
    city,
) -> Application:
    """Step 2 — write the address fields onto the existing draft row.

    Raises:
        ValueError: if the application is not in `draft` state.
    """
    if application.status != Application.STATUS_DRAFT:
        raise ValueError(
            f"Application {application.id} is not in draft state "
            f"(status={application.status})"
        )
    application.address_label = (label or "Home").strip()
    application.street = street.strip()
    application.postal_code = postal_code.strip()
    application.city_id = city.id
    application.save(update_fields=[
        "address_label", "street", "postal_code", "city_id", "updated_at",
    ])
    return application
```

from django.shortcuts import redirect, render
from django.views.decorators.http import require_http_methods

from apps.accounts.forms import ApplicationAddressForm, ApplicationContactForm
from apps.accounts.models import Application
from apps.accounts.services import (
    create_draft_application,
    update_application_address,
)


# ... application_step_1 above ...


def _load_draft_from_session(request) -> Application | None:
    """Return the draft Application bound to this session, or None.

    Side-effect: clears the session key if the application is no longer in
    draft state (already submitted / approved / rejected).
    """
    app_id = request.session.get("application_id")
    if not app_id:
        return None
    try:
        app = Application.objects.get(id=app_id)
    except Application.DoesNotExist:
        request.session.pop("application_id", None)
        return None
    if app.status != Application.STATUS_DRAFT:
        request.session.pop("application_id", None)
        return None
    return app


@require_http_methods(["GET", "POST"])
def application_step_2(request):
    application = _load_draft_from_session(request)
    if application is None:
        return redirect("/apply/")

    if request.method == "POST":
        form = ApplicationAddressForm(request.POST)
        if form.is_valid():
            update_application_address(
                application=application,
                **form.cleaned_data,
            )
            return redirect("/apply/dietary/")
    else:
        form = ApplicationAddressForm(
            initial={
                "label": application.address_label or "Home",
                "street": application.street or "",
                "postal_code": application.postal_code or "",
                "city": application.city_id,
            }
        )

    return render(
        request,
        "accounts/application/step_2.html",
        {"form": form, "step": 2, "application": application},
    )
```

## Pitfalls reviewer will check

- **Second `Application` row.** If the view calls `create_draft_application` again, you'll get two rows per visitor. The service must take the existing instance.
- **`Address` row written prematurely.** This story must NOT write `user_addresses` — the `user_id` FK has nowhere to point. Grep for `Address.objects.create` and confirm absent.
- **City dropdown empty.** `City.objects.all()` is empty until `seed_cities` runs. The unit test creates one inline; the Playwright spec assumes the seed has run. Document this in the PR.
- **City queryset evaluated at import time.** `ModelChoiceField(queryset=City.objects.all())` is fine because Django re-evaluates per request — but if anyone "optimises" it to `City.objects.all()` at module level outside the form definition, new rows added after server start would not appear. Leave it as-is.
- **`update_fields=[…]` missing `updated_at`.** Without it, `auto_now=True` would not fire and the row's `updated_at` would stay at create-time. The reviewer will check the row's `updated_at` shifts.
- **Form `initial` from a model on GET.** The view passes initial values from the application instance so revisiting `/apply/address/` after a back-button doesn't blank the form.
- **Postcode whitespace.** `"3000 "` should be saved as `"3000"`. The service `.strip()`s every text field.

## User story

> **As a** logged-in donor
> **I want** to see my donation history with the option to download a receipt
> **so that** I can manage my giving without phoning the office.

## Acceptance criteria

- [ ] Route `/donor/history/`, gated by `@role_required('donor')`.
- [ ] Lists every `donations` row for the requesting user where
      `status IN ('completed', 'refunded')`. `pending` and `failed` are
      hidden from the donor (operations problem, not a giving record).
- [ ] Columns:
  - Date (`created_at`, formatted `DD Mon YYYY`).
  - Amount (formatted from `amount_cents`: `$AMOUNT.cc`).
  - Campaign (`campaigns.name`).
  - Status (`completed` shown as "Receipted", `refunded` shown as "Refunded").
  - "Download receipt" link → `/donor/history/<donation_id>/receipt/`
    (HTML page; PDF generation deferred to backlog 5.10).
- [ ] Aggregate header:
  - **Total this calendar year** — `SUM(amount_cents)` where
    `status='completed' AND created_at` in current Melbourne calendar year.
  - **Lifetime total** — `SUM(amount_cents)` where `status='completed'`
    across all time.
- [ ] **Object-level permission:** the donor sees ONLY their own
      donations; the receipt-detail view 404s for any other donor's id.
- [ ] Mobile-first 375 px breakpoint passes Playwright smoke
      (header doesn't wrap awkwardly; columns stack on narrow widths).

## Tech / library notes

- **Cents formatting.** Use a small `format_aud(cents)` helper in
  `apps/core/money.py` (create if absent). Tested in isolation. Format:
  `$1,234.56` with comma thousands.
- **Calendar year boundary.** "This calendar year" in `Australia/Melbourne`
  — derive from `timezone.localdate().year`.
- **Status mapping.** Map enum value → human label in the template, not
  in the model: `{ "completed": "Receipted", "refunded": "Refunded" }`.
- **Mobile-first.** Tailwind's default is mobile-first. Use a
  `card`-per-row layout under `md:` and a `<table>` from `md:` up.
- **Receipt detail.** This story renders an HTML page only. PDF download
  is backlog story 5.10 (do not pre-build).

```python
def format_aud(cents: int) -> str:
    """Format cents as $X,XXX.XX (no fancy locale juggling)."""
    if cents is None:
        return "$0.00"
    dollars, c = divmod(int(cents), 100)
    return f"${dollars:,}.{c:02d}"
```

from __future__ import annotations

from django.db.models import Sum
from django.utils import timezone


def list_for(donor_id: int) -> list[dict]:
    from apps.fundraising.models import Donation
    qs = (
        Donation.objects
        .filter(donor_id=donor_id, status__in=["completed", "refunded"])
        .select_related("campaign")
        .order_by("-created_at")
    )
    return [
        {
            "id": d.id,
            "created_at": d.created_at,
            "amount_cents": d.amount_cents,
            "campaign_name": d.campaign.name,
            "status": d.status,
        }
        for d in qs
    ]


def aggregate_for(donor_id: int) -> dict:
    from apps.fundraising.models import Donation
    year = timezone.localdate().year
    ytd = (
        Donation.objects
        .filter(donor_id=donor_id, status="completed",
                created_at__year=year)
        .aggregate(s=Sum("amount_cents"))["s"] or 0
    )
    lifetime = (
        Donation.objects
        .filter(donor_id=donor_id, status="completed")
        .aggregate(s=Sum("amount_cents"))["s"] or 0
    )
    return {"ytd_cents": ytd, "lifetime_cents": lifetime}
```

from django.shortcuts import get_object_or_404, render

from apps.core.decorators import role_required
from apps.dashboards.services import donor_history


@role_required("donor")
def history(request):
    return render(request, "dashboards/donor/history.html", {
        "donations": donor_history.list_for(request.user.id),
        "aggregate": donor_history.aggregate_for(request.user.id),
    })


@role_required("donor")
def receipt(request, donation_id: int):
    from apps.fundraising.models import Donation
    donation = get_object_or_404(
        Donation,
        id=donation_id,
        donor_id=request.user.id,  # object-level guard
        status__in=["completed", "refunded"],
    )
    return render(request, "dashboards/donor/receipt.html", {
        "donation": donation,
    })
```

## Pitfalls reviewer will check

- **Showing pending donations.** A pending donation is a transient
  payment-processor state; revealing it confuses donors.
- **Cents formatting.** `f"${cents/100}"` produces `$25.0`, not
  `$25.00`. Use `format_aud`.
- **Year boundary bug.** `created_at__year` uses the project's
  TIME_ZONE; double-check by seeding a Jan-1 Melbourne donation.
- **Receipt accessible to other donors.** Without
  `donor_id=request.user.id` in the receipt view, donor B can read
  donor A's receipts via URL.
- **Template tag library not loaded.** Forgetting `{% load money %}`
  silently renders nothing.
- **Mobile table overflow.** A `<table>` at 375 px without `overflow-x-auto`
  blows the layout. The story uses a card-list under `md:` to avoid it.

## User story

> **As an** admin opening the laptop on Monday morning
> **I want** one screen that surfaces every actionable thing across the charity
> **so that** I do not have to remember which queue I have not checked.

## Acceptance criteria

- [ ] Route `/admin/home/` returns 200 for users with `role='admin'` and
      redirects everyone else with 403 (via `@role_required('admin')`).
- [ ] Bare path `/` resolves to the same view when the requester is an
      authenticated admin; non-admins continue to the existing role-based
      landing page (kept by the `/` dispatcher introduced earlier).
- [ ] The page renders five cards, each emitting `count` + `link`:
  - **Pending applications** — `submitted` applications (from Epic 01).
  - **Expiring stock** — `ingredient_batches.expiration_date <= today + 3`,
    grouped by `kitchen.name`.
  - **Failed deliveries today** — `deliveries` where
    `status='failed' AND scheduled_date = today`.
  - **Unassigned deliveries today** — `deliveries` where
    `volunteer_id IS NULL AND scheduled_date = today` (overflow from
    Story 4.7).
  - **Recent food-safety failures** — `food_safety_checks.result='fail'`
    in the last 24 hours.
- [ ] Each card is colour-coded by severity (traffic-light):
  - `green` (count == 0)
  - `yellow` (1 ≤ count < threshold)
  - `red` (count ≥ threshold)
  - Thresholds live in `apps/dashboards/services/admin_summary.py`
    (single source of truth, defaulted per card).
- [ ] Cards refresh via HTMX every 5 minutes
      (`hx-get` + `hx-trigger="every 300s"`); a partial template returns the
      card body without the chrome.
- [ ] All counts come from a single service call
      `admin_summary.build()` so a future test only stubs one function.
- [ ] Time zone: all "today" boundaries computed in `Australia/Melbourne`.

## Tech / library notes

- **Schema is read-only.** No new models. Aggregation lives in
  `apps/dashboards/services/`. The view is a thin call into the service.
- **`Card` dataclass.** Return cards as a small frozen dataclass
  (`title`, `count`, `link`, `severity`, `threshold`) so the template
  iterates a uniform shape.
- **Traffic-light helper.** A pure function
  `severity(count, threshold)` returning `"green" | "yellow" | "red"`.
  Test in isolation.
- **HTMX cadence.** 5 minutes = 300 seconds. Apply
  `hx-trigger="load, every 300s"` so the first render happens immediately.
- **Time zone.** Use `django.utils.timezone.localdate()` after
  configuring `TIME_ZONE = "Australia/Melbourne"`. Do not pass naive
  `datetime.now()` into the ORM filter.
- **Severity thresholds (defaults).** Encode them in the service module:
  pending apps 10, expiring stock 5, failed deliveries 1, unassigned
  deliveries 3, food-safety failures 1.
- **CSS.** Reuse the `card` component class from `static/src/input.css`.
  Severity is applied with `border-l-4 border-{color}-500` Tailwind classes.

### Task 2: Build the aggregation service (GREEN — part 1)

**Files:** `apps/dashboards/services/admin_summary.py`

- [ ] **Step 2.1 — Create `apps/dashboards/services/admin_summary.py`.**

```python
"""Admin home dashboard aggregation.

Pure read-only queries; no side effects. The view layer maps each Card
to its template severity colour.
"""
from __future__ import annotations

from dataclasses import dataclass
from datetime import timedelta
from typing import Literal

from django.urls import reverse
from django.utils import timezone

Severity = Literal["green", "yellow", "red"]


@dataclass(frozen=True)
class Card:
    title: str
    count: int
    link: str
    severity: Severity
    threshold: int


# Per-card thresholds. Adjust here; do not hard-code in templates.
_THRESHOLDS = {
    "pending_apps": 10,
    "expiring_stock": 5,
    "failed_deliveries": 1,
    "unassigned_deliveries": 3,
    "fs_failures": 1,
}


def severity(count: int, *, threshold: int) -> Severity:
    if count == 0:
        return "green"
    if count >= threshold:
        return "red"
    return "yellow"


def _count_pending_applications() -> int:
    from apps.accounts.models import Application  # local import to avoid cycles

    return Application.objects.filter(status="submitted").count()


def _count_expiring_stock() -> int:
    from apps.kitchens.models import IngredientBatch

    today = timezone.localdate()
    return IngredientBatch.objects.filter(
        expiration_date__lte=today + timedelta(days=3),
    ).count()


def _count_failed_deliveries_today() -> int:
    from apps.deliveries.models import Delivery

    return Delivery.objects.filter(
        status="failed",
        scheduled_date=timezone.localdate(),
    ).count()


def _count_unassigned_deliveries_today() -> int:
    from apps.deliveries.models import Delivery

    return Delivery.objects.filter(
        volunteer_id__isnull=True,
        scheduled_date=timezone.localdate(),
    ).count()


def _count_fs_failures_24h() -> int:
    from apps.kitchens.models import FoodSafetyCheck

    since = timezone.now() - timedelta(hours=24)
    return FoodSafetyCheck.objects.filter(
        result="fail",
        checked_at__gte=since,
    ).count()


def build() -> list[Card]:
    specs = [
        ("Pending applications", _count_pending_applications,
         "dashboards:applications_list", _THRESHOLDS["pending_apps"]),
        ("Expiring stock", _count_expiring_stock,
         "dashboards:expiring_stock", _THRESHOLDS["expiring_stock"]),
        ("Failed deliveries today", _count_failed_deliveries_today,
         "dashboards:failed_deliveries_today", _THRESHOLDS["failed_deliveries"]),
        ("Unassigned deliveries today", _count_unassigned_deliveries_today,
         "dashboards:unassigned_deliveries_today",
         _THRESHOLDS["unassigned_deliveries"]),
        ("Recent food-safety failures", _count_fs_failures_24h,
         "dashboards:fs_failures_recent", _THRESHOLDS["fs_failures"]),
    ]
    cards: list[Card] = []
    for title, fn, route_name, threshold in specs:
        try:
            link = reverse(route_name)
        except Exception:
            link = "#"
        count = fn()
        cards.append(Card(
            title=title,
            count=count,
            link=link,
            severity=severity(count, threshold=threshold),
            threshold=threshold,
        ))
    return cards
```

> The `try/except` around `reverse()` is intentional: earlier sprints may
> not yet ship every filtered-list route; the card still renders with
> `link="#"`. A follow-up story replaces `"#"` once the filtered list lands.

## Pitfalls reviewer will check

- **Time zone bug.** Using `datetime.now().date()` produces UTC-day, not
  Melbourne-day. Use `timezone.localdate()` after setting `TIME_ZONE`.
- **N+1 risk.** Each card calls a `.count()`. That's five queries — fine.
  Do not "optimise" to one query with subqueries; the readability win is
  worth it and the page is admin-only.
- **HTMX partial bleeds chrome.** If the partial template extends
  `base.html`, the 5-minute refresh re-injects the whole page. Partial
  must be a fragment, not an extending template.
- **Threshold lives in template.** Don't hard-code thresholds in HTML;
  the service is the source of truth.
- **Permission decorator missing.** Without `@role_required('admin')`,
  any logged-in user can hit `/admin/home/`.
- **Reverse-resolution loop.** A missing destination URL must degrade to
  `link="#"`, not 500.

## User story

> **As a** partner staff member at a referring charity
> **I want** to see the members our organisation has referred to MerryMeal,
> their current status, average meal rating, and 90-day retention
> **so that** I can report back to my own board.

## Acceptance criteria

- [ ] Route `/partner/outcomes/`, gated by `@role_required('partner')`
      **AND** the request user must have `partner_id IS NOT NULL`.
      Users with `role='partner'` but a NULL `partner_id` are returned
      a 403 explaining "Your account is not linked to a partner organisation".
- [ ] Table columns:
  - Member name (`users.full_name`).
  - Suburb (derived from `user_addresses.city.name` — primary address).
  - Status (`active` if `users.is_active=1 AND users.deleted_at IS NULL`,
    else `inactive`).
  - Enrolment date (`users.created_at`).
  - Last delivery (`MAX(deliveries.delivered_time)` for that member).
  - Average rating (`AVG(delivery_feedback.rating)`) — display "—" if no ratings.
- [ ] Aggregate header on top of the table:
  - **Total referred** — count of users with `users.partner_id = me.partner_id`.
  - **Currently active** — same scope filtered to `is_active=1`.
  - **90-day retention %** — of members enrolled ≥ 90 days ago, % still
    active. If denominator is 0 → display `—`.
- [ ] CSV export at `/partner/outcomes/?format=csv` returning the same
      rows with a UTF-8 BOM (for Excel).
- [ ] **Security:** a partner user **must not** be able to see, by any
      means (URL fudging, query string, CSV), rows belonging to another
      `partner_id`. Object-level filter lives in the service.
- [ ] Mobile-readable; desktop-primary at 1024 px.

## Tech / library notes

- **Object-level permission.** The view derives `partner_id` from
  `request.user.partner_id`, passes it to the service, and the service
  filters every query by that value. The view never accepts a
  `partner_id` URL parameter. Even if a malicious admin guesses an
  alternative, there is no parameter to manipulate.
- **CSV.** Use Python's stdlib `csv` with `StringIO`, prepend
  `﻿` (BOM) so Excel detects UTF-8 correctly.
- **Suburb lookup.** Joining `user_addresses → cities` is sufficient. If
  a member has multiple addresses, pick the first by `id` (deterministic).
- **Aggregations.** Pre-compute `last_delivery` and `avg_rating` with
  `annotate(...)` to avoid N+1.
- **Retention.** "90-day retention" = of users where
  `created_at <= today - 90d`, fraction with `is_active=1`. If no
  qualifying users: render "—", not `0%`.
- **Time zone.** `Australia/Melbourne` for `today` boundaries.

### Task 2: Aggregation service (GREEN — part 1)

**Files:** `apps/dashboards/services/partner_outcomes.py`

- [ ] **Step 2.1 — Create `apps/dashboards/services/partner_outcomes.py`.**

```python
"""Partner outcomes — read-only aggregation, partner-scoped.

The single entry point is `build(partner_id)`; it returns a dict with
`rows` and `aggregate`. The view layer is responsible for choosing the
partner_id (always from `request.user.partner_id` — never from URL/query).
"""
from __future__ import annotations

from dataclasses import dataclass, asdict
from datetime import timedelta
from typing import Optional

from django.db.models import Avg, Max, OuterRef, Subquery
from django.utils import timezone


@dataclass(frozen=True)
class OutcomeRow:
    member_id: int
    full_name: str
    suburb: str
    status: str
    enrolment_date: str
    last_delivery: Optional[str]
    avg_rating: Optional[float]


def _base_queryset(partner_id: int):
    from apps.accounts.models import User
    return User.objects.filter(role="member", partner_id=partner_id)


def _rows(partner_id: int) -> list[OutcomeRow]:
    from apps.accounts.models import User
    from apps.deliveries.models import Delivery, DeliveryFeedback

    last_delivery = (
        Delivery.objects
        .filter(member_id=OuterRef("pk"))
        .order_by("-delivered_time")
        .values("delivered_time")[:1]
    )
    avg_rating = (
        DeliveryFeedback.objects
        .filter(delivery__member_id=OuterRef("pk"))
        .values("delivery__member_id")
        .annotate(avg=Avg("rating"))
        .values("avg")[:1]
    )
    qs = (
        _base_queryset(partner_id)
        .annotate(
            last_delivery=Subquery(last_delivery),
            avg_rating=Subquery(avg_rating),
        )
        .order_by("full_name")
        .values(
            "id", "full_name", "is_active", "created_at",
            "last_delivery", "avg_rating",
        )
    )
    # Suburb: a separate small query to keep readable.
    from apps.accounts.models import UserAddress
    suburbs = dict(
        UserAddress.objects.filter(user_id__in=[u["id"] for u in qs])
        .order_by("user_id", "id")
        .values_list("user_id", "city__name")
    )
    rows = []
    for u in qs:
        rows.append(OutcomeRow(
            member_id=u["id"],
            full_name=u["full_name"],
            suburb=suburbs.get(u["id"], ""),
            status="active" if u["is_active"] else "inactive",
            enrolment_date=u["created_at"].date().isoformat() if u["created_at"] else "",
            last_delivery=(
                u["last_delivery"].date().isoformat() if u["last_delivery"] else None
            ),
            avg_rating=(
                float(u["avg_rating"]) if u["avg_rating"] is not None else None
            ),
        ))
    return rows


def _aggregate(partner_id: int) -> dict:
    qs = _base_queryset(partner_id)
    total = qs.count()
    active = qs.filter(is_active=True).count()
    cutoff = timezone.localdate() - timedelta(days=90)
    enrolled_90d_ago = qs.filter(created_at__date__lte=cutoff)
    denom = enrolled_90d_ago.count()
    if denom == 0:
        retention = None
    else:
        retained = enrolled_90d_ago.filter(is_active=True).count()
        retention = round(retained / denom * 100, 1)
    return {
        "total_referred": total,
        "currently_active": active,
        "retention_pct_90d": retention,
    }


def build(partner_id: int) -> dict:
    return {
        "rows": [asdict(r) for r in _rows(partner_id)],
        "aggregate": _aggregate(partner_id),
    }
```

```python
import csv
from io import StringIO

from django.http import HttpResponse, HttpResponseForbidden
from django.shortcuts import render

from apps.core.decorators import role_required
from apps.dashboards.services import partner_outcomes


@role_required("partner")
def outcomes(request):
    partner_id = request.user.partner_id
    if not partner_id:
        return HttpResponseForbidden(
            "Your account is not linked to a partner organisation. "
            "Contact MerryMeal admin to fix this."
        )
    data = partner_outcomes.build(partner_id)

    if request.GET.get("format") == "csv":
        return _csv_response(data)
    return render(request, "dashboards/partner/outcomes.html", data)


def _csv_response(data: dict) -> HttpResponse:
    buf = StringIO()
    buf.write("﻿")  # BOM so Excel reads as UTF-8
    writer = csv.writer(buf)
    writer.writerow([
        "Member", "Suburb", "Status", "Enrolment date",
        "Last delivery", "Average rating",
    ])
    for r in data["rows"]:
        writer.writerow([
            r["full_name"], r["suburb"], r["status"],
            r["enrolment_date"], r["last_delivery"] or "",
            f"{r['avg_rating']:.1f}" if r["avg_rating"] is not None else "",
        ])
    resp = HttpResponse(buf.getvalue(), content_type="text/csv; charset=utf-8")
    resp["Content-Disposition"] = 'attachment; filename="partner-outcomes.csv"'
    return resp
```

## Pitfalls reviewer will check

- **Trusting URL/query for `partner_id`.** If any code path accepts
  `partner_id` from `request.GET` or path, that's a critical bug. The
  ONLY source is `request.user.partner_id`.
- **Returning a 500 on cross-partner attempt.** Behaviour must be an
  empty result, not an exception. The isolation test asserts on this.
- **NULL handling for `retention_pct_90d`.** 0/0 must render `—`, not
  `NaN` or "0.0%".
- **Excel BOM omitted.** Without `﻿`, accented suburbs render as
  garbage in Excel.
- **N+1 on `last_delivery` / `avg_rating`.** Use Subquery, not Python
  loops calling `Delivery.objects.filter(member=...)` per member.
- **Status derivation drift.** "active" vs `is_active=1` — keep
  consistent; do not invent extra states ("paused") without schema.
- **Decorator stacking.** `@role_required('partner')` alone is not
  enough; the view body must short-circuit when `partner_id is None`.

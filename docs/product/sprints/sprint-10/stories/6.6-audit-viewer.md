## User story

> **As an** admin
> **I want** to search the audit log by user, by object type, and by date range
> **so that** I can answer "who changed Margaret's diet on 12 March?".

## Acceptance criteria

- [ ] Route `/admin/audit/`, gated by `@role_required('admin')`.
- [ ] Filters (GET query string):
  - `email=` — free-text match against the actor's email (icontains).
  - `object_type=` — content-type label
    (e.g. `accounts.user`, `accounts.member_profile`).
  - `from=YYYY-MM-DD` and `to=YYYY-MM-DD` — inclusive date range.
  - `action=` — `create | update | delete | access`.
- [ ] Pagination: 50 entries per page (`?page=N`); prev/next links.
- [ ] Each row collapses to show a diff panel revealing the
      before/after JSON change (Alpine.js `x-data="{open: false}"`,
      `<details>` element acceptable).
- [ ] **Read-only:** no UI affordance to create, edit, or delete an
      audit row. No form on the page. No POST handler.
- [ ] Reads from `django-auditlog` tables exclusively. No writes.
- [ ] Performance: list query uses indexed columns
      (`actor__email`, `content_type_id`, `timestamp`); a unit test
      asserts `≤ 4` SQL queries on a page render with 50 rows.

## Tech / library notes

- **`django-auditlog` model.** `auditlog.models.LogEntry`:
  - `actor` (FK to user, nullable for system actions)
  - `content_type` (FK)
  - `object_id`, `object_repr`
  - `action` (0=create, 1=update, 2=delete, 3=access)
  - `changes` (JSONField; `{"field": [before, after], ...}`)
  - `timestamp`
- **Action choices.** Map URL value → integer:
  `{"create": 0, "update": 1, "delete": 2, "access": 3}`.
- **Email filter.** `LogEntry.objects.filter(actor__email__icontains=q)`.
  Index assumption: `users.email` is already unique-indexed.
- **Diff panel.** Render `changes` as a small `<table>` of
  `field | before | after`. Use Alpine.js `x-data="{open:false}"` to
  collapse by default; click row header to expand.
- **Pagination.** Django's `Paginator(qs, 50)`.
- **Read-only enforcement.** The URL conf contains GET only. No
  `delete_view`. The Django admin's `auditlog.LogEntry` `has_add_permission`,
  `has_change_permission`, `has_delete_permission` should return False
  for non-superusers via a sibling admin override — add it as a
  defence-in-depth step.

### Task 2: Audit query service (GREEN — part 1)

- [ ] **Step 2.1 — Create the file.**

```python
"""Audit log read-only queries.

This module **never** writes to LogEntry. The viewer is for inspection
only. Even helper functions should return querysets, not instances.
"""
from __future__ import annotations

from datetime import date
from typing import Optional

from auditlog.models import LogEntry
from django.contrib.contenttypes.models import ContentType

ACTION_CHOICES = {
    "create": LogEntry.Action.CREATE,
    "update": LogEntry.Action.UPDATE,
    "delete": LogEntry.Action.DELETE,
    "access": LogEntry.Action.ACCESS,
}


def filtered(
    *,
    email: Optional[str] = None,
    object_type: Optional[str] = None,
    date_from: Optional[date] = None,
    date_to: Optional[date] = None,
    action: Optional[str] = None,
):
    qs = LogEntry.objects.select_related("actor", "content_type").order_by("-timestamp")
    if email:
        qs = qs.filter(actor__email__icontains=email)
    if object_type:
        app_label, _, model = object_type.partition(".")
        try:
            ct = ContentType.objects.get(app_label=app_label, model=model)
            qs = qs.filter(content_type=ct)
        except ContentType.DoesNotExist:
            qs = qs.none()
    if date_from:
        qs = qs.filter(timestamp__date__gte=date_from)
    if date_to:
        qs = qs.filter(timestamp__date__lte=date_to)
    if action and action in ACTION_CHOICES:
        qs = qs.filter(action=ACTION_CHOICES[action])
    return qs
```

from datetime import date

from django.core.paginator import Paginator
from django.shortcuts import render
from django.views.decorators.http import require_GET

from apps.core.decorators import role_required
from apps.dashboards.services import audit_query


def _parse_date(s):
    if not s:
        return None
    try:
        return date.fromisoformat(s)
    except ValueError:
        return None


@require_GET  # explicitly disallow POST/PUT/DELETE
@role_required("admin")
def audit_viewer(request):
    qs = audit_query.filtered(
        email=request.GET.get("email") or None,
        object_type=request.GET.get("object_type") or None,
        date_from=_parse_date(request.GET.get("from")),
        date_to=_parse_date(request.GET.get("to")),
        action=request.GET.get("action") or None,
    )
    paginator = Paginator(qs, 50)
    page = paginator.get_page(request.GET.get("page") or 1)
    return render(request, "dashboards/admin/audit.html", {
        "page_obj": page,
        "filters": {
            "email": request.GET.get("email", ""),
            "object_type": request.GET.get("object_type", ""),
            "from": request.GET.get("from", ""),
            "to": request.GET.get("to", ""),
            "action": request.GET.get("action", ""),
        },
    })
```

from auditlog.admin import LogEntryAdmin
from auditlog.models import LogEntry
from django.contrib import admin


# Reopen and harden auditlog's admin
class ReadOnlyLogEntryAdmin(LogEntryAdmin):
    def has_add_permission(self, request, obj=None):
        return False

    def has_change_permission(self, request, obj=None):
        return False

    def has_delete_permission(self, request, obj=None):
        return False


try:
    admin.site.unregister(LogEntry)
except admin.sites.NotRegistered:
    pass
admin.site.register(LogEntry, ReadOnlyLogEntryAdmin)
```

## Pitfalls reviewer will check

- **Writeable UI sneaks in.** A POST handler, a `delete_view`, a "purge"
  button — none of these are allowed. Reviewer checks the URL conf and
  the admin.
- **N+1 on actor / content_type.** Forgetting `select_related` fires a
  per-row SQL query. Reviewer runs the page with 50 rows and counts
  queries (≤ 4 acceptable).
- **`object_type` matched naively.** A typo like `accounts.User` (capital)
  should not 500; service catches `ContentType.DoesNotExist` and
  returns `.none()`.
- **Timestamp filter using `__date__` against UTC**. Make sure the
  filter uses project TIME_ZONE; a Melbourne admin filtering for
  "12 March" must see Melbourne-day rows.
- **Diff panel renders raw HTML.** Audit `changes` may contain HTML;
  Django auto-escapes by default — do NOT use `|safe`.
- **Page link drift.** Building `?page=N` URLs by string concat breaks
  when multiple filters are present. Use Django's `querystring` template
  tag or `request.GET.copy()` in the view.

## User story

> **As a** recurring donor
> **I want** to manage or cancel my subscription via email
> **so that** I do not have to phone or email the charity to stop giving.

## Acceptance criteria

- [ ] Route `/donate/manage/` (public). Form: single `email` field +
      "Send link" button.
- [ ] On submit, regardless of whether the email matches a donor:
  - Render a generic "If that email has a subscription, we've sent a link"
    page. **Never** leak whether the email exists.
  - If the email **does** match at least one active recurring donation,
    send an email containing a signed magic link.
- [ ] Magic-link token is a `django.core.signing.dumps(...)` of
      `{"email": <email>, "tid": <token_id>}` with `salt="donations.manage"`
      and `max_age=1800` (30 minutes).
- [ ] **Single-use** enforced via a tiny `MagicLinkToken` model storing
      `(token_id, used_at)`. First click sets `used_at=now()`; subsequent
      clicks → 410 Gone.
- [ ] Magic-link landing page `/donate/manage/<token>/` lists the donor's
      **active** recurring donations:
  - amount (formatted via `dollars`),
  - campaign name,
  - next charge date (read from the linked Stripe `Subscription` via
    dj-stripe's models, or via direct `stripe.Subscription.retrieve` —
    cached for the request).
  - "Cancel subscription" form (POST, CSRF on).
- [ ] Cancel POST calls `stripe.Subscription.delete(sub_id)` (or
      `cancel_at_period_end=True` for graceful cancellation — pick the
      latter; reviewer's call).
- [ ] When Stripe's `customer.subscription.deleted` webhook arrives, the
      handler from Story 5.4 flips matching donations' `status` to
      `cancelled` (already implemented; this story relies on it).
- [ ] Mobile-first at 375 px (Playwright).
- [ ] Token-expiry test: a token older than 30 min returns 410.
- [ ] Re-use test: a used token returns 410 with a "request a new link" CTA.
- [ ] **Rate-limit** the "send link" form to 5 requests per hour per IP
      (use a simple cache-based counter; Story 0.x ratelimit library if available).

## Tech / library notes

- **Why not a Django session-based login?** The donor has no account. A
  magic link is the smallest possible auth surface.
- **Why signed token + DB-side single-use?** `django.core.signing` alone
  gives expiry and tamper-resistance but not single-use. The
  `MagicLinkToken` row holds the `used_at` flag; the token's payload
  carries the row's `token_id`. Two layers of defense.
- **Token ID = `uuid4().hex`.** Stored in `MagicLinkToken.token_id` (unique
  index). The signed payload is `{"email": ..., "tid": ...}`.
- **`MagicLinkToken` retention:** keep used tokens for 30 days so we can
  audit (re-use attempts, etc.) then prune via a periodic Q2 task. Not in
  this story — a follow-up.
- **Next-charge date:** if dj-stripe's `Subscription.current_period_end`
  is populated (its sync job ran), read from there. Otherwise call
  `stripe.Subscription.retrieve(sub_id)` and cache for the request.
- **Cancel mode:** `cancel_at_period_end=True` (graceful) — donor keeps
  benefit of paid period, then no more charges. Webhook
  `customer.subscription.deleted` fires at period end.
- **Public route, generic response:** the `send_magic_link` flow must not
  reveal whether the email is on file. Render the same "we've sent a link"
  page regardless.

```python
from django.db import models


class MagicLinkToken(models.Model):
    """Single-use, 30-min token for the donation-management magic-link flow."""

    token_id = models.CharField(max_length=64, unique=True, default=lambda: uuid.uuid4().hex)
    email = models.EmailField(db_index=True)
    used_at = models.DateTimeField(null=True, blank=True)
    created_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        app_label = "donations"
        db_table = "donation_magic_link_tokens"

    def __str__(self) -> str:
        return f"{self.email} ({'used' if self.used_at else 'unused'})"
```

### Task 2: Implement `services/manage.py` (GREEN — part 1)

```python
from __future__ import annotations

import uuid

import stripe
from django.conf import settings
from django.core.signing import BadSignature, SignatureExpired, dumps, loads
from django.utils import timezone

from apps.donations.models import Donation, MagicLinkToken

_SALT = "donations.manage"
_MAX_AGE = 30 * 60   # 30 minutes


def issue_token(email: str) -> str:
    record = MagicLinkToken.objects.create(email=email, token_id=uuid.uuid4().hex)
    return dumps({"email": email, "tid": record.token_id}, salt=_SALT)


def verify_token(signed: str, *, mark_used: bool = False) -> dict:
    payload = loads(signed, salt=_SALT, max_age=_MAX_AGE)
    try:
        record = MagicLinkToken.objects.get(token_id=payload["tid"], email=payload["email"])
    except MagicLinkToken.DoesNotExist:
        raise BadSignature("Unknown token")
    if record.used_at is not None:
        raise BadSignature("Token already used")
    if mark_used:
        record.used_at = timezone.now()
        record.save(update_fields=["used_at"])
    return payload


def send_magic_link(email: str) -> None:
    """Idempotent. Only sends if a recurring donation exists for the email."""
    from django.core.mail import EmailMultiAlternatives
    from django.template.loader import render_to_string

    has_sub = Donation.objects.filter(
        donor_email=email,
        is_recurring=True,
        stripe_subscription_id__isnull=False,
    ).exclude(status="cancelled").exists()
    if not has_sub:
        return   # silent — DO NOT leak existence

    token = issue_token(email)
    link = settings.SITE_BASE_URL.rstrip("/") + f"/donate/manage/{token}/"
    ctx = {"link": link}
    msg = EmailMultiAlternatives(
        subject="Manage your MerryMeal donation",
        body=render_to_string("donations/emails/magic_link.txt", ctx),
        from_email=settings.DONATIONS_FROM_EMAIL,
        to=[email],
    )
    msg.attach_alternative(render_to_string("donations/emails/magic_link.html", ctx), "text/html")
    msg.send()


def list_active_subscriptions(email: str) -> list[Donation]:
    """Latest row per Stripe subscription. Pulled from dj-stripe if available."""
    rows = (
        Donation.objects
        .filter(donor_email=email, is_recurring=True,
                stripe_subscription_id__isnull=False)
        .exclude(status="cancelled")
        .order_by("stripe_subscription_id", "-created_at")
        .distinct("stripe_subscription_id")
    )
    return list(rows)


def cancel_subscription(*, email: str, subscription_id: str) -> None:
    """Verify the email actually owns the subscription before calling Stripe."""
    owns = Donation.objects.filter(
        donor_email=email, stripe_subscription_id=subscription_id,
    ).exists()
    if not owns:
        raise PermissionError("This email does not own that subscription.")
    stripe.api_key = settings.STRIPE_SECRET_KEY
    stripe.Subscription.modify(subscription_id, cancel_at_period_end=True)
```

> MySQL doesn't support `distinct(field)` — replace with a `Subquery`
> selecting the latest row per `stripe_subscription_id` in MySQL. Reviewer's
> call; tests below are MySQL-agnostic.

```python
from unittest.mock import patch  # only for type-hint hover; remove if unused

from django.core.cache import cache
from django.core.signing import BadSignature, SignatureExpired
from django.shortcuts import redirect, render
from django.views.decorators.http import require_GET, require_POST, require_http_methods

from apps.donations.forms.manage import MagicLinkRequestForm
from apps.donations.services.manage import (
    cancel_subscription,
    list_active_subscriptions,
    send_magic_link,
    verify_token,
)

_RATE_KEY = "donations:manage:req:{ip}"
_RATE_LIMIT = 5
_RATE_WINDOW = 3600


@require_http_methods(["GET", "POST"])
def manage_request(request):
    if request.method == "GET":
        return render(request, "donations/manage_request.html",
                      {"form": MagicLinkRequestForm()})
    # POST
    ip = request.META.get("REMOTE_ADDR", "0.0.0.0")
    cache_key = _RATE_KEY.format(ip=ip)
    hits = cache.get(cache_key, 0)
    if hits >= _RATE_LIMIT:
        return render(request, "donations/manage_sent.html",
                      {"rate_limited": True}, status=429)
    cache.set(cache_key, hits + 1, _RATE_WINDOW)

    form = MagicLinkRequestForm(request.POST)
    if form.is_valid():
        send_magic_link(form.cleaned_data["email"])
    return render(request, "donations/manage_sent.html", {})


@require_GET
def manage_landing(request, token: str):
    try:
        payload = verify_token(token, mark_used=True)
    except (BadSignature, SignatureExpired):
        return render(request, "donations/manage_gone.html", status=410)
    subs = list_active_subscriptions(payload["email"])
    # Store the email in session so the cancel POST below can authorise the donor
    # without re-reading the (now-used) token.
    request.session["donations.manage.email"] = payload["email"]
    return render(request, "donations/manage_list.html",
                  {"subscriptions": subs, "email": payload["email"]})


@require_POST
def manage_cancel(request):
    email = request.session.get("donations.manage.email")
    sub_id = request.POST.get("subscription_id", "")
    if not email or not sub_id:
        return render(request, "donations/manage_gone.html", status=410)
    try:
        cancel_subscription(email=email, subscription_id=sub_id)
    except PermissionError:
        return render(request, "donations/manage_gone.html", status=410)
    return redirect("/donate/manage/cancelled/")
```

```python
import re
from unittest.mock import patch

import pytest
from django.core import mail
from django.urls import reverse

from apps.donations.models import Donation
from apps.donations.tests.factories import CampaignFactory


def _make_recurring_donation(email="a@x.com"):
    c = CampaignFactory(slug="general-fund")
    return Donation.objects.create(
        campaign=c, donor_email=email, amount_cents=2000,
        payment_type="card", status="completed", is_recurring=True,
        stripe_subscription_id="sub_test_X",
    )


@pytest.mark.django_db
def test_request_sends_link_only_for_known_email(client):
    _make_recurring_donation("known@x.com")
    client.post(reverse("donations_manage:request"), {"email": "unknown@x.com"})
    assert len(mail.outbox) == 0   # silent

    client.post(reverse("donations_manage:request"), {"email": "known@x.com"})
    assert len(mail.outbox) == 1
    assert "donate/manage/" in mail.outbox[0].body


@pytest.mark.django_db
def test_landing_lists_active_subscriptions(client):
    _make_recurring_donation()
    client.post(reverse("donations_manage:request"), {"email": "a@x.com"})
    body = mail.outbox[0].body
    match = re.search(r"/donate/manage/(?P<tok>[^/\s]+)/", body)
    token = match.group("tok")
    resp = client.get(f"/donate/manage/{token}/")
    assert resp.status_code == 200
    assert b"sub_test_X" in resp.content
    assert b"$20.00" in resp.content


@pytest.mark.django_db
def test_cancel_calls_stripe(client):
    _make_recurring_donation()
    client.post(reverse("donations_manage:request"), {"email": "a@x.com"})
    token = re.search(r"/donate/manage/([^/\s]+)/", mail.outbox[0].body).group(1)
    client.get(f"/donate/manage/{token}/")   # session primed
    with patch("stripe.Subscription.modify") as m:
        resp = client.post(reverse("donations_manage:cancel"),
                           {"subscription_id": "sub_test_X"})
    assert resp.status_code == 302
    m.assert_called_once_with("sub_test_X", cancel_at_period_end=True)


@pytest.mark.django_db
def test_token_reuse_returns_410(client):
    _make_recurring_donation()
    client.post(reverse("donations_manage:request"), {"email": "a@x.com"})
    token = re.search(r"/donate/manage/([^/\s]+)/", mail.outbox[0].body).group(1)
    client.get(f"/donate/manage/{token}/")
    resp = client.get(f"/donate/manage/{token}/")
    assert resp.status_code == 410
```

## Pitfalls reviewer will check

- **Existence leak in the request response** — biggest privacy risk.
  Reviewer manually POSTs two different emails and confirms identical
  pages and identical status codes.
- **`max_age` not enforced** — `signing.loads` without `max_age=1800`
  accepts decade-old tokens.
- **Single-use not enforced** — without `MagicLinkToken.used_at`, a stolen
  link works forever within 30 min. Test covers it.
- **Cancel without ownership check** — `cancel_subscription` must verify
  the email actually owns `subscription_id`. Otherwise anyone with a
  guess of `sub_…` can cancel anyone.
- **`distinct("field")` on MySQL** — not supported. Replace with subquery
  before merge.
- **CSRF off on cancel POST** — never. The POST is from our own page;
  CSRF must stay on.
- **Rate-limit cache absent in tests** — `LocMem` cache is fine for unit
  tests; Redis in prod. Document in PR.
- **`request.session` lost between landing and cancel** — confirm session
  middleware writes the cookie on the GET response. If the test
  `client.session` doesn't persist, switch to verifying via a fresh
  signed token tied to the subscription_id directly.

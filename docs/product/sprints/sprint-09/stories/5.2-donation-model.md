## User story

> **As a** developer
> **I want** one `Donation` row per pledge with payment status and processor reference
> **so that** the system has a single source of truth for raised money and
> Stripe webhooks can update the row idempotently.

## Acceptance criteria

- [ ] `apps/donations/models/donations.py::Donation` matching schema
      (`donations` table in `merrymeal_schema_corrected.sql`):
  - `donor_id` — `BigIntegerField`, **nullable** (anonymous donors don't
    have an account; schema requires NOT NULL but the donate page is
    public — this is a deliberate, documented divergence).
  - `campaign_id` — `ForeignKey(Campaign, on_delete=PROTECT)`.
  - `amount_cents` — `BigIntegerField` (integer cents only).
  - `payment_type` — enum: `card`, `bank_transfer`, `cash`, `paypal`.
  - `status` — enum: `pending`, `completed`, `failed`, `refunded`,
    plus an extra `cancelled` (used by Story 5.7).
  - `transaction_id` — `CharField(max_length=191, null=True, unique=True)`.
    Uniqueness is the **idempotency hinge** for Story 5.4's webhook.
- [ ] Additional non-schema fields needed by later stories:
  - `donor_email` — `EmailField` (anonymous donor identity for receipts).
  - `is_recurring` — `BooleanField(default=False)`.
  - `stripe_subscription_id` — `CharField(max_length=191, null=True, blank=True)`.
  - `receipt_number` — `CharField(max_length=32, null=True, blank=True, unique=True)`.
  - `updated_at` — `DateTimeField(auto_now=True)` (needed for "next charge" UI in 5.7).
- [ ] Model is **schema only** — zero methods beyond `__str__`.
- [ ] Admin list:
  - Columns: `created_at`, `donor_email`, `campaign`, `amount_display`,
    `status`, `payment_type`, `transaction_id`.
  - Filters: `campaign`, `status`, `payment_type`, `is_recurring`.
  - Default sort: `-amount_cents` (largest first, satisfies acceptance criterion).
  - Search: `donor_email`, `transaction_id`.
- [ ] `DonationFactory` is added to `apps/donations/tests/factories.py` for
      downstream stories.

## Tech / library notes

- **`transaction_id` uniqueness** is mandatory: Story 5.4's webhook
  idempotency depends on `Donation.objects.get_or_create(transaction_id=...)`.
  Without `unique=True` we'd double-count on re-fires.
- **`status` enum** stores values as lowercase strings matching the schema's
  MySQL `ENUM`. Plus `cancelled` for subscriptions (Story 5.7) — not in the
  schema; documented divergence.
- **`donor_id` nullable:** the schema declares NOT NULL, but the public
  donate page (Story 5.3) accepts donations without an account. We capture
  `donor_email` instead and back-link a `donor_id` later if/when the email
  matches a future `User` (Epic 06). Reviewer must approve the divergence
  in this PR's description.
- **No `Decimal` ever.** `amount_cents` is `BigIntegerField`.
- **Indexes:** `(campaign_id, status)` covers the admin filter; add an index
  on `donor_email` for Story 5.7's "find my subscriptions" lookup.
- **`receipt_number`** format is `D<YYYY>-<NNNNNN>` — generated by the
  receipt service in Story 5.5, not the model.

```python
from django.db import models

from apps.donations.models.campaigns import Campaign


class Donation(models.Model):
    """Schema only. State transitions live in apps.donations.services."""

    PAYMENT_TYPE_CHOICES = [
        ("card", "Card"),
        ("bank_transfer", "Bank transfer"),
        ("cash", "Cash"),
        ("paypal", "PayPal"),
    ]

    STATUS_CHOICES = [
        ("pending", "Pending"),
        ("completed", "Completed"),
        ("failed", "Failed"),
        ("refunded", "Refunded"),
        ("cancelled", "Cancelled"),   # subscription cancelled (Story 5.7)
    ]

    # Schema-aligned columns.
    donor_id = models.BigIntegerField(null=True, blank=True)  # nullable: anonymous donors
    campaign = models.ForeignKey(
        Campaign, on_delete=models.PROTECT, related_name="donations",
        db_column="campaign_id",
    )
    amount_cents = models.BigIntegerField()
    payment_type = models.CharField(max_length=20, choices=PAYMENT_TYPE_CHOICES)
    status = models.CharField(max_length=20, choices=STATUS_CHOICES, default="pending")
    transaction_id = models.CharField(max_length=191, null=True, blank=True, unique=True)

    # Extra columns for the donate flow.
    donor_email = models.EmailField()
    is_recurring = models.BooleanField(default=False)
    stripe_subscription_id = models.CharField(max_length=191, null=True, blank=True)
    receipt_number = models.CharField(max_length=32, null=True, blank=True, unique=True)

    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    class Meta:
        app_label = "donations"
        db_table = "donations"
        indexes = [
            models.Index(fields=["donor_id"], name="idx_don_donor"),
            models.Index(fields=["campaign", "status"], name="idx_don_campaign_status"),
            models.Index(fields=["donor_email"], name="idx_don_email"),
        ]

    def __str__(self) -> str:
        return f"#{self.pk} {self.donor_email} {self.amount_cents}c {self.status}"
```

import pytest
from django.urls import reverse

from apps.donations.tests.factories import CampaignFactory, DonationFactory


@pytest.fixture
def admin_user(django_user_model):
    return django_user_model.objects.create_superuser(
        email="admin@example.com", full_name="A", role="admin", password="x",
    )


@pytest.mark.django_db
def test_donation_admin_filters_by_campaign(client, admin_user):
    winter = CampaignFactory(name="Winter")
    summer = CampaignFactory(name="Summer")
    DonationFactory(campaign=winter, amount_cents=10_00, status="completed")
    DonationFactory(campaign=summer, amount_cents=20_00, status="completed")
    client.force_login(admin_user)
    url = reverse("admin:donations_donation_changelist")
    resp = client.get(url, {"campaign__id__exact": winter.id})
    assert resp.status_code == 200
    assert b"Winter" in resp.content


@pytest.mark.django_db
def test_donation_admin_default_order_is_amount_desc(client, admin_user):
    c = CampaignFactory()
    DonationFactory(campaign=c, amount_cents=1_00)
    DonationFactory(campaign=c, amount_cents=100_00)
    client.force_login(admin_user)
    resp = client.get(reverse("admin:donations_donation_changelist"))
    # The $100 row should appear before the $1 row in the rendered HTML.
    body = resp.content.decode()
    assert body.index("$100.00") < body.index("$1.00")

## Pitfalls reviewer will check

- **`amount_cents = DecimalField(...)`** — wrong. Must be `BigIntegerField`.
- **`transaction_id` not unique** — breaks Story 5.4 idempotency. Reviewer
  greps for `unique=True` on this field.
- **Model gains methods like `mark_completed()`** — schema only. Move to
  `services/payments.py` in Story 5.4.
- **`donor_id` declared NOT NULL** — donate page is public. NULL must be
  allowed; the PR description must explain the schema divergence.
- **`on_delete=CASCADE` on campaign FK** — wrong. Use `PROTECT` so an admin
  cannot accidentally drop the campaign and lose receipts at tax time.
- **`receipt_number` not unique** — Story 5.5 generates them in series;
  uniqueness is the safety net.
- **Missing index on `donor_email`** — Story 5.7 reads by email; without
  the index the manage-link lookup is a full table scan.

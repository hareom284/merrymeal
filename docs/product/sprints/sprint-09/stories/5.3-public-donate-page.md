## User story

> **As a** stranger with $50 to spare
> **I want** to give money in under 30 seconds without creating an account
> **so that** the donation happens before I second-guess.

## Acceptance criteria

- [ ] Public route `/donate/` (no login required, CSRF on).
- [ ] Hero section:
  - If `?campaign=<slug>` matches an active `Campaign`, show its name and
    a Tailwind progress bar (`raised / goal`) computed via
    `apps.donations.services.campaigns.raised_cents_for`.
  - Otherwise show "General fund" with no progress bar.
- [ ] Amount chips: `$20`, `$50`, `$100`, plus a `Custom` input
      (min $1, max $10,000). Each chip has a small caption "≈ N meals"
      from Story 5.6 (placeholder copy until 5.6 lands — see notes).
- [ ] Toggle (Alpine.js `x-model`): "Make this monthly" → maps to
      `is_recurring=True` on the resulting `Donation`.
- [ ] Email input (required, validated). Used as `donor_email`.
- [ ] Big `Donate` CTA — full-width on mobile, minimum height **48 px**
      (touch target).
- [ ] On submit (POST `/donate/start/`):
  - Validate via `apps.donations.forms.donate.DonateForm`.
  - Service `apps.donations.services.donate.start_donation(...)` creates a
    `Donation` row in `status='pending'`, **integer cents only**, then
    delegates to `apps.donations.services.stripe.create_checkout_session`
    (Story 5.4 — stub for now, returns `"https://stripe.test/sess_stub"`).
  - HTTP 302 → returned Stripe URL.
- [ ] Mobile-first at 375 × 667 px (iPhone SE). Playwright spec:
  - No horizontal scrollbar.
  - All chips have `clientHeight ≥ 44 px`.
  - `Donate` button has `clientHeight ≥ 48 px`.
  - Apple Pay / Google Pay surface is delegated to Stripe Checkout (covered
    by manual verification in Story 5.4 — not asserted here).
- [ ] `?cancelled=1` query string renders a soft toast "Cancelled — your
      card wasn't charged" above the form.

## Tech / library notes

- **Service signature:**
  `start_donation(*, campaign_slug: str | None, amount_cents: int, donor_email: str, is_recurring: bool) -> tuple[Donation, str]`
  returns the `Donation` (pending) and the Stripe URL.
- **Campaign resolution:** if `campaign_slug` is `None` or unmatched, fall
  back to a sentinel "General fund" campaign — created by data migration
  (Admin/Ops track in the sprint plan). The form view does **not** create
  it on the fly.
- **Currency UX:** the `Custom` input is `inputmode="decimal"` for the
  numeric keypad on mobile but we round to **integer cents** server-side:
  `int(round(float(value) * 100))`. **This is the only place we cast a
  user-typed dollar string to cents — done once, in the form's `clean_amount_dollars`.**
- **No Stripe in this story.** The Stripe service is a stub that returns
  `"https://stripe.test/sess_stub_<donation_id>"`. Story 5.4 replaces it.
- **Alpine.js for chips:** `x-data="{ amount: 5000 }"` and chips push to it.
  Custom input updates the same `amount` via `@input.debounce`.
- **Tailwind:** reuse `.btn-primary`, `.input` from `static/src/input.css`.
  Chip styles: rounded `border-warm-200` selected state `bg-brand-green`.

`apps/donations/forms/donate.py`:

```python
from decimal import Decimal, InvalidOperation

from django import forms

MIN_DOLLARS = Decimal("1.00")
MAX_DOLLARS = Decimal("10000.00")


class DonateForm(forms.Form):
    amount_dollars = forms.CharField(required=True)
    donor_email = forms.EmailField(required=True)
    is_recurring = forms.BooleanField(required=False)
    campaign_slug = forms.CharField(required=False, max_length=255)

    def clean_amount_dollars(self) -> int:
        raw = (self.cleaned_data.get("amount_dollars") or "").strip().lstrip("$").replace(",", "")
        try:
            dollars = Decimal(raw)
        except (InvalidOperation, ValueError):
            raise forms.ValidationError("Enter a valid amount in dollars.")
        if dollars < MIN_DOLLARS:
            raise forms.ValidationError("Minimum donation is $1.")
        if dollars > MAX_DOLLARS:
            raise forms.ValidationError("Maximum online donation is $10,000.")
        cents = int((dollars * 100).to_integral_value())   # half-even rounding
        self.cleaned_data["amount_cents"] = cents
        return raw

    def clean(self):
        cleaned = super().clean()
        if "amount_cents" not in cleaned and "amount_dollars" in cleaned:
            # Defensive — if clean_amount_dollars set it, it'll be here already.
            pass
        return cleaned
```

- [ ] **Step 3.3 — Implement `start_donation`.**

`apps/donations/services/donate.py`:

```python
from __future__ import annotations

from apps.donations.models import Campaign, Donation
from apps.donations.services.stripe import create_checkout_session

GENERAL_FUND_SLUG = "general-fund"


def _resolve_campaign(slug: str | None) -> Campaign:
    if slug:
        try:
            return Campaign.objects.get(slug=slug, is_active=True)
        except Campaign.DoesNotExist:
            pass
    return Campaign.objects.get(slug=GENERAL_FUND_SLUG)


def start_donation(*, campaign_slug: str | None, amount_cents: int,
                   donor_email: str, is_recurring: bool) -> tuple[Donation, str]:
    if not isinstance(amount_cents, int):
        raise TypeError("amount_cents must be int (integer cents)")
    campaign = _resolve_campaign(campaign_slug)
    donation = Donation.objects.create(
        campaign=campaign,
        donor_email=donor_email,
        amount_cents=amount_cents,
        payment_type="card",
        status="pending",
        is_recurring=is_recurring,
    )
    url = create_checkout_session(donation.id, recurring=is_recurring)
    return donation, url
```

- [ ] **Step 3.4 — Implement the views.**

`apps/donations/views/donate.py`:

```python
from django.shortcuts import redirect, render
from django.views.decorators.http import require_GET, require_POST

from apps.donations.forms.donate import DonateForm
from apps.donations.models import Campaign
from apps.donations.services.campaigns import raised_cents_for
from apps.donations.services.donate import GENERAL_FUND_SLUG, start_donation


@require_GET
def donate_page(request):
    slug = request.GET.get("campaign") or GENERAL_FUND_SLUG
    campaign = Campaign.objects.filter(slug=slug, is_active=True).first() \
        or Campaign.objects.filter(slug=GENERAL_FUND_SLUG).first()
    raised = raised_cents_for(campaign) if campaign else 0
    return render(request, "donations/donate.html", {
        "form": DonateForm(initial={"campaign_slug": slug if slug != GENERAL_FUND_SLUG else ""}),
        "campaign": campaign,
        "raised_cents": raised,
        "cancelled": request.GET.get("cancelled") == "1",
    })


@require_POST
def donate_start(request):
    form = DonateForm(request.POST)
    if not form.is_valid():
        return render(request, "donations/donate.html", {
            "form": form,
            "campaign": Campaign.objects.filter(slug=GENERAL_FUND_SLUG).first(),
            "raised_cents": 0,
            "cancelled": False,
        })
    _donation, url = start_donation(
        campaign_slug=form.cleaned_data.get("campaign_slug") or None,
        amount_cents=form.cleaned_data["amount_cents"],
        donor_email=form.cleaned_data["donor_email"],
        is_recurring=form.cleaned_data["is_recurring"],
    )
    return redirect(url)
```

## Pitfalls reviewer will check

- **Float math on the dollar value** — `clean_amount_dollars` must use
  `Decimal` and `int(...)`, never plain `float * 100`.
- **General-fund campaign not seeded** — Admin/Ops track ships a data
  migration; if missing, the form's "no slug" path 500s. Reviewer
  re-runs `python manage.py migrate` on a fresh DB and confirms.
- **CTA below 48 px** — touch-target regression. Playwright catches it.
- **Horizontal scroll on 375 px** — usually a missing `max-w-md` or a
  too-wide chip grid. Playwright catches it.
- **Email validation client-only** — server form must also validate.
- **Recurring toggle not propagated** — confirm `is_recurring=True` shows
  up on the DB row when the checkbox is checked.
- **CSRF disabled** — never. The `donate_start` view must accept POST with CSRF.

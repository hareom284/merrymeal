## User story

> **As an** admin
> **I want** a live campaign-progress dashboard with per-campaign donation lists
> **so that** I know whether to push the appeal harder this week and can
> pull a CSV at tax time.

## Acceptance criteria

- [ ] Route `/admin/campaigns/` (note: distinct from Django's built-in
      `/admin/donations/campaign/`). Lives under `apps.dashboards`, gated
      by `@role_required("admin")` (the decorator already exists in
      `apps/core/decorators.py`).
- [ ] Lists **active** campaigns (`is_active=True`) only. Each row shows:
  - campaign name + slug,
  - progress bar (raised / goal),
  - raised in dollars + goal in dollars (via the `dollars` template tag),
  - progress percentage,
  - **days remaining** until `end_at` (or "—" if open-ended).
- [ ] Each row links to `/admin/campaigns/<slug>/` — a detail page with:
  - the same progress header,
  - a paginated list of donations (page size = 25, default sort
    `-created_at`),
  - filters: status (`completed`, `pending`, `refunded`, `cancelled`).
- [ ] "Download CSV" link on the detail page → `/admin/campaigns/<slug>/export.csv`.
  CSV columns: `created_at`, `donor_email`, `amount_cents`, `amount_dollars`,
  `status`, `payment_type`, `transaction_id`, `receipt_number`,
  `is_recurring`. Header row included.
- [ ] **Money invariant:** all displayed dollar amounts come from the
      `dollars` filter (no inline string formatting).
- [ ] Mobile-readable (not mobile-primary); responsive Playwright at
      **1024 px** width.
- [ ] CSV export uses Python's `csv.writer` with an HTTP streaming
      response (avoids loading all donations into memory for large campaigns).

## Tech / library notes

- **Why not extend Django Admin?** The Django Admin changelist (Story 5.1)
  is sufficient for CRUD but cluttered for "executive summary at a glance".
  This story builds a clean, branded view on top.
- **Progress snapshot dataclass:**

```python
@dataclass(frozen=True, slots=True)
class CampaignProgress:
    campaign: Campaign
    raised_cents: int
    goal_cents: int
    pct: int            # capped to 100
    days_remaining: int | None
```

- **Days remaining:** `(campaign.end_at.date() - today_in_melbourne).days`,
  clamped to ≥ 0; `None` if `end_at is None`.
- **CSV streaming:** Django's `StreamingHttpResponse` + a `csv.writer`
  writing to a small `Echo` buffer.
- **Pagination:** Django's `Paginator` with `?page=N`.
- **Access control:** `@role_required("admin")` — already in `apps/core/decorators.py`.
- **Don't use `Donation.objects.all()` for export.** Always scope to the
  campaign and use `iterator(chunk_size=500)` so we never load 10 k
  donations into memory.

### Task 2: Implement the service (GREEN — part 1)

`apps/dashboards/services/campaign_progress.py`:

```python
from __future__ import annotations

import datetime as dt
from dataclasses import dataclass

from django.db.models import Sum
from django.utils import timezone

from apps.donations.models import Campaign, Donation


@dataclass(frozen=True, slots=True)
class CampaignProgress:
    campaign: Campaign
    raised_cents: int
    goal_cents: int
    pct: int
    days_remaining: int | None


def _raised(campaign: Campaign) -> int:
    return (
        Donation.objects
        .filter(campaign=campaign, status="completed")
        .aggregate(t=Sum("amount_cents"))["t"]
        or 0
    )


def _days_remaining(campaign: Campaign) -> int | None:
    if campaign.end_at is None:
        return None
    today = timezone.localdate()
    end = timezone.localtime(campaign.end_at).date()
    return max(0, (end - today).days)


def progress_snapshot(campaign: Campaign) -> CampaignProgress:
    raised = _raised(campaign)
    pct = 0 if not campaign.goal_cents else min(100, raised * 100 // campaign.goal_cents)
    return CampaignProgress(
        campaign=campaign,
        raised_cents=raised,
        goal_cents=int(campaign.goal_cents),
        pct=int(pct),
        days_remaining=_days_remaining(campaign),
    )


def list_active_campaigns() -> list[CampaignProgress]:
    return [progress_snapshot(c) for c in Campaign.objects.filter(is_active=True).order_by("name")]
```

```python
import csv

from django.core.paginator import Paginator
from django.http import Http404, StreamingHttpResponse
from django.shortcuts import get_object_or_404, render
from django.views.decorators.http import require_GET

from apps.core.decorators import role_required
from apps.dashboards.services.campaign_csv import iter_csv_rows
from apps.dashboards.services.campaign_progress import (
    list_active_campaigns,
    progress_snapshot,
)
from apps.donations.models import Campaign, Donation


class _Echo:
    def write(self, value):
        return value


@require_GET
@role_required("admin")
def index(request):
    return render(request, "dashboards/admin/campaigns.html", {
        "campaigns": list_active_campaigns(),
    })


@require_GET
@role_required("admin")
def detail(request, slug: str):
    campaign = get_object_or_404(Campaign, slug=slug)
    snap = progress_snapshot(campaign)
    qs = Donation.objects.filter(campaign=campaign).order_by("-created_at")
    status_filter = request.GET.get("status")
    if status_filter:
        qs = qs.filter(status=status_filter)
    paginator = Paginator(qs, 25)
    page = paginator.get_page(request.GET.get("page"))
    return render(request, "dashboards/admin/campaign_detail.html", {
        "snap": snap, "page": page, "status_filter": status_filter or "",
    })


@require_GET
@role_required("admin")
def export_csv(request, slug: str):
    campaign = get_object_or_404(Campaign, slug=slug)
    qs = Donation.objects.filter(campaign=campaign).order_by("created_at")
    writer = csv.writer(_Echo())
    response = StreamingHttpResponse(
        (writer.writerow(row) for row in iter_csv_rows(qs)),
        content_type="text/csv",
    )
    response["Content-Disposition"] = f'attachment; filename="{slug}-donations.csv"'
    return response
```

`apps/dashboards/services/campaign_csv.py`:

```python
from __future__ import annotations

from typing import Iterable

from apps.donations.models import Donation


def iter_csv_rows(qs) -> Iterable[list[str]]:
    yield [
        "created_at", "donor_email", "amount_cents", "amount_dollars",
        "status", "payment_type", "transaction_id", "receipt_number",
        "is_recurring",
    ]
    for d in qs.iterator(chunk_size=500):
        dollars = f"{d.amount_cents // 100}.{d.amount_cents % 100:02d}"
        yield [
            d.created_at.isoformat() if d.created_at else "",
            d.donor_email,
            d.amount_cents,
            dollars,
            d.status,
            d.payment_type,
            d.transaction_id or "",
            d.receipt_number or "",
            "yes" if d.is_recurring else "no",
        ]
```

## Pitfalls reviewer will check

- **`/admin/campaigns/` shadowed by Django Admin** — the include must
  precede `path("admin/", admin.site.urls)` in `config/urls.py`.
- **CSV loads entire queryset into memory** — must use `iterator(...)`
  and `StreamingHttpResponse`.
- **CSV exports `amount_cents` only, no dollars column** — humans open
  CSVs in Excel and read dollars. Reviewer opens the file and confirms.
- **Pending donations counted in `raised`** — only `completed`. Test
  guards this.
- **Days-remaining underflow** — past `end_at` must clamp to 0, not show
  negative.
- **`role_required` missing or applied as a plain decorator stack** —
  CSRF protection on POST flows still applies; this is read-only so all
  routes are `require_GET`.
- **CSV download blocked by CSRF** — no, it's GET. But ensure no `@csrf_protect`
  was added by reflex.
- **Mobile-only viewport** — this story is desktop-primary; Playwright
  uses 1024 px.

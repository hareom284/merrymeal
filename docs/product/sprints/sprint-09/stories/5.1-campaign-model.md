> **I want** to define fundraising campaigns with a goal, start/end dates, and optional partner attribution
> **so that** donations can be grouped by appeal and corporate sponsorship is visible.

## Tech / library notes

- **Money:** every monetary column is `BigIntegerField` storing **integer
  cents**. Never `DecimalField` for money, never `FloatField`. Reasoning:
  Stripe APIs talk cents, integer math is exact, and we never want a
  $0.005 rounding nasty in a receipt total.
- **Display:** the `dollars` template tag is the single conversion point.
  Format: `${dollars}.{cc}` with thousands separators (e.g. `$1,234.50`).
- **`partner_id`:** kept as a raw `BigIntegerField` (matches schema) until
  Epic 02 introduces the `Partner` model. Comment the field accordingly.
- **`slug`:** auto-fill from `name` on first save via the admin form (not the
  model — schema only). Use a `prepopulated_fields = {"slug": ("name",)}`
  on `CampaignAdmin`.
- **Tests:** `pytest-django` + `factory-boy`. Use the existing
  `pytest.ini` config; no new fixtures required.

```python
class Campaign(models.Model):
    """Schema only. See apps.donations.services.campaigns for logic."""

    name = models.CharField(max_length=255)
    slug = models.SlugField(max_length=255, unique=True)
    goal_cents = models.BigIntegerField()  # integer cents; never floats
    start_at = models.DateTimeField(null=True, blank=True)
    end_at = models.DateTimeField(null=True, blank=True)
    is_active = models.BooleanField(default=True)

    # Raw FK to partners.id (kept as BigInteger until Epic 02 adds the Partner model).
    partner_id = models.BigIntegerField(null=True, blank=True)

    created_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        app_label = "donations"
        db_table = "campaigns"
        indexes = [models.Index(fields=["partner_id"], name="idx_campaign_partner")]

    def __str__(self) -> str:
        return self.name
```

- [ ] **Step 4.3 — Implement `donation_extras.py`.**

```python
from django import template

register = template.Library()


@register.filter
def dollars(amount_cents):
    """Format integer cents as '$1,234.56'. Floats raise TypeError on purpose."""
    if amount_cents is None:
        amount_cents = 0
    if isinstance(amount_cents, float):
        raise TypeError("Money is integer cents — got float. Fix the caller.")
    if not isinstance(amount_cents, int):
        raise TypeError(f"dollars() expects int, got {type(amount_cents).__name__}")
    sign = "-" if amount_cents < 0 else ""
    cents = abs(amount_cents)
    whole, remainder = divmod(cents, 100)
    return f"{sign}${whole:,}.{remainder:02d}"
```

```python
from __future__ import annotations


def raised_cents_for(campaign) -> int:
    """Sum of completed donations for this campaign, in integer cents."""
    try:
        from apps.donations.models import Donation  # circular-safe lazy import
    except ImportError:
        return 0
    return (
        Donation.objects.filter(campaign_id=campaign.id, status="completed")
        .aggregate(total=__import__("django.db.models", fromlist=["Sum"]).Sum("amount_cents"))
        .get("total")
        or 0
    )
```

```python
from django.contrib import admin
from django.utils.html import format_html

from apps.donations.models import Campaign
from apps.donations.services.campaigns import raised_cents_for
from apps.donations.templatetags.donation_extras import dollars


@admin.register(Campaign)
class CampaignAdmin(admin.ModelAdmin):
    list_display = ("name", "is_active", "goal_display", "raised_display", "progress_bar")
    list_filter = ("is_active",)
    search_fields = ("name", "slug")
    prepopulated_fields = {"slug": ("name",)}

    @admin.display(description="Goal")
    def goal_display(self, obj):
        return dollars(obj.goal_cents)

    @admin.display(description="Raised")
    def raised_display(self, obj):
        return dollars(raised_cents_for(obj))

    @admin.display(description="Progress")
    def progress_bar(self, obj):
        raised = raised_cents_for(obj)
        pct = 0 if not obj.goal_cents else min(100, int(raised * 100 / obj.goal_cents))
        return format_html(
            '<div class="progress" style="background:#eee;width:160px;height:10px;border-radius:4px;">'
            '<div style="width:{}%;background:#0f766e;height:10px;border-radius:4px;"></div>'
            '</div> <span>{}%</span>',
            pct, pct,
        )
```

  `BigIntegerField`. Reviewer greps for `DecimalField` in `apps/donations/`.
- **Model contains business logic** — any method other than `__str__` is a
  red flag. Move it to `services/`.
- **Floats sneaking through the filter** — the `TypeError` test is the
  guard. Do not soften it to a warning.
- **Schema discrepancy: `slug` is not in `merrymeal_schema_corrected.sql`** —
  this story consciously adds it. Note it in the PR description so the
  schema doc can be updated in a follow-up.
- **Partner FK left as raw integer** — intentional until Epic 02. Reviewer
  confirms the comment is present.
- **`prepopulated_fields` missing** — admins will hate hand-typing slugs.

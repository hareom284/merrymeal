## User story

> **As a** donor who just paid
> **I want** an immediate "thank you" page and a copy of my receipt by email
> **so that** I know it went through and I have evidence at tax time.

## Acceptance criteria

- [ ] Route `/donate/thanks/?session_id=<sid>` (public). Renders:
  - Donation amount (formatted via the `dollars` template tag).
  - Campaign name.
  - Receipt number (`D<YYYY>-<NNNNNN>`).
  - "We've emailed a copy to <email>" line.
  - Impact line ("Your $50 ≈ 17 meals") from Story 5.6.
- [ ] If the session ID isn't found, render a soft "still processing" page
      with a refresh hint (the webhook usually arrives within seconds).
- [ ] **Receipt email** sent automatically when `Donation.status` transitions
      to `completed` (i.e. inside `apps.donations.services.payments`):
  - HTML + plain-text alternative (`emails/receipt.html`, `emails/receipt.txt`).
  - Includes: amount in dollars, campaign name, donation date in
    **Australia/Melbourne** timezone, receipt number, charity ABN +
    address from settings, one-line impact (Story 5.6).
  - From: `settings.DONATIONS_FROM_EMAIL` (defaults to
    `receipts@merrymeal.org.au`).
- [ ] **Receipt number generation** is atomic and gap-tolerant:
  `D<year>-<incremental zero-padded to 6 digits>`. Computed by
  `apps/donations/services/receipts.py::assign_receipt_number(donation)`.
  Counter implementation = a `select_for_update()` on the latest matching
  receipt of the same year. Reviewer can swap for a dedicated sequence
  table later.
- [ ] PDF attachment is **out of scope**; HTML email contains
  "you can print this for your records".
- [ ] Mobile-first thanks page at 375 × 667 px (Playwright).
- [ ] Charity ABN + address are read from settings:
  - `DONATIONS_CHARITY_ABN` (e.g. `12 345 678 901`)
  - `DONATIONS_CHARITY_ADDRESS` (multi-line string)
  These appear in `.env.example` as placeholders.

## Tech / library notes

- **Idempotent emailing:** call `send_receipt_email` from the
  status-transition path (`apply_checkout_completed`, `apply_invoice_paid`)
  but only when status actually moves from non-completed → completed. The
  webhook tests in Story 5.4 already exercise re-fires; assert via Django's
  `mail.outbox` that exactly **one** email was sent on a duplicate event.
- **Timezone:** receipt timestamps render in `Australia/Melbourne`. Use
  `timezone.localtime(donation.created_at)` — `USE_TZ` is `True`.
- **Receipt numbering invariants:**
  - Format: `D2026-000001`.
  - Year resets numbering — the first receipt of 2027 is `D2027-000001`.
  - Concurrency-safe under simultaneous webhooks (use
    `transaction.atomic()` + `select_for_update`).
  - **No gaps for the same row.** A `Donation` that gets a number keeps it
    forever (idempotent re-assignment is a no-op).
- **Email backend in tests:** `EMAIL_BACKEND = 'django.core.mail.backends.locmem.EmailBackend'`
  is already set in `config/settings/test.py`. If not, add it.
- **Charity ABN format** is plain text — no validation; storefront concern.

### Task 2: Implement `receipts.py` (GREEN — part 1)

```python
from __future__ import annotations

from django.conf import settings
from django.core.mail import EmailMultiAlternatives
from django.db import transaction
from django.template.loader import render_to_string
from django.utils import timezone

from apps.donations.models import Donation


def assign_receipt_number(donation: Donation) -> str:
    if donation.receipt_number:
        return donation.receipt_number

    year = timezone.localtime(donation.created_at or timezone.now()).year
    prefix = f"D{year}-"

    with transaction.atomic():
        last = (
            Donation.objects
            .select_for_update()
            .filter(receipt_number__startswith=prefix)
            .order_by("-receipt_number")
            .first()
        )
        seq = 1 if last is None else int(last.receipt_number.split("-")[1]) + 1
        donation.receipt_number = f"{prefix}{seq:06d}"
        donation.save(update_fields=["receipt_number", "updated_at"])
    return donation.receipt_number


def send_receipt_email(donation: Donation) -> None:
    """Render + send. Caller guarantees idempotency by only invoking on transition."""
    from apps.donations.services.impact import meals_for_amount   # Story 5.6

    ctx = {
        "donation": donation,
        "campaign": donation.campaign,
        "meals": meals_for_amount(donation.amount_cents),
        "abn": settings.DONATIONS_CHARITY_ABN,
        "address": settings.DONATIONS_CHARITY_ADDRESS,
        "receipt_number": assign_receipt_number(donation),
        "local_dt": timezone.localtime(donation.created_at or timezone.now()),
    }

    text = render_to_string("donations/emails/receipt.txt", ctx)
    html = render_to_string("donations/emails/receipt.html", ctx)
    msg = EmailMultiAlternatives(
        subject=f"Your MerryMeal receipt — {ctx['receipt_number']}",
        body=text,
        from_email=settings.DONATIONS_FROM_EMAIL,
        to=[donation.donor_email],
    )
    msg.attach_alternative(html, "text/html")
    msg.send()
```

from apps.donations.models import Donation

## Pitfalls reviewer will check

- **Duplicate email on re-fire** — biggest risk. The transition check is
  the guard. Reviewer runs the idempotency test.
- **`select_for_update` outside a transaction** — silent no-op on MySQL
  without `atomic()`. Reviewer greps for `atomic`.
- **Receipt number not saved before send** — if the email renders the
  number then a crash skips the save, the next donation reuses it.
  Save first, then send.
- **Timezone leak** — `donation.created_at` is UTC; without
  `timezone.localtime` the receipt prints UTC, which looks wrong to
  Aussie donors at 11 pm.
- **ABN hard-coded** — must come from settings (env-overridable).
- **HTML email missing plain-text alternative** — Gmail/Outlook spam
  scores climb. Send both via `EmailMultiAlternatives`.
- **PDF attachment added** — explicitly out of scope. Reject if attempted.

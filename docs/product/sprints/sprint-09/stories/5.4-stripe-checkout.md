## User story

> **As a** developer
> **I want** a real Stripe Checkout session for one-time and recurring donations, with
> a signature-verified webhook that flips our `Donation` to `completed`
> **so that** we hand off PCI scope, inherit Apple Pay / Google Pay, and our
> ledger is never wrong even when Stripe re-fires events.

## Acceptance criteria

- [ ] `apps/donations/services/stripe.py::create_checkout_session(donation_id, recurring: bool) -> str`
      builds a real Stripe Checkout session via `dj-stripe` and returns the URL.
- [ ] Success URL: `/donate/thanks/?session_id={CHECKOUT_SESSION_ID}`.
      Cancel URL: `/donate/?cancelled=1`.
- [ ] One-time mode (`recurring=False`): line item with `unit_amount=amount_cents`,
      `currency="aud"`, `quantity=1`, `mode="payment"`.
- [ ] Recurring mode (`recurring=True`): subscription with a monthly price
      created on the fly (or fetched from a cache keyed on amount + currency),
      `mode="subscription"`.
- [ ] Checkout metadata includes `donation_id` so the webhook can link back.
- [ ] Webhook endpoint `POST /stripe/webhook/`:
  - Signature verified using `STRIPE_WEBHOOK_SECRET` via dj-stripe's helper.
  - Bad signature → **HTTP 401**, no DB write.
  - Handles `checkout.session.completed` (one-time): flips matching
    `Donation` to `completed`, sets `transaction_id` to the session ID.
  - Handles `invoice.paid` (recurring): for the first invoice, updates the
    existing pending `Donation`; for subsequent invoices, creates a new
    `Donation` row linked by `stripe_subscription_id`.
  - Handles `customer.subscription.deleted` (for Story 5.7): flips
    matching donations' `status` to `cancelled`.
  - **Idempotent.** Re-firing the same event does **not** create a second
    row — enforced by `Donation.transaction_id` uniqueness + an explicit
    `get_or_create`.
- [ ] Env vars added to `.env.example` (no real keys):
  - `STRIPE_PUBLISHABLE_KEY=pk_test_replace_me`
  - `STRIPE_SECRET_KEY=sk_test_replace_me`
  - `STRIPE_WEBHOOK_SECRET=whsec_replace_me`
- [ ] `INSTALLED_APPS` += `djstripe`. `djstripe.middleware` configured per docs.
- [ ] Receipt-currency setting: `DONATIONS_CURRENCY = "aud"` in `config/settings/base.py`.

## Tech / library notes

- **`dj-stripe` ≥ 2.8** does the heavy lifting for signature verification.
  Use `djstripe.event_handlers` decorators to map event types → our
  service functions. Alternative: hand-write the view and call
  `stripe.Webhook.construct_event(payload, sig, secret)` ourselves; both
  patterns are tested in this story.
- **Idempotency**: the webhook calls
  `Donation.objects.update_or_create(transaction_id=session_id, defaults={...})`.
  The `unique=True` on `transaction_id` (Story 5.2) is the database-level
  safety net even if our code re-orders.
- **Recurring invoices:** Stripe sends `invoice.paid` for **each** monthly
  charge. The first one corresponds to the pending `Donation` created by
  Story 5.3; we identify it by checking `stripe_subscription_id`. Later
  invoices create new rows with the same `stripe_subscription_id` and a
  new `transaction_id=<invoice_id>`.
- **AUD currency.** Stripe expects 3-letter lower-case ISO. Pulled from
  `settings.DONATIONS_CURRENCY` so a future market change is one line.
- **Manual smoke:** `stripe listen --forward-to localhost:8000/stripe/webhook/`,
  then `stripe trigger checkout.session.completed`. Re-fire to prove no
  duplicate row.
- **Secrets:** never commit real keys. `.env` is gitignored; CI uses
  fake `whsec_test_...` injected via `pytest.ini` env config.

Replace `apps/donations/services/stripe.py`:

```python
from __future__ import annotations

import stripe
from django.conf import settings

from apps.donations.models import Donation


def _configure_sdk() -> None:
    stripe.api_key = settings.STRIPE_SECRET_KEY


def create_checkout_session(donation_id: int, recurring: bool) -> str:
    _configure_sdk()
    donation = Donation.objects.select_related("campaign").get(pk=donation_id)
    if not isinstance(donation.amount_cents, int):
        raise TypeError("amount_cents must be int (integer cents)")

    line_item = {
        "price_data": {
            "currency": settings.DONATIONS_CURRENCY,
            "product_data": {"name": f"Donation — {donation.campaign.name}"},
            "unit_amount": donation.amount_cents,
        },
        "quantity": 1,
    }
    if recurring:
        line_item["price_data"]["recurring"] = {"interval": "month"}

    session = stripe.checkout.Session.create(
        mode="subscription" if recurring else "payment",
        line_items=[line_item],
        success_url=_absolute(settings.DONATIONS_SUCCESS_URL),
        cancel_url=_absolute(settings.DONATIONS_CANCEL_URL),
        customer_email=donation.donor_email,
        metadata={"donation_id": str(donation.id)},
        subscription_data={"metadata": {"donation_id": str(donation.id)}} if recurring else None,
    )
    return session.url


def _absolute(path: str) -> str:
    """Stripe needs absolute URLs. Read base from settings.SITE_BASE_URL or env."""
    from django.conf import settings as s
    base = getattr(s, "SITE_BASE_URL", "http://localhost:8000")
    return base.rstrip("/") + path if path.startswith("/") else path
```

```python
from __future__ import annotations

from apps.donations.models import Campaign, Donation


def apply_checkout_completed(session: dict) -> Donation:
    session_id = session["id"]
    donation_id = session.get("metadata", {}).get("donation_id")
    donation, _ = Donation.objects.update_or_create(
        transaction_id=session_id,
        defaults={
            "status": "completed",
            "payment_type": "card",
        },
    )
    if donation_id and str(donation.id) != str(donation_id):
        # We found a previously-stamped row; pending row from Story 5.3 may need closing.
        Donation.objects.filter(id=donation_id, status="pending").update(
            status="completed", transaction_id=session_id,
        )
    elif donation_id:
        Donation.objects.filter(id=donation_id).update(
            status="completed", transaction_id=session_id,
        )
    return donation


def apply_invoice_paid(invoice: dict) -> Donation:
    invoice_id = invoice["id"]
    subscription_id = invoice.get("subscription")
    amount = int(invoice["amount_paid"])
    email = invoice.get("customer_email") or ""

    # First invoice flips the existing pending donation.
    pending = Donation.objects.filter(
        stripe_subscription_id=subscription_id, status="pending"
    ).first()
    if pending:
        pending.status = "completed"
        pending.transaction_id = invoice_id
        pending.save(update_fields=["status", "transaction_id", "updated_at"])
        return pending

    # Subsequent invoice → brand-new row for the same subscription.
    existing = Donation.objects.filter(transaction_id=invoice_id).first()
    if existing:
        return existing   # idempotent re-fire

    sibling = Donation.objects.filter(stripe_subscription_id=subscription_id).first()
    campaign = sibling.campaign if sibling else Campaign.objects.get(slug="general-fund")
    return Donation.objects.create(
        campaign=campaign,
        donor_email=email or (sibling.donor_email if sibling else ""),
        amount_cents=amount,
        payment_type="card",
        status="completed",
        is_recurring=True,
        stripe_subscription_id=subscription_id,
        transaction_id=invoice_id,
    )


def apply_subscription_deleted(subscription: dict) -> int:
    sub_id = subscription["id"]
    return Donation.objects.filter(stripe_subscription_id=sub_id).update(status="cancelled")
```

`apps/donations/views/checkout.py`:

```python
from __future__ import annotations

import json

import stripe
from django.conf import settings
from django.http import HttpResponse, JsonResponse
from django.views.decorators.csrf import csrf_exempt
from django.views.decorators.http import require_POST

from apps.donations.services.payments import (
    apply_checkout_completed,
    apply_invoice_paid,
    apply_subscription_deleted,
)


@csrf_exempt
@require_POST
def stripe_webhook(request):
    sig_header = request.META.get("HTTP_STRIPE_SIGNATURE", "")
    payload = request.body
    try:
        event = stripe.Webhook.construct_event(
            payload, sig_header, settings.STRIPE_WEBHOOK_SECRET,
        )
    except (ValueError, stripe.error.SignatureVerificationError):
        return HttpResponse(status=401)

    obj = event["data"]["object"]
    etype = event["type"]
    if etype == "checkout.session.completed":
        apply_checkout_completed(obj)
    elif etype == "invoice.paid":
        apply_invoice_paid(obj)
    elif etype == "customer.subscription.deleted":
        apply_subscription_deleted(obj)
    # Unknown types: 200 OK so Stripe doesn't retry forever.
    return JsonResponse({"received": True})
```

## Pitfalls reviewer will check

- **Real Stripe key in `.env.example`** — instant revert + key rotation.
- **`csrf_exempt` missing on the webhook** — Stripe doesn't send CSRF; the
  view will 403 silently and you'll discover it in production.
- **Signature verification skipped on dev** — the 401 test must run with the
  real `construct_event` mocked, not bypassed.
- **`amount_cents` arithmetic on floats** — sneaks in if you accept dollars
  on the form and convert here. The cast lives in Story 5.3's form only.
- **Idempotency relies on app code, not DB** — the `unique=True` on
  `transaction_id` must be in the migration; verify with `\d donations` (or
  `SHOW CREATE TABLE donations`).
- **Recurring branch never tested** — the `invoice.paid` test path is
  mandatory; reviewer greps for it.
- **`stripe_subscription_id` not propagated to the first row** — set it in
  Story 5.3 if `is_recurring=True`, or in the checkout-completed handler
  by reading `session.subscription`. Either is fine but pick one.
- **AUD/USD mismatch** — `unit_amount` currency must match
  `settings.DONATIONS_CURRENCY`.

## User story

> **As a** member of MerryMeal
> **I want** to see today's meal — name, photo, key ingredients, and any of my allergens flagged in red
> **so that** I know what is arriving and trust that the kitchen knows my needs.

## Acceptance criteria

- [ ] Route `/dashboard/` (the existing dashboards URL) when
      `request.user.role == 'member'` renders the new "today's meal" card.
      Other roles route to their own dashboards (caregiver: Story 3.8;
      admin: stays on Django admin / planner).
- [ ] The card displays:
  - Meal name (`font-display`, large).
  - Photo (placeholder image if `meal.photo` is absent — Story 3.11
    will add real uploads).
  - Bullet list of the meal's key ingredients (top 5 by name).
  - Delivery-status badge: "Planned · <date>" for now. (Epic 04 swaps
    this for live delivery status.)
  - Allergen warnings — for each allergen the *member* has that the
    meal contains, render a red badge: `⚠ Contains <allergen name>`.
- [ ] If there is no MealPlan for today at the member's assigned
      kitchen, show a friendly empty state: "No meal scheduled today —
      your next is on <date>" (or just "your next will be planned
      soon" if no future plan exists).
- [ ] **Mobile-first at 375 × 667** with **18 px body font**. The
      whole card must fit without horizontal scroll. Touch targets
      (any tappable elements like a "rate this meal" link from a
      later story) are ≥ 44 px.
- [ ] No N+1 queries: the view runs ≤ 4 queries (member, plan, meal +
      ingredients, member allergies).

## Tech / library notes

- The service is the only place that knows how to assemble a "today"
  view. Reused verbatim by Story 3.8's caregiver list.
- "Assigned kitchen" for v1 = the kitchen whose `service_radius_km`
  contains the member's primary address. If multiple kitchens cover the
  member, pick the **closest**. If none cover, fall back to the
  geographically closest kitchen and the member will see a frozen meal
  (which matches the policy). Encapsulate that decision in
  `apps/planning/services/assignment.py::find_serving_kitchen(member)` —
  add it to the same module since it sits next to `assign_meal_type`.
- For the allergen badge, call
  `apps.planning.services.allergen.meal_allergens_for_member(meal, member)`.
  If Story 3.5 has not merged yet, stub the import behind a
  `try/except ImportError` and return `[]` — the test for the badge can
  be marked `xfail(strict=False)` until 3.5 lands.
- 18 px body font: set on the dashboard page's `<body>` wrapper or via
  a Tailwind utility (`text-[18px] sm:text-lg`); do not bump the global
  base size — other pages assume 16 px.

### Task 2: Implement the service (GREEN — part 1)

**Files:** `apps/dashboards/services/member_today.py`

```python
# apps/dashboards/services/member_today.py
from __future__ import annotations

from typing import TypedDict
from django.utils import timezone

from apps.planning.models import MealPlan


class TodayCard(TypedDict):
    has_meal: bool
    meal_name: str | None
    meal_photo_url: str | None
    ingredient_names: list[str]
    allergens: list  # list[Allergy]
    delivery_status_label: str
    service_date: object  # date
    next_plan_date: object | None


def _find_kitchen_for(member):
    from apps.planning.services.assignment import find_serving_kitchen
    return find_serving_kitchen(member)


def get_today_card(member) -> TodayCard:
    today = timezone.localdate()
    kitchen = _find_kitchen_for(member)
    plan = None
    if kitchen is not None:
        plan = (
            MealPlan.objects
            .select_related("meal", "kitchen")
            .filter(kitchen=kitchen, service_date=today)
            .first()
        )

    if plan is None:
        next_plan = None
        if kitchen is not None:
            next_plan = (
                MealPlan.objects
                .filter(kitchen=kitchen, service_date__gt=today)
                .order_by("service_date")
                .first()
            )
        return TodayCard(
            has_meal=False, meal_name=None, meal_photo_url=None,
            ingredient_names=[], allergens=[],
            delivery_status_label="No meal scheduled today",
            service_date=today,
            next_plan_date=next_plan.service_date if next_plan else None,
        )

    meal = plan.meal
    ingredients = list(meal.ingredients.order_by("name")[:5])

    try:
        from apps.planning.services.allergen import meal_allergens_for_member
        allergens = meal_allergens_for_member(meal, member)
    except ImportError:
        allergens = []

    return TodayCard(
        has_meal=True,
        meal_name=meal.name,
        meal_photo_url=getattr(meal, "photo_url", None),
        ingredient_names=[i.name for i in ingredients],
        allergens=list(allergens),
        delivery_status_label=f"Planned · {plan.service_date:%a %d %b}",
        service_date=plan.service_date,
        next_plan_date=None,
    )
```

def find_serving_kitchen(member):
    """Closest kitchen by Haversine; None if no kitchens exist or no address."""
    from apps.kitchens.models import Kitchen
    addr = _primary_address(member)
    if addr is None:
        return None
    kitchens = list(Kitchen.objects.all())
    if not kitchens:
        return None
    return min(
        kitchens,
        key=lambda k: haversine_km(
            float(addr.latitude), float(addr.longitude),
            float(k.latitude), float(k.longitude),
        ),
    )

from django.contrib.auth.decorators import login_required
from django.shortcuts import render

@login_required
def member_dashboard_view(request):
    if request.user.role != "member":

        "greeting_name": request.user.full_name.split()[0],

def _route_non_member(request):
    from django.shortcuts import redirect
    if request.user.role == "caregiver":
        return redirect("dashboards:caregiver")  # Story 3.8 will register this
    return redirect("admin:index")

## Pitfalls reviewer will check

- **N+1 on ingredients/allergies.** Use `select_related("meal")` and
  iterate `meal.ingredients` after fetching — or `prefetch_related` if
  you also walk `contains_allergens`. The `django_assert_max_num_queries(8)`
  fence catches regressions.
- **`timezone.now()` vs `timezone.localdate()`.** Today's date in
  Melbourne, not UTC. Use `localdate()`.
- **Hardcoded "Margaret".** The mock in the previous view had a
  greeting referencing a name. Make sure your replacement reads
  `request.user.full_name` and handles single-word names without
  crashing on `.split()[0]`.
- **18 px applied globally.** Don't bump the base — apply at the page
  wrapper. Other pages (login, admin planner) should keep their sizes.
- **Allergen badge missing when 3.5 lands.** Once 3.5 ships, remove the
  `try/except ImportError` fallback so a regression is loud, not silent.
- **No-plan path 500s on `kitchen=None`.** Cover with a test (the first
  service test does).

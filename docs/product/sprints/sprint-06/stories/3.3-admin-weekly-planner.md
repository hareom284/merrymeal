## User story

> **As an** admin / dietitian
> **I want** a one-screen grid of (kitchen × day) cells where I drop a meal into each cell
> **so that** I can plan a whole week in five minutes per kitchen instead of editing rows one-by-one in the Django admin.

## Acceptance criteria

- [ ] Route `/admin/planner/` registered in `apps.planning.urls`, mounted
      from `config/urls.py` (not under `/admin/` — that's Django's own
      admin). Restricted to users with `role == 'admin'` via the
      `@role_required("admin")` decorator from `apps.core.decorators`.
- [ ] Defaults to **next week** (Mon–Sun) when no `?week=` is provided.
      Accepts `?week=YYYY-MM-DD`; uses the Monday of the week containing
      that date. Invalid input → 400 with a clear message.
- [ ] Grid layout: kitchens down the left column, days (Mon–Sun) across
      the top. Each cell shows:
  - Existing meal name (if a `MealPlan` row exists for that
    (kitchen, day)) plus the planned quantity badge.
  - A "Set meal" button when the cell is empty; "Edit" when populated.
- [ ] "Set meal" / "Edit" opens an **HTMX modal**: a `<select>` of all
      active meals + `planned_quantity` number input. Save POSTs back
      and the cell re-renders inline via HTMX swap.
- [ ] On save:
  - Create-or-update the `MealPlan` row for `(kitchen, service_date)`.
  - `day_of_week` derived from `service_date`.
  - `meal_type` set to `'frozen'` on Sat/Sun, else `'fresh'`. The
    *per-member* outside-radius case is **not** applied here — that
    happens at delivery generation time (Epic 04). This story only
    captures the kitchen-wide planning choice.
  - `published_by = request.user`.
- [ ] Desktop-primary: target viewport 1280 × 800. Must remain
      *readable* (no horizontal scroll, columns collapse to scrollable
      strip) at 768 px.

## Tech / library notes

- The `/admin/planner/` URL deliberately sits *outside* Django's admin
  site (which is at `/admin/`). Using it under `/admin/` would conflict
  with Django's catch-all admin URLs. Mount via `path("admin/planner/",
  include("apps.planning.urls"))` — Django dispatches longer paths first.
- Restriction is by **role**, not `is_staff`. `apps.core.decorators.role_required("admin")`
  redirects non-admins to the login page with a "needs admin" message.
- HTMX modal pattern: the "Set meal" button does
  `hx-get="/admin/planner/cell/?kitchen=K&date=YYYY-MM-DD"
  hx-target="#modal" hx-swap="innerHTML"`. The form POSTs to the same
  URL with `hx-target="#cell-K-DATE"` to swap the cell in place.
- The grid is **server-rendered**; HTMX only handles the modal + the
  single cell re-render. No SPA, no React.
- Mobile: the grid uses CSS Grid with `min-width` on cells; on narrow
  screens it scrolls horizontally inside its container. Add a
  `overflow-x-auto` Tailwind utility — that satisfies the
  "mobile-readable" bar without rebuilding the layout.

# apps/planning/services/planner.py
import datetime as dt

from apps.planning.models import MealPlan

DAY_KEYS = ["mon", "tue", "wed", "thu", "fri", "sat", "sun"]


def upsert_cell(*, kitchen, meal, service_date: dt.date,
                planned_quantity: int, published_by) -> MealPlan:
    """Create or update the MealPlan for (kitchen, service_date).

    Sets `day_of_week` from the date and `meal_type` to 'frozen' on Sat/Sun
    else 'fresh'. The per-member 10 km decision is applied later, at
    delivery generation (Epic 04). See `assign_meal_type` for the
    per-member rule.
    """
    weekday = service_date.weekday()
    plan, _ = MealPlan.objects.update_or_create(
        kitchen=kitchen,
        service_date=service_date,
        defaults={
            "meal": meal,
            "day_of_week": DAY_KEYS[weekday],
            "meal_type": "frozen" if weekday in (5, 6) else "fresh",
            "planned_quantity": planned_quantity,
            "published_by": published_by,
        },
    )
    return plan
```

# apps/planning/views/admin_planner.py
import datetime as dt

from django.http import HttpResponseBadRequest
from django.shortcuts import render
from django.utils import timezone

from apps.core.decorators import role_required
from apps.kitchens.models import Kitchen
from apps.planning.forms.planner import MealPlanCellForm
from apps.planning.models import MealPlan
from apps.planning.services.planner import upsert_cell


def _monday_of(d: dt.date) -> dt.date:
    return d - dt.timedelta(days=d.weekday())


def _parse_week(raw: str | None) -> dt.date:
    if not raw:
        today = timezone.localdate()
        return _monday_of(today) + dt.timedelta(days=7)  # next week
    try:
        return _monday_of(dt.date.fromisoformat(raw))
    except ValueError as exc:
        raise ValueError(f"invalid week: {raw!r}") from exc


@role_required("admin")
def planner_view(request):
    try:
        monday = _parse_week(request.GET.get("week"))
    except ValueError as exc:
        return HttpResponseBadRequest(str(exc))

    days = [monday + dt.timedelta(days=i) for i in range(7)]
    kitchens = list(Kitchen.objects.order_by("name"))
    plans = (
        MealPlan.objects
        .filter(service_date__gte=days[0], service_date__lte=days[-1])
        .select_related("meal", "kitchen")
    )
    by_cell = {(p.kitchen_id, p.service_date): p for p in plans}
    grid = [
        {
            "kitchen": k,
            "cells": [{"date": d, "plan": by_cell.get((k.id, d))} for d in days],
        }
        for k in kitchens
    ]
    return render(request, "planning/admin/planner.html", {
        "monday": monday,
        "days": days,
        "grid": grid,
        "prev_week": (monday - dt.timedelta(days=7)).isoformat(),
        "next_week": (monday + dt.timedelta(days=7)).isoformat(),
    })


@role_required("admin")
def cell_edit_view(request):
    """GET → modal partial. POST → upsert + cell partial."""
    try:
        kitchen_id = int(request.GET.get("kitchen") or request.POST.get("kitchen"))
        service_date = dt.date.fromisoformat(
            request.GET.get("date") or request.POST.get("date")
        )
    except (TypeError, ValueError):
        return HttpResponseBadRequest("bad kitchen/date")
    kitchen = Kitchen.objects.get(pk=kitchen_id)

    if request.method == "POST":
        form = MealPlanCellForm(request.POST)
        if form.is_valid():
            plan = upsert_cell(
                kitchen=kitchen,
                meal=form.cleaned_data["meal"],
                service_date=service_date,
                planned_quantity=form.cleaned_data["planned_quantity"],
                published_by=request.user,
            )
            return render(request, "planning/admin/_cell.html", {
                "kitchen": kitchen, "date": service_date, "plan": plan,
            })
    else:
        existing = MealPlan.objects.filter(
            kitchen=kitchen, service_date=service_date
        ).first()
        form = MealPlanCellForm(initial={
            "meal": existing.meal_id if existing else None,
            "planned_quantity": existing.planned_quantity if existing else 20,
        })
    return render(request, "planning/admin/_cell_form.html", {
        "form": form, "kitchen": kitchen, "date": service_date,
    })
```

## Pitfalls reviewer will check

- **URL collision with Django admin.** `/admin/planner/` must not be
  registered *inside* Django's `admin.site`. It's a normal view, mounted
  before or beside Django's `path("admin/", admin.site.urls)`.
- **`is_staff` instead of role.** A new admin who hasn't been flagged
  `is_staff` should still reach the planner — `role_required("admin")`
  is the right gate.
- **Default week.** Must default to *next* Monday, not *this* Monday.
  Editing the past week is not the common case.
- **Per-member 10 km rule leaking in.** Don't call `assign_meal_type`
  from `upsert_cell`. That function is per-(member, kitchen, date); the
  planner row is per-(kitchen, date).
- **HTMX CSRF token missing.** The modal form must include
  `{% csrf_token %}` or POSTs return 403. Easy to miss in a partial.
- **Race on update.** `update_or_create` is sufficient at this scale; if
  two admins double-save the same cell, last write wins (acceptable).

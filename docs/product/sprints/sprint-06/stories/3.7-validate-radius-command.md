## User story

> **As the** MerryMeal team
> **I want** one command that walks every active member's primary address
> and asserts the fresh/frozen decision for today is consistent with
> `assign_meal_type`
> **so that** silent drift — someone hand-edits a `MealPlan`, a member
> moves house, a kitchen radius gets nudged — does not lead to wrong
> deliveries.

## Acceptance criteria

- [ ] `python manage.py validate_radius_assignments`:
  - Exits **0** if every active member is consistent with
    `assign_meal_type` for today's date (Melbourne local).
  - Exits **1** otherwise.
  - Prints one line per inconsistency:
    `member <id> <email>  kitchen <id>  expected=<fresh|frozen>  actual=<fresh|frozen>`.
  - Prints a final summary: `OK: N members consistent` or
    `FAIL: N inconsistencies across M members`.
- [ ] Supports `--date YYYY-MM-DD` to pin the date (useful in tests &
      backfills). Default is today.
- [ ] Supports `--quiet` to suppress the per-member output (CI logs
      stay small on the happy path).
- [ ] Nightly Django-Q2 schedule wraps the command and emails the
      `ADMIN_EMAIL` env var on non-zero exit.
- [ ] Wired into CI as a **non-blocking** job — the result is visible in
      the PR check log but does not gate merge yet.

> **Definition of "actual"** for v1: the most recent `MealPlan` row for
> the (kitchen, date) pair tells us the *kitchen-wide* type. The
> per-member outside-radius case is applied at delivery generation time
> (Epic 04), so until Epic 04 lands, "actual" === "kitchen-wide
> meal_type for that plan". The command's job is to catch the
> kitchen-wide rule breaking, not to second-guess delivery generation
> that doesn't yet exist. Once Epic 04 introduces `Delivery.meal_type`,
> a follow-up story updates this command to walk `Delivery` rows.

## Tech / library notes

- Use `BaseCommand.handle` + `self.stdout` for output, `sys.exit(1)` for
  the failure path. Returning `'1'` from `handle` also works in newer
  Django but `sys.exit` is unambiguous.
- The command must **not** depend on a particular delivery model — read
  only `MealPlan`, `Kitchen`, `User`, `UserAddress`.
- Use `freezegun` / `pytest-freezer` in tests to pin "today" — the date
  default makes the command time-sensitive.
- Q2 scheduling: the project uses Django-Q2 (per epic 00). The schedule
  function lives in `config/q2_schedules.py` and is invoked from an
  `AppConfig.ready()` hook elsewhere. If that file does not yet exist,
  create it and document the entry point.
- Failure email uses `django.core.mail.send_mail` with `ADMIN_EMAIL`
  from env. Wrap the call in a `try/except` so a broken SMTP config
  doesn't crash the nightly task — log the error and continue.
- CI non-blocking: add `continue-on-error: true` on the new job in
  `ci.yml`. Mark the job's display name with "(non-blocking)" so the
  signal is clear in PR reviews.

### Task 2: Implement the command (GREEN — part 1)

**Files:** `apps/planning/management/commands/validate_radius_assignments.py`

```python
# apps/planning/management/commands/validate_radius_assignments.py
import datetime as dt
import sys

from django.core.management.base import BaseCommand
from django.utils import timezone

from apps.accounts.models import User
from apps.kitchens.models import Kitchen
from apps.planning.models import MealPlan
from apps.planning.services.assignment import assign_meal_type
from apps.planning.services.exceptions import AddressMissingError


class Command(BaseCommand):
    help = "Verify every active member's fresh/frozen assignment for today."

    def add_arguments(self, parser):
        parser.add_argument("--date", help="ISO date (YYYY-MM-DD)")
        parser.add_argument("--quiet", action="store_true")

    def handle(self, *args, **options):
        target = (
            dt.date.fromisoformat(options["date"])
            if options.get("date") else timezone.localdate()
        )
        quiet = options["quiet"]

        plans_by_kitchen_date = {
            (p.kitchen_id, p.service_date): p
            for p in MealPlan.objects.filter(service_date=target)
        }

        members = User.objects.filter(role="member", is_active=True)
        kitchens = list(Kitchen.objects.all())

        ok = bad = skipped = 0
        for member in members:
            for kitchen in kitchens:
                try:
                    expected = assign_meal_type(member, kitchen, target)
                except AddressMissingError:
                    skipped += 1
                    break  # one skip per member is enough
                plan = plans_by_kitchen_date.get((kitchen.id, target))
                if plan is None:
                    continue   # no plan for this (kitchen, date) — nothing to validate
                actual = plan.meal_type
                if expected != actual:
                    bad += 1
                    if not quiet:
                        self.stdout.write(
                            f"member {member.id} {member.email}  "
                            f"kitchen {kitchen.id}  "
                            f"expected={expected}  actual={actual}"
                        )
                else:
                    ok += 1

        if skipped and not quiet:
            self.stdout.write(f"skipped: {skipped} member(s) without addresses")

        if bad:
            self.stdout.write(self.style.ERROR(
                f"FAIL: {bad} inconsistency(ies); {ok} consistent"
            ))
            sys.exit(1)
        self.stdout.write(self.style.SUCCESS(f"OK: {ok} consistent"))
        sys.exit(0)
```

### Task 3: The nightly Q2 task (GREEN — part 2)

**Files:** `apps/planning/tasks/validate.py`

```python
# apps/planning/tasks/validate.py
import io
import logging

from django.conf import settings
from django.core import mail
from django.core.management import call_command

log = logging.getLogger(__name__)


def run_nightly_validation() -> int:
    """Run validate_radius_assignments; email admin on failure.

    Returns the command's exit code so Q2 can record it.
    """
    out = io.StringIO()
    code = 0
    try:
        call_command("validate_radius_assignments", stdout=out)
    except SystemExit as exc:
        code = int(exc.code or 0)

    if code != 0:
        body = out.getvalue() or "validate_radius_assignments exited non-zero"
        admin_to = getattr(settings, "ADMIN_EMAIL", None)
        if admin_to:
            try:
                mail.send_mail(
                    subject="[MerryMeal] validate_radius_assignments FAILED",
                    message=body,
                    from_email=getattr(settings, "DEFAULT_FROM_EMAIL",
                                       "noreply@merrymeal.example"),
                    recipient_list=[admin_to],
                    fail_silently=True,
                )
            except Exception:
                log.exception("Failed to email admin about validation failure")
    return code
```

## Pitfalls reviewer will check

- **`sys.exit` inside `call_command`.** It raises `SystemExit`; tests
  must use `pytest.raises(SystemExit)` and read `.value.code`. Easy to
  miss and the test silently passes the failure path.
- **"Today" in UTC.** `dt.date.today()` is UTC; use
  `timezone.localdate()` for Melbourne. The freezegun tests pin this so
  it can't drift unnoticed.
- **N members × M kitchens explosion.** v1 walks all pairs in Python.
  Acceptable up to ~10k×10. If profiling shows slowness, push to a SQL
  bulk computation in a follow-up; **don't** preempt.
- **Email storm.** If the nightly task fails for a month while no one
  notices, an admin gets 30 emails. Acceptable for v1, but note in the
  retro to revisit.
- **CI non-blocking forever.** Add a TODO in the workflow comment:
  "flip to required after one clean week of staging runs."
- **`AddressMissingError` masks real bugs.** A member with no address is
  *skipped*, not failed. Reviewer will check that the skipped count
  appears in the output.

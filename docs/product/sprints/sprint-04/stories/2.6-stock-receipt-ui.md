## User story

> **As a** kitchen staff member
> **I want** a single screen to record a new batch (ingredient, quantity, expiry, lot)
> **so that** I do not have to learn the Django admin to log stock arrivals.

## Acceptance criteria

- [ ] Route `/kitchen/stock/receive/` resolves to `kitchens:stock_receive`.
- [ ] Form fields: `kitchen` (auto-selected and read-only if the user has access to exactly one), `ingredient`, `quantity`, `received_at` (defaults to today), `expiration_date` (required), `lot_number` (optional).
- [ ] On submit (`POST` valid): one `IngredientBatch` row is created by `apps.kitchens.services.stock.receive_batch(...)`.
- [ ] On success: redirect to `/kitchen/stock/receive/?created=<pk>` which shows a green toast + a "Receive another?" CTA.
- [ ] Restricted by `@role_required("kitchen_staff", "admin")`. `member` users get 403.
- [ ] Mobile-first at **375 px**; tablet layout (`md:` breakpoint ≥ 768 px) adds a second column for the optional fields (`received_at`, `lot_number`).
- [ ] All interactive controls have a minimum **44 px** touch-target height.
- [ ] Playwright test in `tests_e2e/stock_receive.spec.ts` verifies the form at 375 px and 768 px.

## Tech / library notes

- **Layering.** Follow `apps/accounts/` exactly: `forms/` (HTML + validation), `services/` (state change), `views/` (HTTP thin). No business logic in views.
- **Role gate.** Use `@role_required("kitchen_staff", "admin")` from `apps.core.decorators` (already exists per Sprint 01).
- **Kitchen scoping (simple version for this sprint).** Until "user ↔ kitchen" membership lands (Sprint 06), use this rule: if `Kitchen.objects.count() == 1`, auto-select and render it as read-only text; otherwise show a dropdown. This keeps the demo true to the AC without coupling to a membership model we don't have yet.
- **`received_at` default.** Server-side default = `date.today()` in `__init__` if not provided. Template uses `<input type="date" value="{{ today|date:'Y-m-d' }}">` so the field is editable.
- **Mobile-first layout.** Single column at 375 px; at `md:` (≥ 768 px), wrap the form in `md:grid md:grid-cols-2 md:gap-6` and put optional fields in the right column. Use the existing `.input`, `.btn-primary`, `.card`, `.label` classes from `static/src/input.css`.
- **44 px floor.** The shared `.input` and `.btn-primary` classes are already at `min-h-[44px]`; keep that contract by **not** overriding it locally.
- **Success UX.** Redirect (POST-redirect-GET) to `?created=<pk>`. On GET, if `created` is present, render a green toast block and a "Receive another?" link that points to the same URL without the query string. No JS needed for the toast.
- **Playwright.** Reuse the config from Story 0.11. Add a second project entry for tablet (`{ width: 768, height: 1024 }`) so a single `npm run test:e2e` covers both viewports.

```python
from datetime import date
from decimal import Decimal
from typing import Optional

from django.db import transaction

from apps.kitchens.models import IngredientBatch, Ingredient, Kitchen


@transaction.atomic
def receive_batch(
    *,
    user,
    kitchen: Kitchen,
    ingredient: Ingredient,
    quantity: Decimal,
    expiration_date: date,
    received_at: Optional[date] = None,
    lot_number: Optional[str] = None,
) -> IngredientBatch:
    """Record an incoming stock batch. Single source of truth for the
    business rules around stock receipt.

    Args:
        user: the staff member performing the action (used for auditing
              once an audit log exists — currently unused, kept for the
              upcoming Sprint 05 audit story).
        kitchen, ingredient, quantity, expiration_date: required fields.
        received_at: defaults to today.
        lot_number: optional supplier-provided lot.

    Raises:
        ValidationError if any field is invalid (quantity <= 0,
        expiration_date < received_at).
    """
    batch = IngredientBatch(
        kitchen=kitchen,
        ingredient=ingredient,
        quantity=quantity,
        expiration_date=expiration_date,
        received_at=received_at or date.today(),
        lot_number=lot_number or None,
    )
    batch.full_clean()
    batch.save()
    return batch
```

```python
from datetime import date
from decimal import Decimal

from django import forms

from apps.kitchens.models import Ingredient, IngredientBatch, Kitchen


class StockReceiveForm(forms.Form):
    kitchen = forms.ModelChoiceField(queryset=Kitchen.objects.all())
    ingredient = forms.ModelChoiceField(queryset=Ingredient.objects.all().order_by("name"))
    quantity = forms.DecimalField(
        max_digits=10,
        decimal_places=2,
        min_value=Decimal("0.01"),
    )
    received_at = forms.DateField(
        required=True,
        initial=date.today,
        widget=forms.DateInput(attrs={"type": "date"}),
    )
    expiration_date = forms.DateField(
        required=True,
        widget=forms.DateInput(attrs={"type": "date"}),
    )
    lot_number = forms.CharField(required=False, max_length=80)

    def clean(self):
        cleaned = super().clean()
        received = cleaned.get("received_at")
        expiration = cleaned.get("expiration_date")
        if received and expiration and expiration < received:
            self.add_error(
                "expiration_date",
                "Expiration date must be on or after the received date.",
            )
        return cleaned
```

```python
from datetime import date

from django.shortcuts import redirect, render
from django.urls import reverse

from apps.core.decorators import role_required
from apps.kitchens.forms.stock import StockReceiveForm
from apps.kitchens.models import Kitchen
from apps.kitchens.services.stock import receive_batch


@role_required("kitchen_staff", "admin")
def stock_receive_view(request):
    kitchens = Kitchen.objects.all()
    single_kitchen = kitchens.first() if kitchens.count() == 1 else None

    if request.method == "POST":
        form = StockReceiveForm(request.POST)
        if form.is_valid():
            batch = receive_batch(
                user=request.user,
                kitchen=form.cleaned_data["kitchen"],
                ingredient=form.cleaned_data["ingredient"],
                quantity=form.cleaned_data["quantity"],
                expiration_date=form.cleaned_data["expiration_date"],
                received_at=form.cleaned_data["received_at"],
                lot_number=form.cleaned_data.get("lot_number") or None,
            )
            return redirect(f"{reverse('kitchens:stock_receive')}?created={batch.pk}")
    else:
        initial = {"received_at": date.today()}
        if single_kitchen:
            initial["kitchen"] = single_kitchen.pk
        form = StockReceiveForm(initial=initial)

    return render(
        request,
        "kitchens/stock/receive.html",
        {
            "form": form,
            "single_kitchen": single_kitchen,
            "just_created_id": request.GET.get("created"),
        },
    )
```

## Pitfalls reviewer will check

- **Business logic in the view, not the service.** If `receive_batch` is bypassed and the view writes `IngredientBatch.objects.create(...)` directly, the future Sprint 06 stock-deduction will not see the same code path. Single entry point.
- **`@role_required` returns 302 for anonymous, 403 for wrong-role.** Tests cover both — do not loosen.
- **Kitchen auto-selection is a render concern, not a security one.** The form still accepts `kitchen` as a `ModelChoiceField`; a malicious user could POST any kitchen id. That is fine for this sprint (the kitchen-membership model lands Sprint 06); add a `# TODO(sprint-06)` next to `cleaned_data["kitchen"]` so the next reader knows.
- **`<input type="date">` is the mobile-native picker** — do not replace it with a custom JS calendar. Cooks on Android tablets get the OS picker for free.
- **44 px floor enforced via `.input` and `.btn-primary`** — never set `h-8` on these elements.
- **PRG (Post-Redirect-Get).** Submitting must redirect; refreshing the success page must not re-create a batch. Tests cover the redirect; manual refresh confirms.
- **`<label>`s wired to inputs by `for=id_*`** — the Playwright `getByLabel(/quantity/i)` selector depends on Django's default `id_<field>` ids being present.
- **`tablet-chrome` Playwright project, not a duplicate config file.** Adding a project is one paragraph; cloning the config is a maintenance trap.

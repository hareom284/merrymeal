## User story

> **As a** visitor (member or caregiver)
> **I want** a clear landing page with one "Apply" button
> **so that** I do not have to read marketing copy to find the entry point.

## Acceptance criteria

- [ ] Route `GET /` renders the landing page **for anonymous visitors**
      (no login redirect).
- [ ] Logged-in members are still redirected to their dashboard at `/` —
      the landing view distinguishes by `request.user.is_authenticated`.
- [ ] One full-width hero section: the charity tagline + a single
      `<a class="btn-primary">Apply for meals</a>` linking to `/apply/`
      (which is a 404 in this sprint; Story 1.7 wires it).
- [ ] Three cards below the hero:
  - "For members" → secondary link to `/apply/`.
  - "For caregivers" → secondary link to `/apply/?for_other=1`.
  - "For volunteers" → secondary link to `mailto:hello@merrymeal.org.au`.
- [ ] Mobile-first at 375 px:
  - Hero stacks above the cards.
  - Cards stack vertically below 768 px.
  - All interactive targets ≥ 44 px tall.
  - No horizontal scrollbar (`document.documentElement.scrollWidth <= 375`).
- [ ] At ≥ 768 px (Tailwind `md:`), the three cards lay out in a 3-column
      grid below the hero.
- [ ] WCAG AA contrast on every text block (tested via the `axe` browser
      extension; findings — even "no violations" — pasted into the PR body).
- [ ] Page title is `MerryMeal — Warm meals, delivered with care`.

> **Why move the member dashboard.** Currently `/` is `dashboards:member`
> (login-required). Adding a public landing page at `/` requires the
> member dashboard to move. New target: `/dashboard/`. Old test
> `test_dashboard_lives_at_root` will need to be updated to
> `test_dashboard_lives_at_slash_dashboard` — this is expected.

## Tech / library notes

- Reuse `templates/base.html` (already mobile-first, max-w-md). For a
  full-bleed hero on the landing page, override the `content` block and use
  Tailwind utilities; do NOT alter `base.html`'s `<main>` wrapper.
- Tailwind responsive classes:
  - Stack-by-default + 3-col at `md:`: `grid grid-cols-1 gap-4 md:grid-cols-3`.
  - 44 px tap target on every `<a class="btn-primary | btn-secondary">` —
    already enforced by `static/src/input.css` (`button, [role="button"] {
    min-height: 44px; }`); using the existing button classes inherits that.
- No new Django app needed — keep landing in `dashboards`, since the file
  already owns "what does the root URL render?".
- Playwright config (`tests_e2e/playwright.config.ts`) already runs at
  `375 × 667`. Add a new spec; do not change the config.

**Files:** `apps/dashboards/views/landing.py`,
`apps/dashboards/views/__init__.py`

- [ ] **Step 3.1 — Create `apps/dashboards/views/landing.py`.**

```python
from django.shortcuts import redirect, render


def landing_view(request):
    """Public landing page at `/`.

    Logged-in users are redirected to their dashboard; everyone else gets
    the marketing landing.
    """
    if request.user.is_authenticated:
        return redirect("/dashboard/")
    return render(request, "dashboards/landing.html")
```

### Task 7: Write the failing Playwright spec (RED — mobile)

- [ ] **Step 7.1 — Create `tests_e2e/landing.spec.ts`.**

```typescript
import { test, expect } from '@playwright/test';

test.describe('Landing page — mobile viewport floor (375 × 667)', () => {
  test('hero CTA "Apply for meals" is visible without scrolling', async ({ page }) => {
    await page.goto('/');
    const cta = page.getByRole('link', { name: /apply for meals/i });
    await expect(cta).toBeVisible();
    const box = await cta.boundingBox();
    expect(box).not.toBeNull();
    // Hero CTA must be inside the initial 667 px viewport.
    expect(box!.y + box!.height).toBeLessThanOrEqual(667);
  });

  test('CTA is at least 44 px tall', async ({ page }) => {
    await page.goto('/');
    const cta = page.getByRole('link', { name: /apply for meals/i });
    const box = await cta.boundingBox();
    expect(box!.height).toBeGreaterThanOrEqual(44);
  });

  test('three persona cards are present and stack vertically at 375 px', async ({ page }) => {
    await page.goto('/');
    const memberCard = page.getByRole('heading', { name: /for members/i });
    const caregiverCard = page.getByRole('heading', { name: /for caregivers/i });
    const volunteerCard = page.getByRole('heading', { name: /for volunteers/i });

    await expect(memberCard).toBeVisible();
    await expect(caregiverCard).toBeVisible();
    await expect(volunteerCard).toBeVisible();

    // Vertical stack at 375 px: caregiver's top is below member's bottom.
    const memberBox = await memberCard.boundingBox();
    const caregiverBox = await caregiverCard.boundingBox();
    expect(caregiverBox!.y).toBeGreaterThan(memberBox!.y + memberBox!.height);
  });

  test('no horizontal scrollbar at 375 px', async ({ page }) => {
    await page.goto('/');
    const scrollWidth = await page.evaluate(
      () => document.documentElement.scrollWidth
    );
    expect(scrollWidth).toBeLessThanOrEqual(375);
  });
});
```

## Pitfalls reviewer will check

- **`/` is unauthenticated.** A reviewer hitting `/` while logged out must
  see the landing, NOT a login redirect.
- **Authenticated users still go to `/dashboard/`.** The view returns a 302
  with `Location: /dashboard/`; do not render the landing for an authed user.
- **Member dashboard URL name unchanged:** `{% url 'dashboards:member' %}`
  must resolve to `/dashboard/`. Any template that hard-codes `"/"` is now
  broken — search for `href="/"` in `templates/` and fix.
- **`/apply/` is a 404 in this sprint.** That is intentional; Story 1.7
  wires it. The href is hard-coded — do not introduce a `{% url %}` lookup
  for a route that does not yet exist (the test runner will choke).
- **Use existing `.btn-primary` / `.btn-secondary` classes** — they already
  enforce ≥ 44 px. Inline Tailwind that bypasses these is a bug.
- **No login redirect on `/`.** A missing `landing_view` and accidentally
  leaving `member_dashboard_view` mapped to `""` will pass the
  authenticated-user test but fail the anonymous one.
- **`mailto:` link** for volunteers — not `/apply/?role=volunteer`. The
  volunteer onboarding flow is out of Epic 01.
- **Hero CTA visible without scrolling at 375 × 667** — this is the
  Playwright assertion; if the page header (Inter at 17 px base) pushes the
  CTA below the fold, tighten the hero's vertical spacing.

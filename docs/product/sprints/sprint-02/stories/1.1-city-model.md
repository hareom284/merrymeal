## User story

> **As an** admin
> **I want** a list of cities to choose from when creating addresses
> **so that** members and kitchens share a controlled city vocabulary instead of
> free-text typos.

## Tech / library notes

- `City` subclasses `apps.core.models.SoftDeleteModel` — that mixin already
  brings `created_at`, `updated_at`, `deleted_at`, the default
  `SoftDeleteManager`, and an `all_objects = AllObjectsManager()` escape hatch.
- The schema column is `name VARCHAR(120) NOT NULL`. In Django:
  `models.CharField(max_length=120)`.
- **Models are schema-only.** No `soft_delete()` method on the model — call
  `apps.core.services.soft_delete(city)` from tests / services.
- **Django admin is registered but not yet URL-mounted** in this project
  (`config/urls.py` deliberately omits it). Story 1.1 only asserts the
  registration exists (`admin.site._registry`); when the admin URL is wired
  later, the City admin will work.
- Use `pytest` + `pytest-django` markers (`@pytest.mark.django_db`).

```python
from django.db import models

from apps.core.models import SoftDeleteModel


class City(SoftDeleteModel):
    """Controlled vocabulary of delivery cities.

    Schema only. Soft delete is performed via
    `apps.core.services.soft_delete(city)`, not on the model itself.
    """

    name = models.CharField(max_length=120)

    class Meta:
        app_label = "accounts"
        db_table = "cities"
        ordering = ("name",)

    def __str__(self) -> str:
        return self.name
```

```python
from django.core.management.base import BaseCommand

from apps.accounts.models import City


STARTER_CITIES = ["Melbourne", "Geelong", "Ballarat", "Bendigo", "Frankston"]


class Command(BaseCommand):
    help = "Seed the cities table with starter Victorian cities. Idempotent."

    def handle(self, *args, **options):
        created = 0
        for name in STARTER_CITIES:
            _, was_created = City.objects.get_or_create(name=name)
            if was_created:
                created += 1
        self.stdout.write(
            self.style.SUCCESS(
                f"seed_cities: {created} created, {len(STARTER_CITIES) - created} already existed."
            )
        )
```

## Pitfalls reviewer will check

- **`City` is registered, not just imported.** A bare `from .models import City`
  in `admin.py` without `@admin.register(City)` will silently leave it
  unregistered.
- **`name` length is 120, not 255.** Schema says `VARCHAR(120)`. Wider drift
  here means a future MySQL DDL audit fails.
- **No `delete()` override on `City`.** Soft delete is performed via
  `apps.core.services.soft_delete(city)`. The model must remain schema-only.
- **Seed list is exactly 5 cities, exact spellings.** "Melbourne CBD",
  "Frankston North", typos, or duplicates will all break Story 1.5's seed
  pattern that copies this one.
- **`db_table = "cities"`.** Django's default would be `accounts_city` — wrong
  and will fail the `test_city_db_table_is_cities` assertion.
- **Migration is named `0002_city.py`** (auto-generated). Do not rename it; do
  not squash 0001 + 0002.

## User story

> **As a** kitchen staff member
> **I want** a one-screen form on my tablet/phone to log today's safety checks
> **so that** compliance is a 2-minute habit, not a paperwork chore.

## Acceptance criteria

- [ ] Route `/kitchen/safety/check/` rendered by `apps.food_safety.views.check.check_view`.
- [ ] The form has one **check-type selector** at the top (radio or segmented control). Selecting a type swaps the rest of the form via Alpine.js `x-show`:
  - `storage_temp`, `cooking_temp`, `cold_chain` → `temperature_celsius` input (°C, `step=0.1`, required), notes textarea (optional).
  - `hygiene`, `cleaning`, `pest_control` → pass/fail radio (required), notes textarea (optional).
- [ ] On submit:
  - `checked_by = request.user`, `checked_at = timezone.now()` set by the view — never trusted from the request.
  - Temp-only types: `result` derives from a threshold map (see Tech notes); the user does **not** pick pass/fail directly.
  - Hygiene-style types: `result` comes from the radio.
  - A row is created via `apps.food_safety.services.checks.record_check(...)`.
- [ ] On success: green Tailwind toast "Saved." appears for 3 s; the page reloads (or the "already done today" list re-renders).
- [ ] **Already-done-today list** at the top of the page shows that user's check rows for the current local date, grouped by check_type, with the result badge (green / red).
- [ ] Restricted to roles `kitchen_staff` and `admin` via `@role_required('kitchen_staff', 'admin')`. Anyone else → 403.
- [ ] Mobile-first at **375 × 667 px**:
  - Minimum 44 px touch targets on every input, radio, and button.
  - No horizontal scroll at 375 px.
  - Form is single-column at `<sm` breakpoint, two-column from `md:` upwards (tablet).
- [ ] Playwright passes at 375 px (mobile) and at 768 px (iPad portrait).

## Tech / library notes

- **Threshold map for temp-type results.** Per Australian food-safety guidelines (FSANZ 3.2.2):
  - `storage_temp` PASS if `temperature_celsius <= 5.0` (cold) or `>= 60.0` (hot hold).
  - `cooking_temp` PASS if `temperature_celsius >= 75.0`.
  - `cold_chain` PASS if `temperature_celsius <= 5.0`.
  - Anything outside those bands → FAIL.
  - The thresholds live in `apps.food_safety.services.checks.THRESHOLDS` as a dict so they are one-line to tune. **Do not hard-code them in the view or template.**
- **Service layer is the only writer.** Views never call `FoodSafetyCheck.objects.create(...)` directly. The service takes plain kwargs and returns the created instance. This keeps the view trivial and the service testable without a request.
- **`checked_at` is computed in the service**, not in the form. Forms cannot be trusted to set "now" because a malicious payload could backdate a check.
- **"Already done today" query.** Service helper `today_checks_for(user)` returns `QuerySet[FoodSafetyCheck]` filtered to `checked_by=user` and `checked_at__date == localdate()`. Use `django.utils.timezone.localdate()` so the day boundary respects `Australia/Melbourne`, not UTC.
- **Alpine.js for the conditional rendering.** `x-data="{ type: 'storage_temp' }"` on the form, then `x-show="['storage_temp','cooking_temp','cold_chain'].includes(type)"` on the temperature block, etc. Alpine is already loaded in `base.html` (see README).
- **No HTMX swap for the success state** — a plain POST + redirect is simpler and lets `messages.success(...)` carry through the toast. HTMX is overkill for a single-form page.
- **Form widget classes.** Use the project's `.input`, `.label`, `.btn-primary` Tailwind component classes from `static/src/input.css` so the form matches every other screen.
- **44 px touch targets** — `class="h-11"` on inputs and buttons (44 px = 11 × 4 px Tailwind unit). Radios get `class="h-6 w-6"` plus an `aria-label` so the tap target is the whole `<label class="block py-3">` row, not the 24-px circle.

```python
from decimal import Decimal
from typing import Optional

from django.db.models import QuerySet
from django.utils import timezone

from apps.food_safety.models import FoodSafetyCheck


# Result thresholds for temperature-typed checks.
# (low_inclusive, high_inclusive) — PASS if any inclusive band matches.
THRESHOLDS: dict[str, list[tuple[Optional[Decimal], Optional[Decimal]]]] = {
    "storage_temp": [(None, Decimal("5.0")), (Decimal("60.0"), None)],
    "cooking_temp": [(Decimal("75.0"), None)],
    "cold_chain":   [(None, Decimal("5.0"))],
}


def derive_result(check_type: str, temperature_celsius: Decimal) -> str:
    bands = THRESHOLDS.get(check_type)
    if bands is None:
        raise ValueError(f"derive_result not defined for {check_type!r}")
    for low, high in bands:
        if (low is None or temperature_celsius >= low) and \
           (high is None or temperature_celsius <= high):
            return FoodSafetyCheck.Result.PASS
    return FoodSafetyCheck.Result.FAIL


def record_check(*, kitchen, user, check_type: str,
                 temperature_celsius: Optional[Decimal],
                 result: Optional[str],
                 notes: str) -> FoodSafetyCheck:
    """Write one food-safety check. Server owns checked_at and derives result
    for temperature types."""
    if check_type in THRESHOLDS:
        if temperature_celsius is None:
            raise ValueError("temperature_celsius required for temp check")
        result = derive_result(check_type, temperature_celsius)
    else:
        if result not in {"pass", "fail"}:
            raise ValueError("result required for non-temp check")
        temperature_celsius = None

    return FoodSafetyCheck.objects.create(
        kitchen=kitchen,
        check_type=check_type,
        temperature_celsius=temperature_celsius,
        result=result,
        checked_by=user,
        checked_at=timezone.now(),
        notes=notes or "",
    )


def today_checks_for(user) -> QuerySet[FoodSafetyCheck]:
    today = timezone.localdate()
    return (
        FoodSafetyCheck.objects
        .filter(checked_by=user, checked_at__date=today)
        .order_by("checked_at")
    )
```

from django import forms

from apps.food_safety.models import FoodSafetyCheck
from apps.food_safety.services.checks import THRESHOLDS


_TEMP_TYPES = set(THRESHOLDS.keys())


class CheckForm(forms.Form):
    check_type = forms.ChoiceField(choices=FoodSafetyCheck.CheckType.choices)
    temperature_celsius = forms.DecimalField(
        max_digits=5, decimal_places=2, required=False,
        widget=forms.NumberInput(attrs={"step": "0.1", "class": "input h-11"}),
    )
    result = forms.ChoiceField(
        choices=[("", "—")] + list(FoodSafetyCheck.Result.choices),
        required=False,
    )
    notes = forms.CharField(
        required=False,
        widget=forms.Textarea(attrs={"rows": 3, "class": "input"}),
    )

    def clean(self):
        data = super().clean()
        check_type = data.get("check_type")
        if check_type in _TEMP_TYPES:
            if data.get("temperature_celsius") in (None, ""):
                self.add_error("temperature_celsius",
                               "Temperature is required for this check.")
            data["result"] = ""           # server derives it; drop any client input
        else:
            if data.get("result") not in {"pass", "fail"}:
                self.add_error("result", "Choose pass or fail.")
            data["temperature_celsius"] = None
        return data
```

from django import forms
from django.contrib import messages
from django.shortcuts import redirect, render
from django.views.decorators.http import require_http_methods

from apps.core.decorators import role_required
from apps.food_safety.forms.check import CheckForm
from apps.food_safety.services.checks import record_check, today_checks_for
from apps.kitchens.models import Kitchen


class _KitchenChoice(forms.Form):
    kitchen = forms.ModelChoiceField(queryset=Kitchen.objects.all())


@role_required("kitchen_staff", "admin")
@require_http_methods(["GET", "POST"])
def check_view(request):
    if request.method == "POST":
        kitchen_form = _KitchenChoice(request.POST)
        form = CheckForm(request.POST)
        if kitchen_form.is_valid() and form.is_valid():
            record_check(
                kitchen=kitchen_form.cleaned_data["kitchen"],
                user=request.user,
                check_type=form.cleaned_data["check_type"],
                temperature_celsius=form.cleaned_data["temperature_celsius"],
                result=form.cleaned_data["result"] or None,
                notes=form.cleaned_data["notes"],
            )
            messages.success(request, "Saved.")
            return redirect("food_safety:safety-check")
    else:
        kitchen_form = _KitchenChoice()
        form = CheckForm(initial={"check_type": "storage_temp"})

    return render(request, "food_safety/check.html", {
        "form": form,
        "kitchen_form": kitchen_form,
        "today_rows": today_checks_for(request.user),
    })
```

## Pitfalls reviewer will check

- **`checked_at` derived in the view, not the form.** A form field for `checked_at` is a trust bug — a tampered POST could backdate a check. The service calls `timezone.now()`.
- **Result for temp checks comes from `derive_result`, not from the form.** If the template shows pass/fail radios when `check_type=storage_temp`, that is a bug — cook is reporting a temperature, not a verdict.
- **`role_required` decorator is on the view function**, not added as a TODO. The test for the `member` role returning 403 is non-negotiable.
- **`timezone.localdate()` for the "today" filter** — not `date.today()` (UTC) and not `datetime.now().date()`. Australia/Melbourne is UTC+10/+11; the day boundary matters at 6 am and 2 pm local time when the cook hands over to the next shift.
- **Touch targets ≥ 44 px.** `h-11` Tailwind class on every input and button. The Playwright test enforces this; do not weaken the assertion.
- **No horizontal scroll at 375 px.** Common cause: the check-type grid uses `min-w-[400px]`. Use `grid-cols-2` (which is what the template above does) and let it grow at `md:`.
- **`x-cloak`** on the Alpine-conditional blocks so they do not flash on first paint before Alpine boots. If you forget, the form briefly shows both widgets and looks broken.
- **Validation errors render inline** under the field — not as a Django form-wide `{% if form.errors %}` block at the top. The test asserts `name="result"` errors are visible next to the radios.
